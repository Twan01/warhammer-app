## 1. Design System

### shadcn/ui Configuration

| Token | Value |
|-------|-------|
| Style | New York |
| Base color | Zinc |
| Border radius | `0.5rem` |

These values are passed to the shadcn CLI at install time and written into `components.json`. Do not deviate from New York style — it produces denser, sharper components that match the "serious dashboard" aesthetic.

### Dark Mode Token Map

All tokens below are CSS custom properties set in the `:root.dark` (or `.dark`) block inside `src/styles/globals.css`. These override shadcn's generated defaults to hit the intended palette.

| Role | CSS variable | Resolved value |
|------|-------------|---------------|
| Page background | `--background` | `zinc-950` (~`#09090b`) |
| Card / surface | `--card` | `zinc-900` (~`#18181b`) |
| Popover / dropdown | `--popover` | `zinc-900` |
| Primary action | `--primary` | `zinc-50` |
| Primary foreground | `--primary-foreground` | `zinc-900` |
| Secondary surface | `--secondary` | `zinc-800` |
| Secondary foreground | `--secondary-foreground` | `zinc-50` |
| Muted surface | `--muted` | `zinc-800` |
| Muted text | `--muted-foreground` | `zinc-400` |
| Accent fill | `--accent` | `zinc-800` |
| Accent text | `--accent-foreground` | `zinc-100` |
| Destructive | `--destructive` | `red-500` |
| Border | `--border` | `zinc-800` |
| Input border | `--input` | `zinc-800` |
| Focus ring | `--ring` | `zinc-300` |

Light mode tokens are generated by shadcn defaults and left as-is — the app is dark-mode-only in v1 (SETUP-04), so light tokens are never applied at runtime.

### Typography

shadcn New York ships with `font-sans` (system font stack). No custom font is introduced in Phase 1. Body text uses `zinc-50` (`--foreground`); secondary / helper text uses `zinc-400` (`--muted-foreground`).

### Border Radius

`--radius: 0.5rem` (set in `globals.css`). Shadcn derives `--radius-sm`, `--radius-md`, `--radius-lg` from this. Cards, dialogs, and sidebar items use `rounded-md` (`--radius-md`).

## 3. Layout Structure

The app shell is a two-column layout: a sidebar on the left and a main content area on the right. The sidebar collapses to icon-only mode; the main area fills all remaining width.

```
┌──────────────────────────────────────────────────────────┐
│  Sidebar (240px expanded / 48px collapsed)  │  Main area │
│                                              │            │
│  [Logo / App name]                           │  <Router   │
│                                              │   outlet>  │
│  Nav items (icons + labels)                  │            │
│  ─────────────────── (spacer)                │            │
│  Settings (pinned bottom)                    │            │
│                                              │            │
│  [Collapse toggle]                           │            │
└──────────────────────────────────────────────────────────┘
```

The layout root is a `flex h-screen` container. The sidebar has `flex-shrink-0`; the main area has `flex-1 overflow-auto`.

## 4. Sidebar

### Dimensions

| State | Width |
|-------|-------|
| Expanded | `240px` |
| Collapsed (icon-only) | `48px` |

Transition: `transition-width duration-200 ease-in-out` (CSS transition, not JS animation).

### Collapse Persistence

Collapse state is stored in `localStorage` under the key `"sidebar:collapsed"` (boolean string `"true"` / `"false"`). On mount, the sidebar reads this key synchronously to avoid a layout flash. If the key is absent, default is expanded.

### Collapse Toggle Button

- Position: pinned to the bottom of the sidebar, above the Settings item
- Icon: `ChevronsLeft` (expanded state) / `ChevronsRight` (collapsed state) from `lucide-react`
- Button style: `ghost` variant, `size="icon"`, full width in expanded state, centered in collapsed state
- Aria label: `"Collapse sidebar"` / `"Expand sidebar"`

### App Name / Logo

- Expanded: app name `"HobbyForge"` in `font-semibold text-sm` next to a small shield/sword icon (lucide `Swords` or `Shield`)
- Collapsed: icon only, centered
- Located at the top of the sidebar, padding `p-4`

### Navigation Items

Use the shadcn `Sidebar` component tree (`SidebarProvider`, `Sidebar`, `SidebarContent`, `SidebarMenu`, `SidebarMenuItem`, `SidebarMenuButton`) as the structural base. This gives collapsible behavior, keyboard navigation, and tooltip support out of the box.

| Label | Icon (lucide-react) | Route |
|-------|---------------------|-------|
| Dashboard | `LayoutDashboard` | `/` |
| Collection | `Package` | `/collection` |
| Painting Projects | `Palette` | `/painting-projects` |
| Recipes | `BookOpen` | `/recipes` |
| Paints | `Droplets` | `/paints` |
| Settings | `Settings` | `/settings` |

Settings is visually separated from the main nav items by a `flex-1` spacer (pushes Settings to the bottom of the sidebar). It uses the same `SidebarMenuButton` style as other items.

### Active Item Highlight

- Style: `bg-accent text-accent-foreground rounded-md` (shadcn `isActive` prop on `SidebarMenuButton`)
- Current route detected via TanStack Router's `useMatchRoute` or the `SidebarMenuButton`'s `isActive` prop wired to route match

### Icon-Only Mode (Collapsed)

- Labels are hidden (`sr-only` or `hidden`)
- Each nav item shows a `Tooltip` (shadcn `TooltipProvider` + `Tooltip`) with the label on hover
- Tooltip side: `"right"`

### Sidebar Background

`bg-zinc-900` (maps to `--card` / sidebar surface). A `border-r border-border` separates it from the main area.

## 5. Main Content Area

- Background: `bg-background` (`zinc-950`)
- Padding: `p-6` applied inside each placeholder page (not on the container itself — keeps the router outlet clean)
- Each placeholder page renders a centered `<h1>` with the page name in `text-2xl font-semibold` and a `text-muted-foreground` subline ("Coming soon in Phase N")

## 6. Dark Mode — FOUC Prevention

Per SETUP-04, add the following inline script as the **first child** of `<head>` in `index.html`, before any stylesheet `<link>`:

```html
<script>document.documentElement.classList.add('dark')</script>
```

This runs synchronously before paint. Because the app is dark-mode-only, the class is unconditionally applied — no `localStorage` check needed. Tailwind's `darkMode: 'class'` strategy activates all `dark:` variants via this class.

## 7. Placeholder Pages

All six routes render a placeholder component. The placeholder is a shared `PlaceholderPage` component that accepts `title` and `phase` props.

```
src/app/
  dashboard/page.tsx        → <PlaceholderPage title="Dashboard" phase={5} />
  collection/page.tsx       → <PlaceholderPage title="Collection" phase={3} />
  painting-projects/page.tsx → <PlaceholderPage title="Painting Projects" phase={4} />
  recipes/page.tsx          → <PlaceholderPage title="Recipes" phase={4} />
  paints/page.tsx           → <PlaceholderPage title="Paints" phase={2} />
  settings/page.tsx         → <PlaceholderPage title="Settings" phase={5} />
```

`PlaceholderPage` renders:

```tsx
<div className="flex flex-col items-center justify-center h-full gap-2 text-center p-6">
  <h1 className="text-2xl font-semibold">{title}</h1>
  <p className="text-muted-foreground text-sm">Coming in Phase {phase}</p>
</div>
```

## 10. Component Responsibilities (Phase 1)

| Component | File | Responsibility |
|-----------|------|----------------|
| `AppLayout` | `src/components/common/AppLayout.tsx` | Root flex container (sidebar + main area) |
| `AppSidebar` | `src/components/common/AppSidebar.tsx` | Sidebar shell, collapse toggle, localStorage persistence |
| `NavItem` | `src/components/common/NavItem.tsx` | Single sidebar nav entry with icon, label, tooltip, active state |
| `PlaceholderPage` | `src/components/common/PlaceholderPage.tsx` | Shared placeholder used by all 6 routes |
| `QueryProvider` | `src/components/common/QueryProvider.tsx` | TanStack Query `QueryClient` provider with desktop defaults |
| Route pages | `src/app/*/page.tsx` | One per route — renders `<PlaceholderPage />` in Phase 1 |

TanStack Query client defaults (SETUP-05):

```ts
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5,    // 5 minutes
      gcTime: 1000 * 60 * 10,      // 10 minutes
      refetchOnWindowFocus: false,
    },
  },
});
```
