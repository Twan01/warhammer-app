# Milestone v0.2.13: Data Integrity, Diagnostics & Product Coherence

**Status:** SHIPPED 2026-05-15
**Phases:** 73-78
**Total Plans:** 13

## Overview

Make HobbyForge trustworthy and guided — stable data identity, transactional writes, centralized points resolution, data health visibility, backup/export, and an actionable dashboard that tells the user what to do next.

## Phases

**Goal**: The schema is extended with two new tables and a CI-friendly version parity script guards against version drift
**Depends on**: Nothing (first phase of milestone)
**Requirements**: DI-05
**Plans**: 2 plans

- [x] 73-01-PLAN.md — Migrations 026+027 and lib.rs registration
- [x] 73-02-PLAN.md — Version parity script + version bumps to 0.2.13

### Phase 74: Applied Recipe Identity Hardening

**Goal**: Applied recipe step progress is keyed by recipe_step_id so reordering steps never moves completion markers
**Depends on**: Phase 73
**Requirements**: DI-01, DI-02
**Plans**: 2 plans

### Phase 75: Transactional Recipe Graph Save

**Goal**: Saving a recipe always completes fully or not at all — partial saves are structurally impossible
**Depends on**: Phase 73
**Requirements**: DI-03, DI-04
**Plans**: 2 plans

Plans:
- [x] 75-01-PLAN.md — saveRecipeGraph() transactional function + unit tests
- [x] 75-02-PLAN.md — Wire RecipeFormSheet onSubmit to saveRecipeGraph + batch invalidation

**Details:**
saveRecipeGraph() wraps the entire recipe save (metadata + sections + steps) in BEGIN/COMMIT/ROLLBACK. Five-phase diff (delete removed, update existing, insert new) maintained inside the transaction. 10 cache keys invalidated on success. No nested BEGIN calls.

## Milestone Summary

**Key Decisions:**
- Transactions: flat inline SQL only — tauri-plugin-sql cannot nest BEGIN/COMMIT
- Backup must use VACUUM INTO via Rust command, not raw file copy
- order_index back-fill SQL JOINs through recipe_sections for section disambiguation
- Points resolver: pure function in src/lib/ consumed by all query sites
- ReadyToPlayCard uses SQL COALESCE directly (not resolveUnitPoints()) — accepted architectural divergence
- promoted_to_reminder column vestigial — all forgotten rules surface as reminders unconditionally

**Issues Resolved:**
- COALESCE site-3 divergence: dashboard.ts upgraded to 4-level chain in Phase 76
- VACUUM INTO via tauri-plugin-sql: resolved by creating Rust backup_database command in Phase 77
- Cache invalidation for forgotten-rules query key added to all battle log mutations

**Tech Debt Carried Forward:**
- Dead column: promoted_to_reminder (migration 027) typed but never written/read
- Architectural divergence: ReadyToPlayCard computes points via SQL COALESCE instead of resolveUnitPoints()
