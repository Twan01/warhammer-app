# Milestone v0.2.15: Painting Mode

**Status:** SHIPPED 2026-05-20
**Phases:** 84-88
**Total Plans:** 11

## Overview

A focused, distraction-free Painting Mode that turns an applied recipe into a clear step-by-step execution experience at the painting desk. Keyboard-driven with Space/Arrow/Escape shortcuts, section-aware navigation, paint readiness warnings, and atomic session logging.

## Phases

### Phase 84: Data Layer + Early Tests

**Goal**: The data operations that power Painting Mode are correct, transactional, and tested before any UI exists
**Depends on**: Phase 83 (v0.2.14 shipped)
**Plans**: 2 plans
**Requirements**: DL-01, DL-02, DL-03, DL-04, TS-01, TS-02, TS-03

Plans:

- [x] 84-01: Atomic completeStepWithSession + useCompleteStep hook + tests
- [x] 84-02: usePaintingModeState navigation hook + tests

**Details:**
Transactional step completion (BEGIN/COMMIT wrapping step progress upsert + session insert), section-aware ordering (COALESCE section order_index then step order_index), 6-key React Query cache invalidation contract, navigation hook with first-incomplete selection, prev/next/jumpTo, per-section progress map, 11 TDD tests.

### Phase 85: Core Execution UI

**Goal**: User can execute a painting step from a focused, distraction-free view with all the information they need at the desk
**Depends on**: Phase 84
**Plans**: 3 plans
**Requirements**: SE-01..SE-05, SP-01..SP-04, PR-01..PR-03, PX-01

Plans:

- [x] 85-01: ScrollArea install + StepMetadataRow + PaintReadinessBanner + tests
- [x] 85-02: SectionNavigator component + tests
- [x] 85-03: StepFocalView + PaintingModeView root + integration tests

**Details:**

SectionNavigator left panel with collapsible sections, progress badges, optional section badges, clickable step sub-items. PaintReadinessBanner dismissible amber warning for missing paints. PaintingModeView root composing split-panel layout.
