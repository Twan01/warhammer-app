## Tech Debt (2 accepted items)

### Phase 78: Dead Column (promoted_to_reminder)

Migration 027 adds `promoted_to_reminder INTEGER NOT NULL DEFAULT 0` to battle_logs. The column is typed in `BattleLog` interface but neither `createBattleLog` nor `updateBattleLog` SQL includes it. `getRecentForgottenRules` ignores the flag — all forgotten rules surface as reminders unconditionally. The design chose automatic promotion (per CONTEXT D-15/D-16), making the column vestigial.

### Phase 76: Architectural Divergence (PV-01 adjacent)

`ReadyToPlayCard` computes total points via `effective_points` from SQL COALESCE instead of `resolveUnitPoints()`. The numeric result is identical (same 4-level COALESCE priority), but this creates two code paths for the same business rule. Minor — summary card doesn't need per-unit source labeling.
