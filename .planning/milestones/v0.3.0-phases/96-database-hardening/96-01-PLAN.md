<objective>
Harden the main hobbyforge.db database with WAL journal mode, indexes on all FK and temporal columns, and CHECK constraints preventing invalid data at the schema level.

Purpose: Eliminate SQLITE_BUSY risks during concurrent reads/writes, accelerate JOIN/WHERE/ORDER BY queries on FK and temporal columns via index scans, and prevent invalid data (negative points, out-of-range percentages) from ever reaching the database.

Output: Modified client.ts with WAL+busy_timeout PRAGMAs, new migration 033 with 32 index creations and 2 table recreations (units, paints) adding CHECK constraints, and migration registration in lib.rs.
</objective>
