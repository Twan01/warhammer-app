## v0.2.10 Requirements (All Complete)

### Recipe Hardening

- [x] **RH-01**: Fresh install and existing DB both create recipe_sections with all 4 workflow metadata columns (section_type, technique, execution_mode, applies_to) — Phase 61
- [x] **RH-02**: Section-aware log session uses stable section reference — renaming a section does not break session analytics — Phase 61
- [x] **RH-03**: Workflow metadata UX refined — section_type values match user mental model, progressive disclosure preserved for simple recipes — Phase 61

### Applied Recipes

- [x] **AR-01**: Schema: unit_recipe_assignments + unit_recipe_step_progress tables with stable composite key (survives DELETE-all + re-INSERT save) — Phase 62
- [x] **AR-02**: User can apply one or more recipes to a unit from Collection/Unit Detail with section/step preview — Phase 63
- [x] **AR-03**: User can tick sections/steps as completed for a specific unit assignment — progress stored separately from recipe template — Phase 63
- [x] **AR-04**: Applied recipe progress visible on unit detail — checklist-like section/step view with completion percentage — Phase 63
- [x] **AR-05**: Log Session can optionally complete an applied recipe step during session creation — Phase 64
- [x] **AR-06**: Kanban cards and CurrentFocusCard show applied recipe progress and next step (replaces session-derived position when available) — Phase 64
- [x] **AR-07**: User can apply same recipe to multiple selected units — each gets separate progress — Phase 63

### Points Import

- [x] **PI-01**: Schema: points import tables in hobbyforge.db with source_name, source_version, import_date, row_count — Phase 65
- [x] **PI-02**: User can import points from CSV file — validation, versioning, error logging, import history — Phase 65
- [x] **PI-03**: Points freshness visible — source name, version, import date, stale/fresh/unknown badges on army lists and rules hub — Phase 65
- [x] **PI-04**: After import, user can see per-unit points deltas (increased/decreased/new/removed) and affected army lists — Phase 65
- [x] **PI-05**: 5-level points resolution: list override > loadout override > imported points > unit default > unknown — applied atomically across all COALESCE sites — Phase 65

### List Validation

- [x] **LV-01**: Hard validation warnings on army list: points exceeded, unknown/stale points, manual override in use, unowned/unbuilt/unpainted/not-battle-ready units — Phase 66
- [x] **LV-02**: User can assign tactical role tags (anti_tank, screening, objective_holder, etc.) to units — Phase 66
- [x] **LV-03**: Army list shows tactical role coverage — aggregated tag strengths/weaknesses with visual indicators — Phase 66
- [x] **LV-04**: Army list detail shows health summary panel: points total, ownership %, readiness %, freshness status, warning count — Phase 66

### Game Day Integration

- [x] **GD-01**: Game Day pre-game warnings: surface points freshness, readiness gaps, tactical coverage warnings, and stale data alerts before playing — Phase 67
