**Status:** ✅ SHIPPED 2026-05-13

## Overview

Turn recipes into actionable painting plans with per-unit progress tracking, add a points import data layer with freshness tracking via Wahapedia sync, and harden army list validation with tactical role coverage and health summary panels. Game Day mode surfaces all validation warnings before a game.

## Milestone Summary

**Key Decisions:**
- synced_unit_points cache in hobbyforge.db solves cross-DB JOIN problem (rules.db cannot be JOINed directly)
- 5-level COALESCE: alu.points_override > sup.points > uo.points > u.points > 0
- Dashboard queries intentionally NOT updated with synced points (different concern)
- PointsFreshnessBadge is self-contained — queries useRulesSyncMeta internally
- clearArmyListPointsLimit solves COALESCE-blocks-NULL pitfall for points_limit
- GameDayReadinessPanel is a new presentation component (not ArmyListSummaryBar reuse)
- Full-replacement UPDATE on army_list_units must pass all fields (points_override, notes, tactical_role)

**Issues Resolved:**
- Migration registration gaps for 018-020 (fixed in Phase 61)
- Cross-DB JOIN limitation for points data (solved with synced_unit_points cache table)
- COALESCE blocks NULL clearing for points_limit (solved with dedicated clear function)

**Issues Deferred:**
- Phase 62 Nyquist non-compliant (wave_0_complete: false) — low severity
- Stale comment in armyLists.ts:196 references old 4-level COALESCE — info only

**Technical Debt Incurred:**
- Phases 61-64 lack VERIFICATION.md files (procedural gap; code verified via integration checker)
