## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, kbd badge padding |
| sm | 8px | Compact element spacing, button gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, StepFocalView top padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |

Exceptions:
- Mark Done button: `h-12` (48px height) — maintains existing touch target from StepFocalView

## Interaction Contracts

### StepFocalView — Button Pair Layout

Two separate buttons, not a split button. Layout: full-width primary "Mark Done" on top, full-width secondary "Done + Log Session" below it. No icon on the secondary button beyond text label (keeps visual weight clearly secondary).

```
[ ✓ Mark Done          Space ]   ← primary, h-12, variant="default"
[ Done + Log Session         ]   ← secondary, h-10, variant="outline"
```

- "Mark Done" button: `variant="default"`, `h-12`, full width, `Check` icon left, `Space` kbd badge right — **unchanged from existing**
- "Done + Log Session" button: `variant="outline"`, `h-10`, full width, no icon, `mt-2` spacing after "Mark Done"
- Both buttons: disabled when `isCompleted || isAllComplete`
- "Done + Log Session": opens `PaintingSessionSheet`; does NOT directly call `handleMarkDone`

Source: CONTEXT.md D-04, D-05 (user decision) + RESEARCH.md Pattern 2

### PaintingSessionSheet — Layout

Sheet side: `right`, width `sm` (fits the 2-field form). Rendered as sibling of `PaintingModeView` in `PaintingModePage` (not nested inside StepFocalView).

**Read-only prefill display** (top of sheet, before form fields):
- Unit name — `text-sm font-semibold`
- Recipe name — `text-sm text-muted-foreground`
- Section name (if present) — `text-xs text-muted-foreground`
- Step name — `text-xs text-muted-foreground`

Displayed in a `bg-muted rounded-md p-3` summary block above the form fields.

**User-editable fields** (form, react-hook-form + zod):
1. Duration (minutes) — `Input` type="number", min=1, max=1440, default=30, label "Duration (min)", placeholder "30"
2. Notes — `Input` (or `Textarea`), optional, max 2000 chars, label "Notes", placeholder "Optional notes"

**Footer actions**:
- Primary: "Save & Mark Done" — `variant="default"`, disabled while `isPending`
- Secondary: "Keep Working" — `variant="ghost"`, closes sheet without completing step

Source: CONTEXT.md D-01, D-02, D-03 + RESEARCH.md Pattern 1 + `src/features/dashboard/LogSessionSheet.tsx` reference

### EP-06 Empty State — No Applied Recipe

Shown in `PaintingModePage` when `assignment` is null after loading (invalid/direct URL access). Does NOT replace the existing "assignment not found" error; this is the content state when route is valid but recipe has no steps (edge case).

Entry point surfaces suppress the Paint CTA entirely when no assignment exists (D-14). The empty state is the safety net only.

Layout: centered in the main content area, consistent with `StepFocalView`'s `isAllComplete` state:
```
[ Palette icon — h-12 w-12 text-muted-foreground opacity-40 ]
[ Heading: "No painting recipe applied" — text-lg font-semibold ]
[ Body: "This unit doesn't have a painting recipe applied yet." — text-sm text-muted-foreground ]
[ CTA button: "Apply a Recipe" — variant="outline", navigates to unit detail ]
```

Source: CONTEXT.md D-13

## Keyboard & Accessibility

- Existing keyboard shortcuts (Space = mark done, ← → = navigate, Esc = exit) must remain fully functional when `PaintingSessionSheet` is closed
- When `PaintingSessionSheet` is open: Space and arrow shortcuts are automatically suppressed because `react-hotkeys-hook` blocks on focused `HTMLInputElement` (verified in Phase 86 PX-05)
- All new buttons must have accessible text (either visible label or `aria-label` for icon-only buttons)
- `PaintingSessionSheet` Sheet follows existing SheetHeader / SheetTitle / SheetDescription pattern — screen readers get context from SheetDescription
- All entry point buttons: use visible text labels, not icon-only buttons (14px icon + text label)

Source: REQUIREMENTS.md PX-05 (already complete) + RESEARCH.md Pitfall 2
