## Tech Debt

No critical blockers. Two non-critical items accumulated.

### Item 1: PROJ-02 Requirements Doc Inconsistency (Phase 4)

**File:** `.planning/REQUIREMENTS.md`, PROJ-02 entry  
**Issue:** REQUIREMENTS.md text still says "Cards grouped into columns by `status_painting`, ordered by `PAINTING_STATUS_ORDER` constant" with the original "empty columns hidden" language (from CONTEXT.md). Shipped behavior shows all 11 columns always rendered.  
**Why accepted:** Intentional UX decision by the executor — all columns are always visible as drag-and-drop targets. User explicitly approved during Phase 4 human-verify checkpoint (04-03-SUMMARY.md). `KanbanBoard.test.tsx` was updated to assert 11 columns.  
**Remediation:** Update REQUIREMENTS.md PROJ-02 text to remove "empty columns hidden" language — one line edit.

### Item 2: Fragile DOM Query in PaintingProjectsPage (Phase 4)

**File:** `src/features/painting-projects/PaintingProjectsPage.tsx` lines 30–33  
**Issue:** Empty-state CTA button wired via `document.querySelector<HTMLButtonElement>('button[type="button"][aria-haspopup="dialog"]')?.click()` — a fragile DOM query that may silently fail if aria attributes change.  
**Why accepted:** Not blocking any current feature; works in production. Low priority.  
**Remediation:** Replace with a `ref` or `useState` pattern to directly call `setAddPickerOpen(true)` instead of DOM manipulation.
