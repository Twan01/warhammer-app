## Interaction Contract

### Collapsible Warning Detail

- **Trigger element:** The "Warnings: N" text span — same element that shows the count. No separate chevron button; the entire span is clickable.
- **Trigger styling:** `cursor-pointer` added to span. Same color as current warning count text (`text-destructive` when hard warnings exist, `text-amber-500` otherwise).
- **Chevron indicator:** `ChevronDown` icon (4px gap after text, `h-3 w-3`, rotates 180deg when open via `transition-transform`).
- **Open state:** `useState<boolean>(false)` — local to `GameDayReadinessPanel`, not persisted.
- **Animation:** shadcn/ui `CollapsibleContent` provides built-in height animation.
- **Collapsed default:** Panel starts collapsed. Pre-game users scan the summary; they expand for details only if the warning count warrants it.

Source: CONTEXT.md D-03 (D-03 specifies collapsible for warning detail), RESEARCH.md Pattern 3.

### Points Display — Exceeded State

- Points value changes from `text-foreground` (default) to `text-destructive` when `stats.pointsExceeded === true`.
- No animation, no icon — color change only. Same treatment as `ArmyListSummaryBar`.

Source: CONTEXT.md D-04, ArmyListSummaryBar line 79.

### Role Coverage Pills

- Covered role: `bg-secondary text-secondary-foreground rounded-full px-2 py-1 text-xs`
- Uncovered role: `border border-dashed border-muted-foreground/40 text-muted-foreground rounded-full px-2 py-1 text-xs bg-transparent`
- Pills wrap on narrow containers (`flex flex-wrap gap-2`) — not horizontal scroll.
- Section hidden entirely when `hasAnyRole === false` (no units have a tactical role assigned).

Source: CONTEXT.md D-08, D-09, ArmyListSummaryBar lines 119-137.

### Readiness Gap List

- Filter criteria: `unit.status_painting !== "Completed" || unit.status_assembly === 0`
- Each row: unit name (left) + status display (right)
- Painting status uses `StatusBadge` component (dot + label pattern)
- Assembly status renders inline text "Not assembled" in `text-muted-foreground text-xs` when `status_assembly === 0`, on the same row as the unit name below the `StatusBadge`
- "All clear" state shown only when `notReadyUnits.length === 0` AND `totalWarnings === 0`

Source: CONTEXT.md D-06, D-07.

### Warning Severity Icons

| Severity | Icon | Color | Usage |
|----------|------|-------|-------|
| Hard (critical) | `AlertCircle` (Lucide) | `text-destructive` | Prefix each hard warning in collapsible detail |
| Soft (informational) | `AlertTriangle` (Lucide) | `text-amber-500` | Prefix each soft warning in collapsible detail |

Icon size: `h-3 w-3` inline with warning text. Source: auto-selected — Lucide conventions, codebase icon pattern.

## States to Implement

| State | Trigger | Visual treatment |
|-------|---------|-----------------|
| Loading | `listLoading === true` in GameDayPage | Existing skeleton (lines 33-41) — unchanged |
| List not found | `!list` in GameDayPage | Existing error panel — unchanged |
| Zero units | `units.length === 0` | Readiness panel renders with 0 pts, 0% ready, no role coverage, "All units battle-ready" (trivially true) |
| All clear | `totalWarnings === 0 && notReadyUnits.length === 0` | `battle-gold/10` pill: "All units battle-ready" |
| Points exceeded | `stats.pointsExceeded === true` | Points value turns `text-destructive` |
| Hard warnings present | `stats.hardWarningCount > 0` | Warning count in `text-destructive` |
| Soft warnings only | `stats.hardWarningCount === 0 && stats.softWarningCount > 0` | Warning count in `text-amber-500` |
| Collapsible open | User clicks warning count trigger | Warning detail section revealed with height animation |
| No roles assigned | `hasAnyRole === false` | Role Coverage section hidden entirely |
