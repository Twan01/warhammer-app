## Color

Using established CSS custom property tokens from `globals.css`:

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` / `--forge-black` | Page background |
| Secondary (30%) | `hsl(var(--card))` / `--panel-elevated` | Cards, SyncStatusCard, ArmyListDetailSheet |
| Accent (10%) | `--battle-gold` (`oklch(0.78 0.17 85)`) | Points freshness "fresh" indicator only when points data is present and fresh; also used in army list progress bar (existing) |
| Destructive | `hsl(var(--destructive))` | Not used in this phase — no destructive actions |

Accent reserved for:
- `PointsFreshnessBadge`: the freshness dot when `freshness === "fresh"` and points data is confirmed present (i.e., `syncMeta.points_count > 0`)
- Do NOT use `--battle-gold` for aging/stale/never states — those use the existing `FRESHNESS_DOT_CLASS` colors

Freshness dot colors (from existing `FRESHNESS_DOT_CLASS` — do not invent new colors):

| Tier | CSS class | Condition |
|------|-----------|-----------|
| fresh | `bg-green-500` | Points synced < 7 days ago |
| aging | `bg-amber-500` | Points synced 7–14 days ago |
| stale | `bg-red-500` | Points synced > 14 days ago |
| never / unknown | `bg-muted-foreground` | No sync ever, or list has no synced points |

Delta diff colors (matching existing SyncStatusCard pattern in `RulesHubPage`):

| Delta type | CSS class |
|-----------|-----------|
| Added (new units with points) | `text-green-600` |
| Removed (units no longer in points data) | `text-red-600` |
| Changed (point cost increased or decreased) | `text-amber-600` |

Source: `SyncStatusCard.tsx` lines 127–143 (verified pattern).

## New Components Specified

### 1. `PointsFreshnessBadge` — `src/features/army-lists/PointsFreshnessBadge.tsx`

Used in: `ArmyListCard` (below faction name) and `ArmyListDetailSheet` (next to points total in `ArmyListSummaryBar`).

Visual contract:
- A 8px × 8px (`h-2 w-2`) rounded-full dot using `FRESHNESS_DOT_CLASS[freshness]`
- Followed by a `text-xs text-muted-foreground` age label: e.g. "Synced today", "Synced 3 days ago", "Never synced"
- When `syncMeta` is loading: render a `Skeleton` of `h-2 w-16` in place of the dot+label
- When `freshness === "never"` AND `syncMeta?.points_count` is 0 or null: label reads "No points data" (neutral, not a warning)
- Wrapped in `<Tooltip>` — trigger is the dot, content is the full age label or "No official points data synced"
- Layout: `flex items-center gap-1.5` — same as SyncStatusCard freshness row

Interaction contract:
- Read-only — no click action
- Tooltip opens on hover (radix default delay)

### 2. `PointsDeltaSection` — `src/features/rules-hub/PointsDeltaSection.tsx`

Used in: `RulesHubPage` sync diff display area, rendered after the existing `lastSyncDiff` stat row (line 124 of `SyncStatusCard.tsx`).

Visual contract:
- Collapsible section with trigger: `"Points Changes"` label + `ChevronDown` icon
- Collapsed by default (closed state)
- Trigger uses same class pattern as `SyncStatusCard` error toggle: `flex items-center gap-1.5 text-xs text-muted-foreground hover:text-foreground transition-colors`
- Summary line when collapsed: `+{delta_added} added / -{delta_removed} removed / ~{delta_changed} changed` using the same color pattern as existing diff row (green/red/amber)
- When `delta_added + delta_removed + delta_changed === 0`: show `text-xs text-muted-foreground "No point changes in this sync"` (no collapsible — static text)
- Expanded content: scrollable list of changed units, max-height `max-h-48 overflow-y-auto`
  - Each row: unit name (text-sm) + old points → new points (`text-xs text-muted-foreground`) + direction icon (`TrendingUp` green / `TrendingDown` red, `h-3 w-3`)
  - Army list impact line below the unit list: `text-xs text-muted-foreground "{N} army list(s) affected: List A, List B"` — only shown when `delta_changed > 0`

Interaction contract:
- Collapsible open/close via `useState` (local, not persisted)
- Army list names in impact line are plain text (no navigation — the detail sheet handles navigation)

## States and Interactions

### Freshness Badge States

| State | Trigger | Visual |
|-------|---------|--------|

| Never synced | `syncMeta.last_sync_at === null` | Gray dot + "No points data" |
| No points data | `syncMeta.points_count === 0 or null` | Gray dot + "No points data" |
| Fresh | `freshness === "fresh"` | Green dot + age label |
| Aging | `freshness === "aging"` | Amber dot + age label |
| Stale | `freshness === "stale"` | Red dot + age label |

### Points Delta Section States

| State | Trigger | Visual |
|-------|---------|--------|
| No prior sync | `lastSyncDiff` prop is null | Section not rendered |
| Zero changes | All delta counts are 0 | Static "No point changes in this sync" text |
| Has changes, collapsed | Default after sync | Summary line with color-coded counts |
| Has changes, expanded | User clicks trigger | Unit list + army impact line |

## Accessibility

- Freshness dots carry no meaning without their adjacent text label — do NOT rely on color alone. The `text-xs text-muted-foreground` age label is always rendered beside the dot.
- `Tooltip` on the freshness dot provides redundant text for screen readers via `TooltipContent`.
- `Collapsible` uses Radix's built-in `aria-expanded` and keyboard support (Enter/Space to toggle).
- All `font-semibold tabular-nums` values for point counts ensure numeric alignment (existing pattern from `ArmyListCard`).
- Interactive cards use `role="button"` + `tabIndex={0}` + `onKeyDown` (existing pattern — no change needed for freshness badge which is non-interactive).
