## Spacing Scale

Declared values (multiples of 4, 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline checkbox-to-label spacing |
| sm | 8px | Step row internal padding, badge gaps |
| md | 16px | Dialog body padding, section content padding, tab content padding |
| lg | 24px | Gap between applied recipe cards, dialog section spacing |
| xl | 32px | Dialog max content area padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Accordion trigger minimum height: 48px (`min-h-12`) — touch target for section collapse toggle
- Checkbox touch target: 48px × 48px via padding on the row wrapper (`min-h-12`)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Step description text, dialog body copy, picker list items |
| Label | 12px | 400 (regular) | 1.4 | Field labels, section metadata badges, "X/Y completed" badge text |
| Heading | 16px | 600 (semibold) | 1.3 | Dialog titles, section accordion trigger label, applied recipe card title |

Size difference (12px Label vs 14px Body) provides sufficient visual distinction without a third weight.

## Color

All values reference CSS custom properties from globals.css (dark mode as primary runtime surface).

| Role | CSS Token | Dark Value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--background` | hsl(240 10% 3.9%) ≈ #09090b | Page background, Sheet backdrop |
| Secondary (30%) | `--card` / `--panel-elevated` | hsl(240 10% 5.9%) ≈ #111113 | Dialog surface, Accordion items, applied recipe cards |
| Accent (10%) | `--faction-accent` (runtime) | #71717a default (zinc-500) | Progress bar fill, checked checkbox fill, active step indicator |
| Destructive | `--destructive` | hsl(0 72.2% 50.6%) | Remove assignment button, destructive confirmation |
| Muted | `--muted-foreground` | hsl(240 5% 64.9%) | Dimmed/disabled already-assigned units in picker, empty state body text |
| Battle Gold | `--battle-gold` | oklch(0.78 0.17 85) | Completion milestone indicator (100% complete badge only) |

Accent reserved for: progress bar fill (`<Progress>`), checked `<Checkbox>` state, current-step indicator dot in step checklist. NOT used on general interactive elements.

## Interaction Contracts

### ApplyRecipeDialog (Unit → Recipe)

**Trigger:** "Apply Recipe" Button in UnitDetailSheet (Details tab, below linked recipes section)
**Container:** `<Dialog>` (not a nested Sheet — confirmed by D-04)
**Width:** `sm:max-w-lg`

**States:**

| State | Display |
|-------|---------|
| Loading recipes | Skeleton list (3 rows at 40px height each) |
| Recipe list | Scrollable Command list, max-height 240px, searchable by name |
| Recipe selected | SectionedTimeline preview renders below picker, in a scrollable area max-height 320px |
| No recipe selected | Preview area hidden; Confirm button disabled |
| Submitting | Confirm button shows loading spinner, disabled |
| Success | Dialog closes; toast "Recipe applied to [unit name]" |
| Error | Toast "Failed to apply recipe. Please try again." Dialog stays open. |

**Footer buttons:** "Discard" (outline) left, "Apply Recipe" (default) right — disabled until recipe selected.

### ApplyToUnitsDialog (Recipe → Units, bulk)

**Trigger:** "Apply to Unit(s)" Button in RecipeDetailSheet SheetFooter
**Container:** `<Dialog>`
**Width:** `sm:max-w-lg`

**States:**

| State | Display |
|-------|---------|
| Loading units | Skeleton list (3 rows) |
| Unit list | Command list with faction filter Select above it; multi-select with checkboxes per row |
| Already-assigned unit | Row dimmed (`opacity-50`) and disabled; tooltip "Already applied" |
| Units selected | Footer shows count: "Apply to 3 units" |
| No units selected | Footer shows: "Apply to Unit(s)" — button disabled |
| Submitting | Button loading spinner, all rows disabled |
| Success | Dialog closes; toast "Recipe applied to 3 units" |
| Error | Toast "Failed to apply recipe to some units. Please try again." |

**Footer buttons:** "Keep browsing" (outline) left, "Apply to N units" (default) right — disabled until at least 1 unit selected.

### AssignmentChecklist

**Sectioned recipe (has sections):**
- Overall `<Progress>` bar at top, with "X% complete" label right-aligned in 12px/regular
- `<Accordion type="multiple">` — all sections collapsed by default, user expands individually
- Each `<AccordionTrigger>`: section name (16px/semibold) + `<Badge variant="outline">` showing "X/Y" at right edge; minimum height 48px (`min-h-12`)
- Each `<AccordionContent>`: list of `<Checkbox>` rows, one per step
  - Checkbox row wrapper: `min-h-12` (48px) touch target
  - Checkbox row: `[checkbox] [step order_index]. [step description]` — 14px/regular at 1.5 line-height
  - Checked state: text gets `line-through text-muted-foreground`
  - Toggle calls `useToggleStepProgress(assignmentId, step.order_index)`
  - Optimistic update: checkbox checked immediately; revert on error with toast

**Flat recipe (no sections, D-09):**
- Overall `<Progress>` bar
- Simple `<ul>` of `<Checkbox>` rows — no Accordion wrapper

### AppliedRecipesTab

**Focal point:** The progress bar on each applied recipe card is the primary visual anchor — it communicates completion state at a glance before the user expands the checklist.

**Empty state (D-14):**
- Centered layout, `ClipboardList` icon (Lucide, 32px, `text-muted-foreground`)
- Heading: "No recipes applied yet" (16px/semibold)
- Body: "Apply a recipe to track your painting progress step by step." (14px/regular, muted)
- CTA Button (default variant): "Apply Recipe" — opens `ApplyRecipeDialog`

**With assignments:**
- List of applied recipe cards (Card component, `bg-panel-elevated`)
- Each card header: recipe name (16px/semibold) + trash icon button (destructive, size="icon", `aria-label="Remove [recipe name] assignment"`)
- Each card body: `<Progress>` bar + "X% complete" label + "X/Y steps" badge
- Click anywhere on card body (not the trash icon) expands inline `<AssignmentChecklist>`
- Expanded state: card grows in place, checklist rendered below progress bar

### Remove Assignment

**Trigger:** Trash icon button on applied recipe card
**Pattern:** `<AlertDialog>` confirmation (reuses existing `alert-dialog` component)
**Title:** "Remove recipe assignment?"
**Body:** "This will delete all step progress for [Recipe Name] on this unit. This cannot be undone."
**Actions:** "Keep Assignment" (outline) + "Remove Assignment" (destructive)
**On confirm:** calls `useDeleteAssignment`, invalidates cache, toast "Recipe assignment removed"
