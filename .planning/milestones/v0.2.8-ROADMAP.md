# Milestone v0.2.8: Rules Data Hub UI / Army Lists 2.0 / Game Day

**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 52–56
**Total Plans:** 12

## Overview

Expose synced Wahapedia rules data in a standalone browser, connect detachment selection to army lists, and deliver a focused Game Day mode for in-game reference — making rules data visible, searchable, and useful for real game preparation.

## Phases

**Depends on**: Phase 51 (v0.2.7 shipped)
**Requirements**: ARMY-06
**Plans**: 3 plans

Plans:
- [x] 52-01: Migration 019 + TypeScript types + army list query extensions
- [x] 52-02: Points import design document (ARMY-06)
- [x] 52-03: Query modules + hooks + sync invalidation

**Details:**

## Milestone Summary

**Key Decisions:**
- User data (favorites, notes, detachment) in hobbyforge.db, never rules.db — survives re-sync
- ATTACH DATABASE not supported — dual-query merge pattern continues
- staleTime: Infinity + sync invalidation for all rules.db hooks
- useWahapediaFactionId required for rules-facing queries (integer returns empty)
- Game Day state in Zustand persist (localStorage) — SQLite deferred
- detachment_name denormalized for rules.db wipe survival
- Page-level Map lookup pattern prevents N+1 hooks on annotations
- Sub-component pattern for hooks-in-loop compliance

**Issues Resolved:**
- Rules data now visible and searchable from dedicated hub
- Army lists connected to detachment rules context
- Game Day mode provides focused in-game reference

**Issues Deferred:**
- Points import implementation (design doc only in v0.2.8)
- Persistent game state in SQLite (localStorage for now)
- Multi-session game resumption

**Technical Debt Incurred:**
- GameDayStratagemCard has button nested inside CollapsibleTrigger — HTML validation warning
- useDetachmentById hook exported but unused (proactive creation)
- getRulesFavoritesByType query exported but unused (consumers prefer full-list + client filter)
- getRulesNoteByKey query exported but unused (consumers prefer full-list + Map lookup)
