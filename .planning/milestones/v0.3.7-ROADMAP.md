**Status:** SHIPPED 2026-05-28

## Overview

Reduce manual data entry and repetitive actions by auto-deriving statuses from recipe completion, managing active projects automatically, pre-filling context intelligently, and surfacing battle-readiness in the army list picker.

## Milestone Summary

**Key Decisions:**
- Override guard uses null params + CASE WHEN pattern — null bypasses write, preserving current DB value
- Assembly/basing/varnish kept as separate query pairs (clarity over consolidation)
- is_active_project auto-SET only in createAssignment/bulkCreateAssignments; auto-CLEAR only at pct=100
- computeUnitReadiness() ships as pure function before any UI — prevents definition divergence
- RecipeFormSheet reads from useActiveFaction() for automatic faction pre-fill (no explicit prop passing needed)
- ApplyRecipeDialog groups recipes into Suggested/Other via CommandGroup when factionId provided

**Issues Resolved:**
- SCP-01 dead code gap: RecipeFormSheet defaultFactionId prop had no callers — fixed by reading from ActiveFactionContext
- SAD-04 override reset: UnitSheet edit path was resetting override flags to 0 — fixed by excluding from edit payload

**Technical Debt Accepted:**
- Phases 100, 101 missing formal VERIFICATION.md (code verified by integration checker, VALIDATION.md exists)
- No requirements-completed frontmatter in SUMMARY files
- status_basing_override and status_varnished_override have no UI setter (guard logic is inert)
