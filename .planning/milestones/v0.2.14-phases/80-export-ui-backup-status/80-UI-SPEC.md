## Spacing Scale

Declared values (multiples of 4 — standard 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline dot spacing |
| sm | 8px | Compact element spacing (`gap-2`) |
| md | 16px | Default element spacing (`gap-4`), card horizontal padding unit |
| lg | 24px | Card content padding (`p-6` = 24px) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Status dot size: `h-2 w-2` (8px) — matches existing sync freshness dot in DataHealthSummaryCard
- Dot-to-label inline gap: 6px (`gap-1.5`) — matches existing DataHealthSummaryCard sync row pattern (line 32: `flex items-center gap-1.5`). Not a multiple of 4 but locked by existing codebase convention for tight inline dot + text rows.
- Dashboard summary row text: `text-xs`, icon size `12px` — matches existing DataHealthSummaryCard pattern (line 37, 46)

## Color

| Role | CSS Token | Tailwind Class | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) | `hsl(var(--background))` | `bg-background` | Page background |
| Secondary (30%) | `hsl(var(--card))` | `bg-card` | Card surfaces, DataHealthSummaryCard container |
| Accent — Healthy | Tailwind utility | `bg-green-500` | Backup freshness dot: ≤7 days |
| Accent — Recommended | Tailwind utility | `bg-amber-500` | Backup freshness dot: 8–30 days |
| Accent — Overdue | Tailwind utility | `bg-orange-500` | Backup freshness dot: >30 days |
| Accent — Never | `hsl(var(--muted-foreground))` | `bg-muted-foreground` | Backup freshness dot: no backup recorded |
| Destructive | `hsl(var(--destructive))` | `text-destructive` | Not used in this phase |
| Muted text | `hsl(var(--muted-foreground))` | `text-muted-foreground` | Status subtitles, age labels |

Accent reserved for:
- The 8px status dot inline with backup age text in BackupCard
- The 8px status dot inline with backup age label in DataHealthSummaryCard
- These are the ONLY elements using color semantics in this phase — no CTA coloring, no border coloring

Color mapping mirrors `FRESHNESS_DOT_CLASS` from `src/lib/syncFreshness.ts`. The new `BACKUP_FRESHNESS_DOT_CLASS` in `src/lib/backupFreshness.ts` must use:
```ts
export const BACKUP_FRESHNESS_DOT_CLASS: Record<BackupFreshness, string> = {
  healthy: "bg-green-500",
  recommended: "bg-amber-500",
  overdue: "bg-orange-500",
  never: "bg-muted-foreground",
};
```

Rationale for `bg-orange-500` (not `bg-red-500`) for Overdue: orange communicates urgency without alarm. Rationale for `bg-muted-foreground` (not red) for Never: grey signals "not yet started" rather than alarming a first-time user — consistent with sync freshness "never synced" using a neutral dot.

## Interaction Contract

### BackupCard — Updated States

**State 1 — Never backed up (no localStorage entry)**
- Dot: `bg-muted-foreground` (neutral grey)
- Label: "Never — No backup yet"
- Button: "Create Backup" (enabled)

**State 2 — Healthy (backed up ≤7 days ago)**
- Dot: `bg-green-500`
- Label: "Healthy — Backed up [today / yesterday / N days ago]"
- Button: "Create Backup" (enabled)

**State 3 — Recommended (8–30 days ago)**
- Dot: `bg-amber-500`
- Label: "Recommended — Backed up N days ago"
- Button: "Create Backup" (enabled)

**State 4 — Overdue (>30 days ago)**
- Dot: `bg-orange-500`
- Label: "Overdue — Backed up N days ago"
- Button: "Create Backup" (enabled)

**State 5 — In progress**
- Dot: unchanged (retains last tier color)
- Button: disabled, shows `<Loader2 className="animate-spin" />` + "Creating backup..."

**State 6 — Success (just completed)**
- Dot updates to `bg-green-500`
- Label updates to "Healthy — Backed up today"
- Toast: `toast.success("Backup created successfully")`

**State 7 — Error**
- Dot: unchanged
- Toast: `toast.error("Backup failed: [error message]")`

### Dot + Label Placement in BackupCard

```
[dot] [tier label] — [age label]
```

Example renders:
- `● Healthy — Backed up today`
- `● Recommended — Backed up 12 days ago`
- `● Overdue — Backed up 45 days ago`
- `● Never — No backup yet`

The dot is `<span className={`inline-block h-2 w-2 rounded-full mr-1.5 ${BACKUP_FRESHNESS_DOT_CLASS[tier]}`} />` placed before the combined label string, replacing the previous plain subtitle text.

### DataHealthSummaryCard — Updated Backup Row

Replaces `<HardDrive size={12} />` + plain text with dot + age text, matching the sync freshness row above it:

```
[dot] [age label]
```

The `HardDrive` icon is removed from the backup row. The dot carries the semantic color. The label is the same human-readable age string as before ("Backed up today", "Backed up N days ago", "No backup").
