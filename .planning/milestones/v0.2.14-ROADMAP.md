# Milestone v0.2.14: Backup 2.0 — Structured Export, Restore & Safety Backups

**Status:** SHIPPED 2026-05-19
**Phases:** 79-83
**Total Plans:** 11

## Overview

Make HobbyForge safe to use long-term by giving the user a reliable way to export, restore, and protect their local data. Structured backup export (.zip with hobbyforge.db via VACUUM INTO + metadata.json), full restore with preview/validation/atomic swap/restart, automatic safety backups before risky operations (restore, rules sync), and progressive backup diagnostics.

## Milestone Summary

**Key Decisions:**

- VACUUM INTO via Rust command (not file copy) for safe, consistent database backups
- Restore requires app restart via relaunch() — tauri-plugin-sql has no reconnect API
- WAL/SHM/journal sidecar files explicitly deleted before file swap in restore
- rules.db excluded from backups — fully regenerable via Wahapedia sync
- Schema version = migration count (integer), not semver
- Two-step restore: validate+preview (non-destructive) then commit (destructive)
- localStorage pattern for backup status metadata (consistent with v0.2.13)
- JS owns restart via relaunch() from @tauri-apps/plugin-process
- Pre-sync safety backup aborts sync if backup fails

**Issues Resolved:**

- Cache invalidation bug: useRulesSync now invalidates SAFETY_BACKUPS_KEY after pre-sync backup
- backupFreshness guard against invalid dates and future timestamps

**Technical Debt Incurred:**

- Nyquist validation partial for Phase 79, missing for Phase 83
