## Spacing Scale

Project-standard 8-point scale. All values confirmed in existing card and layout patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge spacing |
| sm | 8px | Compact element spacing, card interior row gaps (`gap-2`) |
| md | 16px | Default element spacing, card horizontal padding |
| lg | 24px | Section padding, section group gaps (`gap-6` in SectionedTimeline) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Workflow collapsible trigger height: 28px (`h-7`) — matches existing RecipeSectionCard control heights (name Input and Surface Select are also `h-7`)
- Workflow 2×2 grid gap: 8px (`gap-2`) — compact because controls are labeled by their placeholder text, no separate label row needed
- Workflow collapsible content padding: `px-3 pb-3 pt-2` — matches existing CollapsibleContent padding pattern in RecipeSectionCard

Source: `RecipeSectionCard.tsx` existing patterns, `SectionedTimeline.tsx`.

| execution_mode none option | `"— mode —"` |

## Component Specs

### 1. RecipeSectionCard — Workflow Collapsible (RUI-01, RUI-02)

**File:** `src/features/recipes/RecipeSectionCard.tsx`

**Change:** Add a second nested `Collapsible` for workflow metadata below the main card header row, inside the outer `CollapsibleContent` (i.e. it appears when the section is expanded), above `RecipeStepList`.

**Visibility gate (D-04, D-05):**
```tsx
const hasWorkflowMetadata =
  Boolean(section.section_type) ||
  Boolean(section.technique) ||
  Boolean(section.execution_mode) ||
  Boolean(section.applies_to);

const showWorkflowCollapsible =
  sections.length > 1 || hasWorkflowMetadata;
```

Note: `RecipeSectionCard` does not receive `sections` as a prop — the parent `RecipeFormSheet` controls the section list. The visibility condition must be passed in as a prop or derived from a `sectionsCount` prop.

**Prop addition:**
```tsx
interface RecipeSectionCardProps {
  section: DraftSection;
  onChange: (updated: DraftSection) => void;
  onRemove: () => void;
  onCreateNewPaint: (stepLocalId: string) => void;
  sectionsCount: number;  // NEW — number of sections in the form
}
```

The parent `RecipeFormSheet` passes `sectionsCount={sections.length}`.

### 2. SectionedTimeline — Workflow Metadata Display (RUI-03, RUI-04)

**File:** `src/features/recipes/SectionedTimeline.tsx`

**Change:** Extend the section header row to show section_type Badge and dot-separated technique/execution_mode/applies_to inline text.

**Section header — current structure:**
```
[Section Name]  [surface badge]  [Optional badge]  ...right-side metadata...
```

**Section header — after Phase 58:**
```
[section_type badge]  [Section Name]  [surface badge]  [Optional badge]  [dot-separated]  ...right-side metadata...
```

Wait — per D-08, section_type Badge comes before the section name (it's the categorical label). The dot-separated string comes after the section name, following the existing surface/optional badges, before the right-side `ml-auto` span.

**Corrected header order (left to right):**
```
[section_type Badge]  [Section Name text]  [surface Badge]  [Optional Badge]  [· technique · mode · applies_to]  [ml-auto right metadata]
```

**section_type Badge:**
```tsx
{section.section_type && (
  <Badge variant="outline" className="text-xs capitalize">
    {section.section_type}
  </Badge>
)}
```
Position: before `<span className="text-sm font-semibold">{section.name}</span>`.

**Dot-separated metadata string (D-07, D-09, D-10):**

Construct the parts array from non-null fields only:
```tsx
const workflowParts = [
  section.technique,
  section.execution_mode,
  section.applies_to,
].filter(Boolean) as string[];
```

Render inline after existing optional Badge:
```tsx
{workflowParts.length > 0 && (
  <span className="text-xs text-muted-foreground capitalize">
    {workflowParts.join(" · ")}
  </span>
)}
```

Capitalize approach: use `capitalize` Tailwind class — CSS `text-transform: capitalize` capitalizes the first letter of each word. Values like `drybrush` render as `Drybrush`; `oil-enamel` renders as `Oil-enamel`. This is acceptable for the hobby context (per Claude's Discretion — capitalize as-is).

**ASCII mockup — section header, all metadata present:**

```
[basecoat]  Armor Blue  [Armor]  · drybrush · sequential · shoulder pad
```

**ASCII mockup — section header, only section_type set:**

```
[layer]  Cloth
```

**ASCII mockup — section header, no workflow metadata (simple recipe):**

```
Armor Blue  [Skin]  [Optional]
```

**Full section header JSX structure:**
```tsx
<div className="flex items-center gap-2 flex-wrap" data-testid="section-header">
  {section.section_type && (
    <Badge variant="outline" className="text-xs capitalize">
      {section.section_type}
    </Badge>
  )}
  <span className="text-sm font-semibold">{section.name}</span>
  {section.surface && (
    <Badge variant="outline" className="text-xs">
      {section.surface}
    </Badge>
  )}
  {section.optional === 1 && (
    <Badge variant="outline" className="text-xs">
      Optional
    </Badge>
  )}
  {workflowParts.length > 0 && (
    <span className="text-xs text-muted-foreground capitalize">
      {workflowParts.join(" · ")}
    </span>
  )}

  {/* Right-side metadata (unchanged) */}
  <span className="text-xs text-muted-foreground ml-auto flex items-center gap-3">
    ...
  </span>
</div>
```

Note: Add `flex-wrap` to the header div to handle long metadata strings gracefully at narrower widths.

## State Variations

### RecipeSectionCard Workflow Collapsible

| State | Rendering |
|-------|-----------|
| Single section, no metadata | Workflow collapsible not rendered |
| Single section, any metadata field set | Workflow collapsible rendered, starts open |
| Multiple sections (any count) | Workflow collapsible rendered, starts closed |
| Workflow collapsible closed | ChevronRight icon (not rotated), step list visible |
| Workflow collapsible open | ChevronRight icon rotated 90°, 2×2 grid visible above steps |
| section_type = null | Select shows placeholder "Type" |
| technique = null | Select shows placeholder "Technique" |
| execution_mode = null | Select shows placeholder "Mode" |
| applies_to = null or "" | Input shows placeholder "Applies to…" |
| Section being dragged | Existing `opacity: isDragging ? 0.5 : 1` — no change |

### SectionedTimeline Header

| State | Rendering |
|-------|-----------|
| No workflow metadata on section | Header unchanged from current (name + surface + optional badges) |
| section_type set, others null | section_type Badge before name; no dot-separated text |
| technique or execution_mode set | Dot-separated text after optional badge |
| All four fields set | section_type Badge + section name + surface badge + optional badge + dot-separated 3-part string |
| applies_to is long string | `flex-wrap` on header div allows wrap to second line at narrow widths |

## Interaction Contracts

### Workflow Collapsible Toggle

| Target | Interaction | Handler |
|--------|-------------|---------|
| Workflow collapsible trigger row | Toggle workflow fields open/closed | `CollapsibleTrigger` (Radix managed state) |
| Outer section collapsible trigger | Toggle entire section open/closed | Existing `CollapsibleTrigger` on `open` / `setOpen` |

The two collapsibles are independent. Collapsing the outer section hides both the workflow grid and the step list. The inner workflow collapsible state is not reset when the outer section collapses.

### Workflow Field Changes

All four workflow fields follow the `onChange({ ...section, field: value })` callback pattern matching existing RecipeSectionCard conventions. Changes are held in local DraftSection state — no individual field saves. Changes persist when the recipe form is submitted via the existing `RecipeFormSheet` save path.

| Field | Event | Handler |
|-------|-------|---------|
| section_type Select | `onValueChange` | `onChange({ ...section, section_type: v === "__none__" ? null : v })` |
| technique Select | `onValueChange` | `onChange({ ...section, technique: v === "__none__" ? null : v })` |
| execution_mode Select | `onValueChange` | `onChange({ ...section, execution_mode: v === "__none__" ? null : v })` |
| applies_to Input | `onChange` (e) | `onChange({ ...section, applies_to: e.target.value || null })` |

### SectionedTimeline

Read-only display — no new interactions. The section header row is not clickable. Timeline interactions (step expansion, photo display) are unchanged.

## Responsive Behavior

RecipeSectionCard is displayed inside `RecipeDetailSheet` (a Sheet component). Sheet width is fixed by the existing sheet size prop — no breakpoint changes in Phase 58.

The 2×2 workflow grid uses `grid-cols-2 gap-2`. At the sheet's effective content width (~500px), two columns of selects render at ~230px each — sufficient for all option labels.

SectionedTimeline section header uses `flex-wrap`. Long `applies_to` strings (free-text) may cause wrap to a second line — this is acceptable and visually superior to truncation for a recipe detail view.

## Accessibility Notes

### Workflow Collapsible

- `CollapsibleTrigger` uses Radix's built-in `aria-expanded` toggle — handled automatically
- The trigger row must have accessible text: the `"Workflow"` label text inside the trigger is sufficient
- ChevronRight icon: `aria-hidden="true"` (decorative — direction conveyed by `aria-expanded` on the trigger)

### Workflow Select Controls

- Each Select uses shadcn `<SelectTrigger>` which renders a `<button>` with `role="combobox"` — Radix managed
- Placeholder text (`"Type"`, `"Technique"`, `"Mode"`) provides sufficient context in a compact 2×2 grid
- No separate `<label>` elements — placeholder-as-label is acceptable for a compact form where fieldset position communicates meaning

### applies_to Input

- No explicit `<label>` — `placeholder="Applies to…"` is visible until text is entered
- Add `aria-label="Applies to (model area)"` on the Input element to support screen readers after the placeholder disappears

### SectionedTimeline

- section_type Badge is presentational — `aria-hidden` not needed (it carries semantic meaning)
- Dot-separated span: `aria-label` not needed — screen readers read the text content naturally with the dots as punctuation
