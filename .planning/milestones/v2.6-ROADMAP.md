# v2.6 — Rules Sync 2.0 / Rules Data Hub

**Phases:** 42–46
**Status:** Planned
**Pre-requisite:** v2.5 complete

## Goal

Stabilize and extend the existing local rules import architecture so HobbyForge becomes a reliable personal rules and points reference.

## Phases

- [ ] Phase 42: Architecture Audit & Pre-flight — Audit current sync pipeline (TypeScript useRulesSync vs Rust bulk_sync_rules), confirm rules.db schema state, identify gaps, produce architecture note — no code changes
- [ ] Phase 43: Extended Rules Schema — Add or complete tables for datasheet wargear, shared abilities, stratagems, detachments, and detachment abilities via additive migrations to rules.db
- [ ] Phase 44: Sync Pipeline Extension — Extend useRulesSync and/or bulk_sync_rules to fetch and insert extended CSV data (wargear, stratagems, detachments); ensure existing base sync remains stable
- [ ] Phase 45: Sync Metadata & Import Tracking — Source registry (name, URL, version, edition), sync date/row counts/error logs, freshness indicators in UI, last-successful-sync display
- [ ] Phase 46: Manual Overrides & Version Comparison — Visible manual override system for points/stats/keywords/abilities (override persists across re-syncs), data change detection showing what changed after a re-sync

## Requirements

### Audit first

- Confirm current sync path
- Confirm existing rules.db schema
- Confirm whether wargear tables exist and are populated
- Confirm whether stratagems, detachments and detachment abilities are migrated and synced
- Confirm whether getFullDatasheet() can reliably fetch full rules-related data

### Extend schema if required

- Datasheet wargear
- Shared abilities
- Stratagems
- Detachments
- Detachment abilities
- Points data if a reliable source/format is available
- Import logs
- Import source registry

### Sync metadata

- Source name and URL
- Sync date and source version
- Row counts and last successful sync
- Error/failure logs
- Rules edition and warnings

### Manual overrides

Visible manual overrides for: points, unit stats, keywords, ability reminders, loadout points, list-specific points.

### Versioning and comparison

Change detection for: points, unit stats, abilities, keywords, datasheet removals/renames.

## Key risks

- Wahapedia CSV format may change or become unavailable — sync must be failure-safe
- Extended rules tables (wargear, stratagems) may already partially exist in Rust code but not be wired to UI
- Manual overrides must never be silently overwritten by re-sync

## Acceptance criteria

- Existing Playbook import still works
- Rules sync is traceable and failure-safe
- Extended data is available to Playbook, Army Lists and Game Day Mode
- App remains usable offline
- Import failure does not corrupt existing rules data
- User can see last sync date/version and data freshness
