# Requirements Archive: HobbyForge v0.3.0

**Archived:** 2026-05-22
**Status:** All 17 requirements satisfied

## v0.3.0 Requirements (All Complete)

### Error Handling & Resilience

- [x] **ERR-01**: App renders a fallback UI (not blank screen) when any component throws during render
- [x] **ERR-02**: Each route has its own error boundary -- a crash on one page doesn't break other pages
- [x] **ERR-03**: App verifies DB connection and schema integrity before rendering the main layout
- [x] **ERR-04**: Unhandled promise rejections and uncaught errors are captured and logged (not silently swallowed)
- [x] **ERR-05**: Main database uses WAL journal mode and busy_timeout (matching rules-client.ts)

### Database Hardening

- [x] **DBH-01**: All foreign key columns have database indexes
- [x] **DBH-02**: Temporal query columns have indexes (session_date DESC, battle_date DESC)
- [x] **DBH-03**: Data integrity CHECK constraints prevent invalid values
- [x] **DBH-04**: Sync/import operations use batched INSERT statements

### Performance

- [x] **PERF-01**: Route pages are lazy-loaded via React.lazy()
- [x] **PERF-02**: Mutation invalidation chains are precise
- [x] **PERF-03**: Kanban enrichment fetches in batched queries instead of N sequential per-unit calls
- [x] **PERF-04**: High-frequency render components wrapped with React.memo

### Architecture Cleanup

- [x] **ARCH-01**: DB query layer has zero imports from src/features/
- [x] **ARCH-02**: PlaybookTab.tsx decomposed into sub-tab components (each under 300 lines)
- [x] **ARCH-03**: UnitSheet.tsx decomposed into form section components (each under 200 lines)
- [x] **ARCH-04**: ArmyListsPage modal state uses a reducer instead of 14+ individual useState calls
