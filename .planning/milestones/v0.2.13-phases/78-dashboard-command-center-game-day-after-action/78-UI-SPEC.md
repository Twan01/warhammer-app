## Spacing Scale

Declared values (multiples of 4 — matches existing dashboard `gap-6` / `p-6` patterns):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline metadata separators |
| sm | 8px | Compact element spacing (`gap-2`), card internal rows |
| md | 16px | Default form field spacing (`gap-4`), card padding (`p-4`) |
| lg | 24px | Section padding, dashboard grid gap (`gap-6 p-6`) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks (not used in this phase) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Collapsible after-action section header: `px-4 py-2` (8px vertical) — compact trigger, consistent with standard set
- CP tracker pill: `px-4 py-2` (8px vertical) — inherits from GameDayHeader existing pattern
- Section label (uppercase tracking): `text-sm font-semibold uppercase tracking-widest` — matches all existing dashboard section labels

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.4 |
| Heading | 18px (`text-lg`) | 600 (semibold) | 1.25 |

Notes:
- Section labels (e.g., "Next Painting Action", "Ready to Play"): 14px / semibold / uppercase / `tracking-widest` — matches `ArmyReadinessCard`, `DashboardPage` section pattern
- Card metric values (points, counts): `tabular-nums` applied for numeric content
- Paint step description in NextPaintingActionCard: 14px / regular — truncated to 2 lines (`line-clamp-2`)
- Forgotten rules reminder items in ChecklistTab: 14px / regular — matches existing checklist item style

Source: inferred from `ArmyReadinessCard.tsx`, `CurrentFocusCard.tsx`, `DashboardPage.tsx`, `ChecklistTab.tsx`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` → dark: `240 10% 3.9%` | Page background, sheet background |
| Secondary (30%) | `hsl(var(--card))` → dark: `240 10% 5.9%` | All three new dashboard cards, collapsible section bg |
| Accent (10%) | `oklch(0.78 0.17 85)` (`--battle-gold`) | See reserved list below |
| Destructive | `hsl(var(--destructive))` → dark: `0 72.2% 50.6%` | No destructive actions in this phase |

Accent (`--battle-gold` / `text-battle-gold`) reserved for:
- NextPaintingActionCard: step time estimate when paint is fully available (positive signal)
- ReadyToPlayCard: list name when all units are painted and points are valid
- DataHealthSummaryCard: sync dot and warning count badge only when health is clean (zero warnings, backup current)

Semantic colors for status indicators in DataHealthSummaryCard:
- Sync dot green: `text-emerald-500` — sync age < 7 days
- Sync dot amber: `text-amber-500` — sync age 7–30 days
- Sync dot red: `text-destructive` — sync age > 30 days or sync error present
- Warning count badge amber: `bg-amber-500/15 text-amber-600 dark:text-amber-400` — when warning count > 0

Forgotten rules reminder items in ChecklistTab:
- Background: `bg-amber-500/10` — visually distinguished from regular checklist items
- Label tag: `text-amber-600 dark:text-amber-400 text-xs font-semibold` — "Reminder" tag prefix

End Game button in GameDayHeader:
- Variant: `outline` with explicit `border-battle-gold text-battle-gold hover:bg-battle-gold/10` — visually distinct from nav controls

Source: `globals.css` tokens, `ArmyReadinessCard.tsx` `text-battle-gold` usage, CONTEXT.md Specific Ideas

## Card Visual Contracts

### NextPaintingActionCard

```
┌─────────────────────────────────────────────────────────┐
│  NEXT PAINTING ACTION               [section label]     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  [Palette icon 14px]  Section name · Step desc  │   │
│  │  [paint swatch 10px]  Paint name  owned/missing │   │
│  │  [Clock icon 12px]    Est. 15 min                │   │
│  │  [→ link text-xs]     Go to recipe               │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

- Card class: `rounded-lg border border-border/60 bg-card p-4 shadow-sm transition-shadow duration-150 hover:shadow-md` — matches `ArmyReadinessCard` exactly
- Step description: `text-sm` / `line-clamp-2` / regular weight
- Section name: `text-xs text-muted-foreground` prefix before step desc
- Paint indicator: colored dot (12×12px inline `rounded-full`) colored by ownership status: emerald = owned, amber = low, zinc-400 = missing
- Time estimate: `text-xs text-muted-foreground tabular-nums` with Clock icon (12px)
- "Go to recipe" link: `text-xs text-muted-foreground underline-offset-2 hover:underline` — navigates to painting projects page, not a Sheet open

### ReadyToPlayCard

```
┌─────────────────────────────────────────────────────────┐
│  READY TO PLAY                      [section label]     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Black Legion · 1,500 pts                       │   │
│  │  [Shield 12px] 3 unpainted  [Clock 12px] 2d ago │   │
│  │  [warning badge if applicable]                   │   │
│  └─────────────────────────────────────────────────┘   │
```

- List name + points: `text-sm font-semibold` — `text-battle-gold` when fully painted and valid
- Metadata row: `text-xs text-muted-foreground` with Lucide icons (12px each)
- Warning badge: `bg-amber-500/15 text-amber-600 text-xs px-1.5 py-0.5 rounded` — only shown when unpainted count > 0 or sync stale

### DataHealthSummaryCard

```
┌─────────────────────────────────────────────────────────┐
│  DATA HEALTH                        [section label]     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  ● Synced 2d ago    ⚠ 3 warnings    ↑ 5d ago   │   │
│  │  [link: View full report →]                      │   │
│  └─────────────────────────────────────────────────┘   │
```

- Three metrics in a single horizontal row: `flex items-center gap-4`
- Each metric: icon (12px) + `text-xs tabular-nums` value
- Sync dot: filled `rounded-full w-2 h-2` colored per age (emerald/amber/destructive)
- Warning count: Database icon (12px) + count
- Backup age: HardDrive icon (12px) + "X days ago" or "Never"
- "View full report" link: `text-xs text-muted-foreground underline-offset-2 hover:underline` — navigates to `/data-health`

### Dashboard Grid Placement (three new cards)

The three new cards occupy the left column below HobbyPipeline, grouped under a single "Command Center" section label, in order:
1. NextPaintingActionCard (full column width)
2. ReadyToPlayCard (full column width)
3. DataHealthSummaryCard (full column width)

Alternative if left column becomes too tall: move DataHealthSummaryCard to right column above ArmyReadinessCard. Executor discretion.

## Interaction Contracts

### BattleLogSheet — After-Action Section

- Collapsed by default; trigger: `<CollapsibleTrigger>` with ChevronDown icon (rotates 180° when open)
- Trigger label: `text-xs font-semibold text-muted-foreground uppercase tracking-wide` — "After-Action" with icon
- `<Separator />` above the collapsible trigger
- When open, renders three fields below each other with `gap-4`:
  1. Forgotten Rules textarea — `rows={4}`, placeholder: "Rules you forgot this game — one per line"
  2. (MVP and Underperformed selects already exist in the form — move them INTO the collapsible)
  3. Lessons Learned textarea — already exists, move into collapsible
  4. Changes Next Time textarea — already exists, move into collapsible
- Fields inside collapsible use same `TEXTAREA_CLASS` + `FormField` pattern as existing fields

### BattleLogSheet — End Game pre-fill mode

- Accept new optional prop: `prefill?: Partial<BattleLogFormValues>`
- When `prefill` provided, merge into `DEFAULT_VALUES` in `buildDefaultValues`
- Sheet title when pre-filled: "End Game" (not "Log Game")
- Sheet description when pre-filled: "Record your game result and after-action notes."

### GameDayHeader — End Game Button

- Position: rightmost in the top row, after list title area
- Element: `<Button variant="outline" size="sm" className="border-battle-gold text-battle-gold hover:bg-battle-gold/10 shrink-0">` with Flag icon (14px) + "End Game" label
- On click: opens BattleLogSheet with `prefill` containing `army_list_id`, `battle_date: todayISO()`, and `opponent_faction` if set in game day state

### ChecklistTab — Forgotten Rules Reminders

- Reminder items rendered as a non-interactive, non-checkable group above user checklist items
- Group label: `text-xs font-semibold text-muted-foreground uppercase tracking-wide` — "Reminders from last games"
- Each reminder item: `flex items-center gap-3 rounded-md border px-3 py-2 bg-amber-500/10`
  - Left: `<BookOpen size={14} className="text-amber-500 shrink-0" />`
  - Text: `text-sm` rule text
  - Right: `<span className="text-xs text-amber-600 font-semibold ml-auto">Reminder</span>`
- If no forgotten rules in recent logs: reminder group is not rendered (not an empty state)

### Loading States

All three new dashboard cards follow the existing Skeleton pattern:
- Loading: `<Skeleton className="h-24 w-full" />` replacing the card
- Error: card renders with muted error text, no crash
