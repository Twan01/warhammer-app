**File 1 — Create `src/db/queries/recipeSections.ts` with 6 exported functions:**


**Function 1 — getRecipeSections(recipeId: number): Promise&lt;RecipeSection[]&gt;**
```sql
SELECT * FROM recipe_sections WHERE recipe_id = $1 ORDER BY order_index ASC, id ASC
```
Params: `[recipeId]`. ORDER BY uses order_index ASC as primary sort, id ASC as tiebreaker for sections with the same order_index.

**Function 2 — createRecipeSection(input: CreateRecipeSectionInput): Promise&lt;number&gt;**
```sql
INSERT INTO recipe_sections (recipe_id, name, surface, optional, order_index, notes)
VALUES ($1, $2, $3, $4, $5, $6)
```
Params: `[input.recipe_id, input.name, input.surface ?? null, input.optional, input.order_index, input.notes ?? null]`
Returns `result.lastInsertId ?? 0`.

**Function 3 — updateRecipeSection(input: UpdateRecipeSectionInput): Promise&lt;void&gt;**
```sql
UPDATE recipe_sections
SET name = COALESCE($2, name),
    surface = $3,
    optional = COALESCE($4, optional),
    order_index = COALESCE($5, order_index),
    notes = $6,
    updated_at = datetime('now')
WHERE id = $1
```
Params: `[input.id, input.name ?? null, input.surface ?? null, input.optional ?? null, input.order_index ?? null, input.notes ?? null]`

Note: `surface` and `notes` use direct assignment (not COALESCE) because null is a valid value for these fields (user may want to clear them). `name`, `optional`, `order_index` use COALESCE because null means "don't change".

**Function 4 — deleteRecipeSection(id: number): Promise&lt;void&gt;**
```sql
DELETE FROM recipe_sections WHERE id = $1
```
Params: `[id]`. Add comment: `// ON DELETE CASCADE handles recipe_steps cleanup; session links are SET NULL (migration 014)`.

**Function 5 — reorderRecipeSections(sections: { id: number; order_index: number }[]): Promise&lt;void&gt;**
Sequential UPDATE loop:
```typescript
const db = await getDb();
for (const { id, order_index } of sections) {
  await db.execute(
    "UPDATE recipe_sections SET order_index = $1, updated_at = datetime('now') WHERE id = $2",
    [order_index, id]
  );
}
```
Each section gets its new order_index from the array position. No UNIQUE constraint on order_index, so sequential updates are safe.

**Function 6 — getStepCountsBySection(): Promise&lt;SectionStepCount[]&gt;**
Export a new interface first:
```typescript
export interface SectionStepCount {
  section_id: number;
  step_count: number;
}
```
Then the function:
```sql
SELECT section_id, COUNT(*) AS step_count
FROM recipe_steps
WHERE section_id IS NOT NULL
GROUP BY section_id
```
Params: `[]`. The `WHERE section_id IS NOT NULL` guard excludes any orphaned steps (should be none after migration, but defensive).

**Hook 4 — useDeleteRecipeSection() — CRITICAL: 5-key invalidation contract:**
- mutationFn: `({ id }) => deleteRecipeSection(id)`
- Mutation input type: `{ id: number; recipeId: number }`
- onSuccess MUST invalidate ALL 5 keys with this exact comment block:
```typescript
/**
 * CASCADE INVALIDATION CONTRACT -- do not reduce.
 * Deleting a section cascades to recipe_steps (ON DELETE CASCADE).
 * Those step deletions make 4 additional cache keys stale:
 *   RECIPE_PAINTS_KEY   -- per-recipe step list
 *   STEP_COUNTS_KEY     -- batch step counts
 *   RECIPE_AVAILABILITY_KEY -- paint availability badge
 *   RECIPE_SWATCH_KEY   -- swatch color strip
 * All 5 keys must be invalidated to keep UI consistent.
 */
```
The 5 invalidation calls:
1. `qc.invalidateQueries({ queryKey: RECIPE_SECTIONS_KEY(variables.recipeId) })`
2. `qc.invalidateQueries({ queryKey: RECIPE_PAINTS_KEY(variables.recipeId) })`
3. `qc.invalidateQueries({ queryKey: STEP_COUNTS_KEY })`
4. `qc.invalidateQueries({ queryKey: RECIPE_AVAILABILITY_KEY })`
5. `qc.invalidateQueries({ queryKey: RECIPE_SWATCH_KEY })`

**Hook 5 — useReorderRecipeSections():**
- mutationFn: `({ sections }) => reorderRecipeSections(sections)`
- Mutation input type: `{ sections: { id: number; order_index: number }[]; recipeId: number }`
- onSuccess: invalidate `RECIPE_SECTIONS_KEY(variables.recipeId)` only

**Hook 6 — useSectionStepCounts():**

```typescript
queryFn: async () => {

},
```
