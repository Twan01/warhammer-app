### DS-06 / DS-08 Partial Detail

`DashboardPage` renders `<UnitDetailSheet>` without `onDatasheetConflict` or `pendingImportResolution` props. Since these props are optional, TypeScript accepts this silently. When a user opens a unit from the Dashboard and imports a datasheet where imported stats conflict with manually-entered stats, `onDatasheetConflict?.()` is a no-op — the conflict dialog never mounts and conflicting fields are silently dropped.

The primary entry point for datasheet import is CollectionPage, where the wiring is complete. This is a Dashboard-path limitation only.
