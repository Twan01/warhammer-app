## Interaction Contracts

### CurrentFocusCard — Applied Recipe Progress Display (D-06, D-07, D-09)

**When `assignmentProgress` prop is provided (unit has at least one applied assignment):**

Replace the `workflowPosition` hint line with:

```
[Palette icon 12px] [recipeName] [X]/[totalSteps] steps[suffix]
```

- Icon: `Palette` (Lucide, 12px) — same as existing `recipeName` line, `aria-hidden`
- Recipe name: truncated at 20 chars with ellipsis if > 20 chars (matches existing KanbanCard pattern)
- Progress fraction: tabular-nums, format `X/Y steps`
- Suffix: ` (+N more)` appended when `extraRecipeCount > 0` — matches existing `extraRecipeCount` prop behavior
- Color: `text-muted-foreground`, size `text-xs` (12px / regular)
- The `workflowPosition` span is NOT rendered when `assignmentProgress` is provided
- The painting_percentage progress bar remains unchanged below this line

**When `assignmentProgress` prop is null/undefined (no applied assignment — existing behavior):**

Render existing `workflowPosition` hint line unchanged. Progressive enhancement — no regression for units without assignments.

**Loading state:**
- While assignment data is being fetched, render the existing `workflowPosition` hint if available, or nothing if absent — no skeleton in the progress line (avoids layout shift on a secondary metadata field)

### KanbanCard — Applied Recipe Progress Display (D-06, D-07, D-08)

**When `assignmentProgress` and `totalSteps` props are provided:**

Replace the `workflowPosition` italic paragraph with a progress fraction line in the existing metadata row:

```
[existing metadata: updated_at · recipeName (truncated)]
[NEW LINE]: [X]/[Y] steps
```

- Rendered as a `<span>` in the existing `flex-wrap` metadata row below the Progress bar
- Format: `[completedSteps]/[totalSteps] steps` — tabular-nums
- Color: `text-xs text-muted-foreground`
- The `workflowPosition` italic `<p>` block is NOT rendered when `assignmentProgress` is provided

**When no assignment (existing behavior):**

Render existing `workflowPosition` italic hint unchanged.

**Multiple assignments (D-08):**

Display primary assignment (most recently updated by `updated_at`). If `extraAssignmentCount > 0`, append ` (+N more)` to the progress line.

### LogSessionSheet — Auto-Completion Bridge (D-01 through D-04)

**No new UI elements added to the form.** The bridge is invisible to the user
except for toast notifications.

**onSubmit flow (after session create succeeds):**

```
createSession.mutateAsync → success
  ↓
if (recipe_step_id selected):
  1. Check useAssignmentsByUnit for selected unit + recipe
  2. If no assignment: call useCreateAssignment (silent, no toast)
  3. Call useToggleStepProgress(assignmentId, step.order_index, completed=true)
  4. On step progress success: no toast (silent — included in main success toast)
  5. On step progress failure: toast.warning("Session logged but step progress update failed.")
     → Close sheet (partial-failure pattern, same as existing status update failure)
```

**Success toast (no new_status, no failure):** "Session logged." (unchanged)
**Success toast (with new_status):** "Session logged and status updated." (unchanged)
**Partial failure — step progress:** "Session logged but step progress update failed."
**Partial failure — status update (existing):** "Session logged but status update failed." (unchanged)

The step progress partial-failure copy follows the existing status-update failure
pattern from the same file (same phrasing shape: "Session logged but X failed.").
