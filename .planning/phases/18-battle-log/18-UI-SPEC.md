## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge padding, result badge internal padding |
| sm | 8px | Compact element spacing, gap between row lines, icon button gap |
| md | 16px | Default element spacing, form field gaps (`gap-4`), Sheet content padding |
| lg | 24px | Page outer padding (`p-6`), section gaps |
| xl | 32px | Not used — no major layout breaks within this page |
| 2xl | 48px | Empty-state vertical padding (via `py-16` = 64px pattern — see exception) |
| 3xl | 64px | Page-level spacing (empty state `py-16`) |

Exceptions:
- Empty state containers: `py-16` (64px top/bottom) — established pattern from Phase 16 UI-SPEC. Not a spacing token; it is the structural empty-state layout override.
- Row hover action buttons: minimum 32px touch target (`h-8 w-8`) — icon-only buttons.
- Summary bar: `py-3` (12px top/bottom) — internal padding for the summary strip between the page header and the list; accepted exception for this self-contained strip component.

**Source:** Phase 16 UI-SPEC spacing scale; Phase 16 empty-state pattern. Page outer wrapper is `flex flex-col gap-6 p-6` (standard for all pages per Phase 16).

## Color

All tokens reference existing CSS variables from `globals.css`. No new color tokens are introduced in Phase 18.

| Role | Token | Value (dark mode) | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | `--background` | `hsl(240 10% 3.9%)` | Page background, Sheet panel background, expanded notes section background |
| Secondary (30%) | `--card` / `--muted` | `hsl(240 10% 5.9%)` | Log entry row hover state (`hover:bg-muted/50`), expanded section background (`bg-muted/30`) |
| Accent (10%) | `--faction-accent` | `#71717a` (default zinc-500, runtime-mutable) | See reserved-for list below |
| Destructive | `--destructive` | `hsl(0 72.2% 50.6%)` | Delete icon button color, delete confirmation dialog |
| Win badge | green | `bg-green-500/20 text-green-500` | WIN result pill — semantic, not faction accent |
| Loss badge | red | `bg-red-500/20 text-red-500` | LOSS result pill — semantic, not faction accent |
| Draw badge | muted | `bg-muted text-muted-foreground` | DRAW result pill — neutral semantic color |

**Accent reserved for:**
- Primary CTA button ("Log Game") — `variant="default"` uses `bg-faction-accent` (established from Phase 10)
- Edit icon button hover state: `hover:text-faction-accent` (consistent with other edit actions in the codebase)

**NOT used for:** row hover backgrounds, result badges (those use semantic green/red/muted), expanded section backgrounds, form inputs.

**Result badge implementation:** Use `<Badge>` shadcn component with custom className overrides. Do NOT use `variant="default"` (which applies faction-accent). Instead apply explicit bg/text classes:
- WIN: `className="bg-green-500/20 text-green-500 border-0 font-semibold"`
- LOSS: `className="bg-red-500/20 text-red-500 border-0 font-semibold"`
- DRAW: `className="bg-muted text-muted-foreground border-0 font-semibold"`

**Source:** 18-CONTEXT.md §Win/Loss/Draw styling ("color-coded pill badge: green for WIN, red for LOSS, yellow/muted for DRAW"); Phase 16 UI-SPEC §Color; globals.css dark mode tokens.

Uses the shadcn `<Collapsible>` component. Expand is triggered by clicking the row body (not the action buttons). Animation: `data-[state=open]:animate-in data-[state=closed]:animate-out` — use the tw-animate-css classes already in globals.css.

      <div>
        <span className="text-muted-foreground">Opponent </span>

      <div>
        <span className="text-muted-foreground">Underperformed </span>
        <span>{underperforming_unit_name}</span>
      </div>
    )}
    {/* Lessons learned */}

Fields shown in expanded section only if non-null. Fields with no content collapse out entirely (no "—" placeholder in the expanded view — absence is sufficient signal).

**General `notes` column decision (Claude's Discretion):** Do NOT expose `notes` as a fourth freeform field in the UI. The three structured fields (`lessons_learned`, `changes_next_time`, `opponent` + `mvp_unit_id` + `underperforming_unit_id`) give adequate post-game capture. Adding a fourth generic notes field would create UX confusion about which text field to use. The `notes` column remains in the DB schema but is not surfaced in this phase.

## Interaction Contract

### Page-level portal state

BattleLogPage owns all Sheet/Dialog state — mirrors ArmyListsPage architecture exactly:

```
const [sheetOpen, setSheetOpen] = useState(false);
const [editingLog, setEditingLog] = useState<BattleLog | null>(null);
const [deleteDialogOpen, setDeleteDialogOpen] = useState(false);
const [deletingLog, setDeletingLog] = useState<BattleLog | null>(null);
```

No `selectedLogId` pattern needed — battle log entries expand inline (no detail sheet).

### Row expand/collapse

- Clicking anywhere on the row body (not the edit/delete buttons) toggles the `CollapsibleContent` section.
- Expand/collapse uses shadcn `<Collapsible>` — `open` state is local to each `BattleLogRow` component (`useState<boolean>(false)`).
- Animation: use `data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0` on `CollapsibleContent` — same pattern as Phase 15 DatasheetAbilities collapsible. Instant open is acceptable if animation adds complexity.
- A row with no expandable content (no scores, no opponent name, no unit links, no notes fields set) still responds to click but the expanded section is empty — acceptable at this scope.

### Edit and delete hover actions

- Both icon buttons (`<Button variant="ghost" size="icon">`) are `invisible group-hover:visible` on the parent row's `group` class.
- Edit button: `onClick` calls `openEdit(log)` — stops propagation to prevent simultaneously toggling the row expand.
- Delete button: `onClick` calls `openDelete(log)` — stops propagation.
- Both buttons use `e.stopPropagation()` on their click handlers.

### Form validation

Required fields: `battle_date`, `opponent_faction`, `mission`, `result`. All other fields are optional (nullable).

- `battle_date`: required, type="date", default value = today's date formatted as `YYYY-MM-DD`
- `opponent_faction`: required, min 1 character
- `mission`: required, min 1 character
- `result`: required, one of `"Win" | "Loss" | "Draw"`
- All numeric fields (`points_played`, `my_score`, `opponent_score`): optional, `z.number().int().positive().nullable().optional()`
- All ID fields (`army_list_id`, `mvp_unit_id`, `underperforming_unit_id`): optional, nullable integers
- Text fields (`opponent`, `lessons_learned`, `changes_next_time`): optional, `z.string().nullable().optional()`

Validation errors render below each field via `<FormMessage />` — shadcn pattern.

### Focus and keyboard

- Sheet opens with focus on the first required field (`battle_date`).
- Tab order: battle_date → opponent_faction → mission → result → opponent → points_played → my_score → opponent_score → army_list_id → mvp_unit_id → underperforming_unit_id → lessons_learned → changes_next_time → "Discard changes" button → "Log Game" button.
- Escape closes the Sheet (via `onOpenChange` — shadcn Sheet default).
- Escape closes the delete dialog (shadcn Dialog/AlertDialog default).

### Optimistic UI

Delete is NOT optimistic — use standard `mutateAsync` pattern with loading state. The battle log list is short (personal hobby tool) and the extra round-trip latency is acceptable. Match the ArmyListDeleteDialog pattern (await mutation → close dialog → TanStack Query cache invalidation auto-refetches list).

### Hover transitions

- Row hover: `transition-colors duration-150` — consistent with Phase 16 interaction contract.
- Button hover: `duration-150` — fast, snappy.

| `mission` | `battle_logs.mission` | `TEXT` | yes | required; `""` → validation error |
| `result` | `battle_logs.result` | `TEXT` | yes | required; one of "Win" \| "Loss" \| "Draw" |
| `opponent` | `battle_logs.opponent` | `TEXT` | yes | `""` → `null` on submit |
| `points_played` | `battle_logs.points_played` | `INTEGER` | yes | `""` → `null`; parse to int |
| `my_score` | `battle_logs.my_score` | `INTEGER` | yes | `""` → `null`; parse to int |
