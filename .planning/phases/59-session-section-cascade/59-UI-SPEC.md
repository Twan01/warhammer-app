## Interaction Contract

### 3-Level Cascade Selector Flow

This is the primary interaction introduced in Phase 59. The form gains a conditional
middle layer between recipe and step.

**Selector order (top to bottom in form):**
1. Unit (always shown)
2. Update Painting Status (always shown)
3. Recipe (always shown)
4. Section (conditional — see below)
5. Recipe Step (conditional — already existed)
6. Date, Duration, Notes (always shown)

**Section selector visibility rule:**
- SHOW when: `watchedRecipeId != null` AND `sections.length >= 2`
- HIDE when: recipe is null, or recipe has 0 or 1 sections
- When hidden with 1 section: step selector shows all steps unfiltered (backward-compatible)

**FormLabel text:** "Section" (not "Recipe Section" — shorter, consistent with "Recipe" and "Recipe Step" label brevity)

**SelectItem display format:** section name only (e.g., "Armour", "Basing") — no step count suffix. Keeps the picker uncluttered; step count info is available in the RecipeDetailSheet.

**Placeholder text:** "No section"

**`__none__` sentinel:** Use `"__none__"` for the null option, matching the recipe and step selector pattern exactly. `onValueChange` maps `"__none__"` → `null`.

### Cascade Reset Behavior

| Trigger | Fields Cleared |
|---------|---------------|
| `watchedRecipeId` changes | clears `section_name` (local state) + `recipe_step_id` |
| `watchedSectionId` changes | clears `recipe_step_id` only |

Both resets happen via `useEffect` (same pattern as the existing step-clear effect).

### Step Filtering

When a section is selected, the step dropdown filters client-side:
- Filtered: `recipeSteps.filter(s => s.section_id === watchedSectionId)`
- Unfiltered (no section or single-section recipe): `recipeSteps` unchanged

Steps remain sorted by `order_index` ascending after filtering.

### Loading States

Section selector is not explicitly disabled during section fetch (sections are
fetched from the same React Query cache as the recipe list — no separate loading state
needed). If sections are still loading, the SelectContent renders an empty list,
same as step selector behavior when steps are loading.

### Optionality

All three selectors (recipe, section, step) remain optional. The form can submit with:
- No recipe, section, or step selected
- Recipe only
- Recipe + section only
- Recipe + step only (single-section recipe)
- Recipe + section + step

No validation errors for missing section — it is never required.

## State Contract

| State | Location | Type | Notes |
|-------|----------|------|-------|
| `section_name` | React Hook Form field | `string \| null \| undefined` | Submitted to DB; denormalized text |
| `watchedSectionId` | Local `useState` or `useMemo` | `number \| null` | Client-only; used for step filtering; not submitted |
| `sections` | `useRecipeSections(watchedRecipeId)` | `RecipeSection[]` | Fetched reactively |
| `filteredSteps` | Derived from `recipeSteps` + `watchedSectionId` | `RecipeStep[]` | `useMemo` preferred for referential stability |
