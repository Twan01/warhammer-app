## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Table cell padding (vertical), button group gaps, filter chip gaps |
| md | 16px | Page content padding, form field gaps, sheet padding |
| lg | 24px | Section spacing within sheet, filter bar to table gap |
| xl | 32px | Page header to content gap |
| 2xl | 48px | Empty state vertical padding (py-12) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Table row height: 44px minimum (touch/click target standard for interactive rows)
- Active project icon badge: 16px × 16px (h-4 w-4) — already used in Phase 2 for icon sizing

Source: Phase 2 established `p-6 / gap-6 / gap-4 / gap-2 / gap-1` — continued as-is (FactionsPage.tsx, FactionRow.tsx)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Body variant / secondary text | 12px (text-xs) | 400 (normal) | 1.4 |
| Page heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Detail sheet field heading / column header label | 12px (text-xs) | 600 (semibold) | 1.2 |

Rules:
- Exactly 2 weights in use: 400 (normal) and 600 (semibold). Weight 500 / `font-medium` is NOT used anywhere in this phase.
- Table cell content: 14px / weight 400.
- Unit name within cell: 14px / weight 400 (same as body — no weight promotion in cell text).
- Column header labels: 12px / weight 600 (`font-semibold`) / uppercase / tracking-wide — use `text-xs font-semibold text-muted-foreground uppercase tracking-wide`. All `font-medium` in column headers must be replaced with `font-semibold`.
- Page title "Collection": 20px / semibold — matches `text-xl font-semibold` from FactionsPage.tsx.
- SheetTitle: 16px / semibold — inherits shadcn Sheet defaults. Counts as the "heading" weight tier.

Source: Observed in `FactionsPage.tsx`, `FactionRow.tsx`, `UnitSheet.tsx` — continue same scale with weight 500 collapsed into 600.

## Color

All values reference the dark-mode CSS variables from `src/styles/globals.css` (`.dark` block). This app is dark-mode-only at runtime.

| Role | CSS Variable | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` | hsl(240 10% 3.9%) — near-black slate | Page background, table rows |
| Secondary (30%) | `--card` / `--sidebar` | hsl(240 10% 5.9%) — dark zinc | Sheet panels, sidebar, filter bar, popover background |
| Accent (10%) | `faction.color_theme` hex | User-defined per faction | Faction badge background only |
| Destructive | `--destructive` | hsl(0 72.2% 50.6%) — red | Delete button icon, delete dialog confirm button |
| Muted | `--muted-foreground` | hsl(240 5% 64.9%) | Secondary text, column labels, empty state copy |

Accent reserved for:
1. Faction badge background in table rows (`<Badge style={{ backgroundColor: faction.color_theme }}>`) — POLISH-05
2. Faction badge background in unit detail Sheet header

The accent is NOT used for: interactive hover states, sort icons, filter chips, progress bar fill, active project indicator, or any generic interactive element. Progress bar uses shadcn Progress default (primary foreground). Active project uses a lucide `Flame` or `Star` icon in `text-muted-foreground` (toggled to `text-primary` when active).

Source: CONTEXT.md §Faction color accent; `FactionRow.tsx` for pattern reference; REQUIREMENTS.md POLISH-05.

## Visual Hierarchy

Primary focal point: UnitTable is the primary visual focal point of the Collection page. The "Collection" page heading anchors the top-level hierarchy. The filter bar is secondary (above the table, visually subordinate). The detail Sheet is a tertiary layer (overlay, right-side panel).

## Interaction Contracts

### UnitTable (COLL-01, COLL-10, COLL-11)

- Columns in order: Name, Faction, Category, Status, Progress, Points, Models, Active, Actions
- Column widths: Name (flex-grow), Faction (120px), Category (120px), Status (140px), Progress (100px), Points (70px), Models (70px), Active (60px), Actions (80px)
- Row click (anywhere except Action column buttons) opens the unit detail Sheet. The `<tr>` element receives `aria-label="View {unit.name}"` to communicate the row's purpose to assistive technology.
- Sortable columns: Name, Faction, Status, Points, Models — click header to toggle ASC/DESC; default sort is Name ASC
- Sort indicator: `ChevronUp` / `ChevronDown` lucide icon (h-3 w-3) inline with header text; unsorted column shows `ChevronsUpDown` (h-3 w-3) in `text-muted-foreground`
- Pagination: 25 rows per page; controls at table footer — `< Prev [page N of M] Next >` using Button `variant="outline"` `size="sm"`
- Row hover: `hover:bg-muted/50` — same pattern as FactionRow.tsx unit list items

### Actions Column Accessibility

The Actions column (80px) contains icon-only buttons. Each must carry an explicit `aria-label`:

| Button | aria-label |
|--------|------------|
| Delete icon button | `aria-label="Delete {unit.name}"` |
| Edit icon button (if separate from row click) | `aria-label="Edit {unit.name}"` |

The clickable row itself (for opening the detail Sheet) is not an icon button but must have `aria-label="View {unit.name}"` on the `<tr>` or its wrapping interactive element.

### Status Badge / Popover (COLL-10)

- Status rendered as a `<Badge variant="outline">` with status label text
- Badge is keyboard-focusable and clickable — opens a Popover anchored to the badge
- Popover contains a `<Command>` list with all `PAINTING_STATUS_ORDER` entries
- Current status shown with a `Check` icon (h-4 w-4) on the left of the item
- Selecting a new status: closes popover immediately, triggers `useUpdateUnit` with optimistic update, shows Sonner error toast on failure and reverts badge
- Same popover pattern used inside the unit detail Sheet (CONTEXT.md §Unit detail Sheet)
- Pattern source: `CategoryCombobox.tsx` (Popover + Command + shouldFilter)

### Unit Detail Sheet (COLL-09)

- Opens from row click; right-side Sheet (`side="right"`)
- `key={selectedUnit.id}` on SheetContent — prevents stale data when switching units (POLISH-04)
- Content layout: SheetHeader (unit name + faction badge), then stacked field rows in read-only styled text
- Fields displayed: Name, Faction (colored badge), Category, Status (clickable badge), Painting % (Progress bar), Assembly, Basing, Varnished (checkbox-icon read-only), Active Project (icon), Points, Model Count, Owned Count, Priority, Target Date, Purchase Date, Purchase Price, Storage Location, Notes
- Footer buttons: `Edit Unit` (primary) — opens UnitSheet for full edit; `Delete Unit` (ghost, destructive color) — opens delete confirm dialog
- "Edit Unit" button: `variant="default"` width full or auto
- "Delete Unit" button: `variant="ghost"` with `text-destructive` class, positioned left in footer

### Filter Bar (COLL-02 through COLL-07)

- Layout: single horizontal row above the table — `flex items-center gap-2 flex-wrap`
- Search input: `<Input placeholder="Search units..." />` — 200px min-width, live filters on `onChange` (no debounce needed for local data)
- Faction multi-select: Popover trigger button `variant="outline" size="sm"` labeled "Faction" with count badge when active — dropdown shows faction list with checkboxes
- Status multi-select: same pattern as faction — labeled "Status"
- Category multi-select: same pattern — labeled "Category"
- Active project toggle: `<Button variant={active ? "default" : "outline"} size="sm">Active only</Button>` — toggles on/off, no popover
- "Clear filters" text button: `variant="ghost" size="sm"` — appears only when any filter is active; positioned right of filter row
- Filter state: Zustand store in `src/features/units/collectionFilters.ts` (ephemeral, not persisted)

### Loading State (POLISH-02)

- While `useUnits()` `isLoading`: render 5 skeleton rows — `<Skeleton className="h-10 w-full" />` — in place of the table body
- Table header remains visible during loading (no skeleton for headers)
- Filter bar remains visible and interactive during loading (no skeleton)

### Empty State (COLL-12)

- Shown when: units list is empty after filtering, OR no units exist at all
- Layout: `flex flex-col items-center justify-center gap-4 py-16 text-center` — matches `FactionsEmptyState.tsx` pattern exactly
- Icon: `PackageSearch` (lucide-react, h-12 w-12, `text-muted-foreground`)
- Heading: "No units found" (14px filtered state) or "Add your first unit" (no data state) — see Copywriting Contract
- Body text: see Copywriting Contract
- CTA button: `variant="default"` — "Add Unit"

### Delete Confirm (COLL-13 / POLISH-01)

- Reuse `UnitDeleteDialog.tsx` (existing Phase 2 component)
- Trigger: "Delete Unit" button in table Actions column OR in detail Sheet footer
- Dialog: AlertDialog pattern — destructive confirm button labeled "Delete unit"
