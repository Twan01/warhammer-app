    Per D-07 and D-08, create a new batched query and refactor the enrichment hook.

    **Step 1: Add getKanbanProgressByUnitIds to recipeAssignments.ts**

    Export a new interface `KanbanProgressRow` with fields: unit_id (number), assignment_id (number), assignment_count (number), recipe_id (number), recipe_name (string), total_steps (number), completed_steps (number).

    Export `async function getKanbanProgressByUnitIds(unitIds: number[]): Promise<KanbanProgressRow[]>` that:
    - Returns `[]` if unitIds is empty (guard clause)
    - Gets db via `getDb()`
    - Builds IN-clause with `unitIds.map((_, i) => "$" + (i + 1)).join(", ")`
    - Uses a CTE with ROW_NUMBER() OVER (PARTITION BY unit_id ORDER BY created_at DESC) to find the most-recent assignment per unit, plus COUNT(*) OVER (PARTITION BY unit_id) for total_assignments
    - JOINs painting_recipes (for name), LEFT JOINs recipe_steps and unit_recipe_step_progress (for total/completed steps)
    - Filters WHERE rn = 1, GROUP BY unit_id, id, recipe_id, name, total_assignments
    - Passes `unitIds` as params array (only one IN-clause occurrence in the CTE, single param set)

    Follow the SQL structure from 98-PATTERNS.md Pattern 4 (CTE approach). If ROW_NUMBER is not available (fallback), use subquery approach from 98-RESEARCH.md.

    **Step 2: Refactor useKanbanEnrichment.ts**

    Replace the Promise.all + sortedIds.map loop (lines 37-56) with:
    1. Call `getKanbanProgressByUnitIds(sortedIds)` -- single batched query
    2. Iterate the returned rows to populate `appliedProgressMap` and `assignmentIdsMap` with the same shape as before

    Remove now-unused imports: `getAssignmentsByUnit`, `getStepProgress` from recipeAssignments, `getRecipePaintsByRecipe` from recipePaints, `computeAssignmentProgress` from lib, `getRecipeById` from recipes.

    Add new import: `getKanbanProgressByUnitIds` from `@/db/queries/recipeAssignments`.

    Keep existing batched calls for recipeRows and photoRows (lines 30-33) per D-08.

    The KanbanEnrichment return shape (recipeNames, photoCounts, appliedProgress, assignmentIds) must remain identical.

    **Step 3: Create tests/performance/kanbanBatchEnrichment.test.ts**

    Mock `getDb` to return `{ select: selectMock }`. Test:
    - `getKanbanProgressByUnitIds([1, 2])` calls select once with SQL containing `IN ($1, $2)` and params `[1, 2]`
    - `getKanbanProgressByUnitIds([])` returns empty array without calling select (guard clause)
    - SQL contains `ROW_NUMBER()` and `unit_recipe_assignments`
