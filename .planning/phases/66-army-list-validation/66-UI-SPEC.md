## Component Inventory

### 1. Extended ArmyListSummaryBar (modify existing)

**Location:** `src/features/army-lists/ArmyListSummaryBar.tsx`

Current layout (3 stats + progress bar + not-ready list) extends to:

```
+----------------------------------------------------------+
| Total: 1500 / 2000 pts   Owned: 100%   Ready: 75%       |  <- stat row
| [===========----------] 75%                               |  <- progress bar
| [freshness dot] Synced 2 days ago    Warnings: 3 (1 hard)|  <- freshness + warnings row
|                                                           |
| ROLE COVERAGE (only if any unit has a tactical_role)      |
| [Anti-Tank 2] [Screening 1] [Obj Holder 0] [Fire 3] ... |  <- role pills
|                                                           |
| NOT READY (2)                                             |  <- existing not-ready list
| Unit A                              Basecoated            |
| Unit B                              Not Started           |
+----------------------------------------------------------+
```

**Stat row changes:**
- "Total" stat becomes `X / Y pts` when `points_limit` is set; just `X pts` when NULL
- Total text uses `text-destructive` when sum exceeds points_limit
- Add "Owned" stat: `N%` computed as owned units / total units * 100
- "Painted" stat renamed to "Ready" (shorter, matches existing `battleReadyPct` variable)
- Stats remain in a single `flex items-center gap-6` row

**New freshness + warnings row:**
- Left side: embed `<PointsFreshnessBadge />` inline (self-contained, no props)
- Right side: warning count displayed as `Warnings: N` with count in `text-destructive` if any hard warnings, `text-amber-500` if soft-only, hidden if zero warnings
- Row uses `flex items-center justify-between` layout

**New role coverage section:**
- Section label: `text-xs font-semibold text-muted-foreground uppercase tracking-wide` reading "Role Coverage"
- Only rendered when at least 1 unit in the list has a non-null `tactical_role` (progressive disclosure per D-11)
- 7 pill badges in a `flex flex-wrap gap-1.5` container
- Each pill shows: role display label + count of assigned units
- Covered roles (count >= 1): `bg-secondary text-secondary-foreground rounded-full px-2 py-0.5 text-xs`
- Gap roles (count === 0): `bg-transparent border border-dashed border-muted-foreground/40 text-muted-foreground rounded-full px-2 py-0.5 text-xs`

**Props change:** Add `pointsLimit: number | null` and `armyListId: number` to props (units array already passed).

### 2. Inline Warning Icons on ArmyListUnitRow (modify existing)

**Location:** `src/features/army-lists/ArmyListUnitRow.tsx`

Add a warning icon in a new column (column 2, shifting existing columns right) OR prepend to the unit name cell.

**Decision: prepend to unit name cell** — avoids table column count change, keeps compact layout.

- Hard warning present: `<AlertTriangle className="h-3.5 w-3.5 text-destructive inline mr-1" />` before unit name
- Soft warning only: `<Info className="h-3.5 w-3.5 text-amber-500 inline mr-1" />` before unit name
- No warnings: no icon (clean default)
- Tooltip on icon: list the specific warnings for that unit (e.g., "Points exceeded, Not painted")

**Warning computation:** Call `computeUnitWarnings(unit, totalPoints, pointsLimit)` which returns `{ hard: string[], soft: string[] }`. Icon shown based on `hard.length > 0` or `soft.length > 0`.

### 3. Tactical Role Select on ArmyListUnitRow (modify existing)

**Location:** `src/features/army-lists/ArmyListUnitRow.tsx`

Add an inline Select dropdown below the unit name (same cell, below the active loadout name).

```
Unit Name
  Active Loadout Name          <- existing
  [Select role... v]           <- new tactical role dropdown
```

- Uses shadcn `<Select>` (already imported in this file)
- Trigger: `w-28 h-6 text-xs` (compact, matches tier selector sizing)
- Placeholder: "Role..."
- Options: 7 fixed roles from `TACTICAL_ROLES` const array, displayed with human-readable labels
- An 8th option: "None" to clear the assignment (sets `tactical_role` to null)
- On value change: call `updateArmyListUnit.mutate()` with the new `tactical_role` value, preserving `points_override` and `notes` (Pitfall 2 contract)

**Role display labels (TACTICAL_ROLES_DISPLAY):**

| Value | Display Label |
|-------|--------------|
| anti_tank | Anti-Tank |
| screening | Screening |
| objective_holder | Obj. Holder |
| fire_support | Fire Support |
| melee_threat | Melee Threat |
| utility | Utility |
| transport | Transport |

### 4. Points Limit Input on ArmyListSheet (modify existing)

**Location:** `src/features/army-lists/ArmyListSheet.tsx`

Add a `points_limit` number input field to the army list create/edit form.

- Label: "Points Limit"
- Input type: number, min 0, step 50
- Placeholder: "No limit"
- Position: below existing form fields, before the submit button
- When empty/cleared: stores NULL (no limit enforced)
- Zod schema: `points_limit: z.number().int().min(0).nullable().optional()`

### 5. computeUnitWarnings Pure Function (new)

**Location:** `src/lib/computeUnitWarnings.ts`

```typescript
interface UnitWarnings {
  hard: string[];
  soft: string[];
}
```

**Hard warnings (list cannot legally be played as-is):**
- "Points exceeded" — when total list points > points_limit (passed as context, not per-unit)
- "Not in collection" — future use, currently always false per D-15

**Soft warnings (informational for hobby preparation):**
- "Not painted" — status_painting is not "Completed"
- "Not assembled" — status_assembly is 0/false
- "Stale points" — effective_points sourced from stale sync (derive from freshness context)
- "Manual override" — points_override IS NOT NULL
- "Unknown points" — effective_points === 0

### 6. Loading / Skeleton States

**Summary bar loading:** Show 3 `<Skeleton className="h-4 w-20" />` elements in the stat row position, plus `<Skeleton className="h-0.5 w-full" />` for the progress bar. Matches existing Skeleton usage pattern (PointsFreshnessBadge already uses `<Skeleton className="h-2 w-16" />`).

**Role coverage loading:** Not needed — role coverage section is conditionally rendered and data comes from the same query as units (no separate loading state).

## Interaction Contract

### Warning Icon Tooltip
- Trigger: hover on AlertTriangle or Info icon
- Content: comma-separated list of warning strings from `computeUnitWarnings`
- Tooltip component: shadcn `<Tooltip>` + `<TooltipTrigger>` + `<TooltipContent>` (matches PointsFreshnessBadge pattern)

### Tactical Role Dropdown
- Trigger: click on Select trigger on unit row
- Behavior: immediate save on value change (no confirmation step) — matches existing points_override blur-save pattern
- Success feedback: no toast (inline UI update is sufficient, matches tier selector UX)
- Error feedback: `toast.error("Failed to update role. Please try again.")`

### Points Limit Input
- Part of the ArmyListSheet form — saved with the form's submit action, not inline
- Clearing the field sets points_limit to NULL

### Warning Count Badge Tooltip (Claude's Discretion)
- Show a tooltip on the warning count in the summary bar
- Content: `{hardCount} critical, {softCount} informational` breakdown
- Only shown when warning count > 0

### Summary Bar Stat Order
- Points total > Owned % > Ready % (stat row)
- Freshness badge > Warning count (second row)
- Role coverage section (third section, conditional)
- Not-ready unit list (final section, existing)
