## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline badge padding |
| sm | 8px | Intra-card element spacing (e.g. label below number) |
| md | 16px | Card internal padding, list row vertical padding |
| lg | 24px | Section-to-section vertical gap, page horizontal padding |
| xl | 32px | Grid gap between stat cards |
| 2xl | 48px | Major section breaks (between top row, progress row, faction row, lists row) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Faction cards minimum width: 180px (prevents collapse below readable text on narrow windows)
- Faction card left border accent: 4px solid faction.color_theme
- List row height: 44px minimum (touch/click target standard — applies to active projects and recently updated rows)
- Empty state icon: 48px (h-12 w-12) matching CollectionEmptyState established pattern

Source: Established by prior phases (FactionsPage uses `p-6`, `gap-6`, `gap-4`; KanbanCard uses `p-3`, `gap-2`)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 400 (regular) | 1.4 |
| Section heading | 16px (text-base) | 600 (semibold) | 1.25 |
| Page heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Stat display number | 28px (text-3xl) | 600 (semibold) | 1.0 |

Notes:
- Stat display number (28px semibold) is used only for the large number inside the 7 stat cards (4 top + 3 progress). No other element uses this size. The 28px size alone provides sufficient visual differentiation; bold weight is not needed.
- Section headings (16px semibold) match the `text-xl font-semibold` heading established in FactionsPage — use `text-base font-semibold` for subsection labels, `text-xl font-semibold` for page-level heading ("Dashboard").
- Page-level heading "Dashboard": 20px (text-xl), weight 600 — consistent with `h1 className="text-xl font-semibold"` pattern in FactionsPage.
- Stat card label underneath number: 14px, weight 400, color `text-muted-foreground`.
- List row unit name: 14px, weight 600 (semibold).
- Faction card name: 14px, weight 600 (semibold).

Source: Existing components (KanbanCard uses `text-sm font-semibold` for unit name, `text-sm text-muted-foreground` for metadata; CollectionEmptyState uses `text-base font-semibold` for heading)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 10% 3.9%)` — `--background` | Page background |
| Secondary (30%) | `hsl(240 10% 5.9%)` — `--card` | All Card surfaces: stat cards, faction cards, list containers |
| Accent (10%) | `faction.color_theme` (hex, per-faction) | Faction card left border (4px), faction Badge background in list rows only |
| Destructive | `hsl(0 72.2% 50.6%)` — `--destructive` | Not used in this phase (Dashboard is read-only) |

Accent reserved for:
1. Faction card left border — 4px solid `faction.color_theme` per card
2. Faction Badge `backgroundColor` style prop in list rows (Active Projects + Recently Updated) — matching the pattern established in KanbanCard: `style={{ backgroundColor: faction.color_theme }}`

Accent is NOT used on: stat card borders, section headings, CTA button, skeleton loaders, or any generic interactive element.

Muted foreground: `hsl(240 5% 64.9%)` — `--muted-foreground`. Used for all secondary text (stat card labels, list row metadata, empty state body text).

Source: `src/styles/globals.css` dark mode tokens + `src/features/painting-projects/KanbanCard.tsx` faction badge pattern

## Layout Specification

Primary focal point: the 4 top stat cards (specifically the 28px numbers) are the first element the eye reaches; all other sections read in descending visual weight.

### Page structure (vertical stack, top to bottom)

```
DashboardPage (flex flex-col gap-12 p-6)
├── Page heading: "Dashboard" (text-xl font-semibold)
├── [Section] Top Stat Row
│   ├── Section label: none (heading suffices)
│   └── Grid: grid-cols-4 gap-6
│       └── StatCard ×4: Total Models | Fully Painted | Battle-Ready Points | Active Projects
├── [Section] Progress Metrics (gap-8 top margin via section gap)
│   ├── Section label: "Progress" (text-sm font-semibold uppercase tracking-widest text-muted-foreground)
│   └── Grid: grid-cols-3 gap-6
│       └── StatCard ×3: Painting % | Assembly % | Basing %
├── [Section] Faction Summary
│   ├── Section label: "By Faction" (same label style as above)
│   └── Flex: flex flex-wrap gap-4
│       └── FactionCard ×N (min-w-[180px])
└── [Section] Lists
    ├── Section label: none (column headings serve as labels)
    └── Grid: grid-cols-2 gap-6
        ├── ListContainer — "Active Projects" (left)
        └── ListContainer — "Recently Updated" (right)
```

### StatCard anatomy (shared by all 7 stat cards)

```
Card (py-6 px-6)
└── CardContent (flex flex-col gap-1)
    ├── number: text-3xl font-semibold (28px, weight 600)
    └── label: text-sm text-muted-foreground (14px, weight 400)
```

No CardHeader, no CardTitle, no CardDescription — minimal card variant.

### FactionCard anatomy

```
Card (min-w-[180px] border-l-4 border-l-[faction.color_theme] px-4 py-4 cursor-pointer hover:bg-muted/50)
└── flex flex-col gap-2
    ├── faction name: text-sm font-semibold (14px, weight 600)
    ├── "N models" — text-sm text-muted-foreground
    ├── "XX% painted" — text-sm text-muted-foreground
    └── "NNN pts owned / NNN pts painted" — text-sm text-muted-foreground
```

Clicking navigates to `/collection` after setting faction filter in Zustand collectionFilters store.

### ListRow anatomy (Active Projects + Recently Updated)

```
div (flex items-center gap-2 py-2 px-4 min-h-[44px] cursor-pointer hover:bg-muted/50 rounded-md)
├── unit name: text-sm font-semibold (14px, weight 600, flex-1 truncate)
```

Manual implementation (no library dependency). Rules:
- < 1 hour: "Xm" (minutes)
- < 24 hours: "Xh"
- < 7 days: "Xd"

## Data / Formula Contracts

### Painting % (DASH-03)
Formula: `Math.round((sum of painting_percentage across all units) / units.length)` — average painting_percentage by unit count (not by points). Rationale: painting_percentage is already a per-unit 0-100 value; points-weighted average would misrepresent hobby completeness for large underpainted units.

### Assembly % (DASH-04)
Formula: `Math.round((units where status_assembly = 1).length / units.length * 100)`

### Basing % (DASH-04)
Formula: `Math.round((units where status_basing = 1).length / units.length * 100)`

### Fully-painted count (DASH-01)
Formula: `units where status_painting = 'Completed'` — count only.

### Battle-ready points (DASH-01)
Formula: `sum of units.points where status_painting = 'Completed'`. Source: CONTEXT.md.

### Faction painted % (DASH-02)
Formula: `Math.round((faction units where status_painting = 'Completed').length / faction units.length * 100)`

### Active projects count (DASH-01)
Formula: `units where is_active_project = 1` — count only.

### Active projects list (DASH-05)
Query: `units where is_active_project = 1`, ordered by `updated_at DESC`, limit 5.

### Recently updated list (DASH-06)
Query: all units ordered by `updated_at DESC`, limit 5.
