## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, ring-to-text gap inside card |
| sm | 8px | Gap between card metadata rows (name → badge → status → stats) |
| md | 16px | Card horizontal padding, filter bar to grid gap |
| lg | 24px | Page outer padding (matches existing `p-6` on CollectionPage) |
| xl | 32px | Gap between header row and filter bar |
| 2xl | 48px | Empty state vertical padding (`py-16` = 64px, nearest declared) |
| 3xl | 64px | Empty state vertical padding (`py-16` matches existing CollectionEmptyState) |

Exceptions:
- Gallery grid gap: `gap-4` (16px) between cards — standard md token
- Toggle button group: `gap-1` (4px) between the two icon buttons — xs token
- Painting ring diameter: 96px (fixed SVG size, not from spacing scale — design-specified in CONTEXT.md)
- Touch target for toggle buttons: minimum 36px height via `size="icon"` Button variant (shadcn default)

Source: CONTEXT.md §Gallery Card Layout, CollectionPage.tsx (`gap-6 p-6`), CollectionEmptyState.tsx (`py-16 gap-4`)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 14px (text-sm) | 600 (semibold) | 1.25 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Ring percentage | 16px (text-base) | 700 (bold) | 1 (centered SVG text, line-height irrelevant) |

Notes:
- "Body" covers card metadata rows: faction badge label, status text label, model count, points — all `text-sm text-muted-foreground`, weight 400
- "Label" covers unit name on gallery card — `text-sm font-semibold`, weight 600
- "Heading" covers page title "Collection" — `text-xl font-semibold`, matches existing CollectionPage.tsx `h1`
- Ring percentage text: `text-base font-bold` rendered as SVG `<text>` element, centered in the ring

Source: CollectionPage.tsx (`text-xl font-semibold`), UnitTableColumns.tsx (`text-sm`), CONTEXT.md §Card confirmed layout

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` — dark: `hsl(240 10% 3.9%)` | Page background, gallery grid background |
| Secondary (30%) | `hsl(var(--card))` — dark: `hsl(240 10% 5.9%)` | Gallery card surface (`bg-card`), filter bar |
| Accent (10%) | `var(--faction-accent)` — default zinc-500 (`#71717a`) | Faction ring on active FactionSummaryCard only — NOT on PaintingRing |
| Muted foreground | `hsl(var(--muted-foreground))` | Card metadata text (status, model count, points, empty state icon) |
| Destructive | `hsl(var(--destructive))` — dark: `hsl(0 72.2% 50.6%)` | No destructive actions in this phase |
| Primary | `hsl(var(--primary))` | PaintingRing progress arc stroke color |

Accent reserved for:
- Active FactionSummaryCard ring (`ring-faction-accent`) — Phase 11 existing usage
- Active faction Star icon fill (`fill-faction-accent text-faction-accent`) — Phase 11 existing usage
- PaintingRing track: `zinc-600` (`#52525b`) — neutral, NOT faction-accent (locked decision from CONTEXT.md §Painting Ring Appearance)
- PaintingRing fill arc: `hsl(var(--primary))` — app primary, NOT faction-accent (locked decision from CONTEXT.md)
- View toggle active state: `bg-muted` — neutral muted fill, NOT faction-accent (locked decision from CONTEXT.md §View Toggle Placement)

Source: `src/styles/globals.css`, CONTEXT.md §Painting Ring — Appearance ("rings encode painting state, NOT faction identity — no bg-faction-accent on rings")

## PaintingRing Specification

SVG implementation contract for `src/components/common/PaintingRing.tsx`:

| Property | Value |
|----------|-------|
| Outer diameter | 96px (`viewBox="0 0 96 96"`, `width={96} height={96}`) |
| cx / cy | 48, 48 |
| Radius | 38 (leaves 10px for stroke width without clipping) |
| Stroke width | 8px |
| Track circle stroke | `#52525b` (zinc-600 hardcoded — not a CSS variable) |
| Progress arc stroke | `currentColor` with `className="text-primary"` (resolves to `hsl(var(--primary))`) |
| `stroke-linecap` | `round` on progress arc only |
| Rotation | `transform="rotate(-90 48 48)"` to start arc at 12 o'clock |
| `stroke-dasharray` | `2 * Math.PI * 38` ≈ `238.76` |
| `stroke-dashoffset` | `238.76 * (1 - percentage / 100)` |
| Percentage text | SVG `<text x="48" y="48" dominantBaseline="middle" textAnchor="middle"` — `text-base font-bold fill-foreground` |
| Text content | `{percentage}%` |
| `aria-label` | `{percentage}% painted` |
| `role` | `img` |

Source: CONTEXT.md §Painting Ring — Implementation (locked SVG technique), §Painting Ring — Appearance (locked colors)

## Gallery Card Layout Specification

Card structure (top to bottom, all center-aligned horizontally):

```
┌──────────────────────────────┐
│  [PaintingRing 96px]         │  pt-4 pb-2, centered
│  Unit Name (text-sm semibold)│  mt-2
│  [Faction Badge]             │  mt-1
│  Status label (text-sm muted)│  mt-1
│  N models · N pts (text-sm)  │  mt-1
│  [Flame icon if active]      │  mt-1 (conditional)
└──────────────────────────────┘
```

| Property | Value |
|----------|-------|
| Card padding | `px-4 pt-4 pb-4` |
| Card min-width | Governed by grid column width (no explicit min-width) |
| Grid layout | `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4` |
| Card internal layout | `flex flex-col items-center gap-2` |
| Card interaction | `cursor-pointer hover:bg-muted/50` — full card is clickable, opens UnitDetailSheet |
| Card `role` | `button` with `aria-label={unit.name}` and `tabIndex={0}` |
| Active flame icon | `Flame` (lucide-react), `h-4 w-4 text-primary`, shown only when `is_active_project === 1` |
| Faction badge pattern | `<Badge style={{ backgroundColor: faction.color_theme }} className="border-transparent text-white">` — mirrors UnitTableColumns.tsx exactly |
| Status label | `unit.status_painting` value as text — `text-sm text-muted-foreground` |
| Model count + points | `"{model_count} models · {points} pts"` — `text-sm text-muted-foreground`; fallback `"—"` for nulls |

Source: CONTEXT.md §Gallery Card Layout, UnitTableColumns.tsx faction badge pattern

## Loading Skeleton (Gallery Mode)

When `isLoading === true` and view mode is gallery:
- Render a `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4` of 8 skeleton cards
- Each skeleton card: `rounded-xl border bg-card` with same dimensions as a real card
- Internal skeleton layout: `<Skeleton className="h-24 w-24 rounded-full mx-auto" />` for ring placeholder + `<Skeleton className="h-4 w-3/4 mt-2" />` for name + `<Skeleton className="h-5 w-16 mt-1" />` for badge

## Interaction States

| State | Component | Behavior |
|-------|-----------|---------|
| Loading | `UnitGallery` | 8-card skeleton grid replaces card grid |
| Empty (no units) | `UnitGallery` | `<CollectionEmptyState mode="no-data" onAdd={onAdd} />` |
| Empty (filtered) | `UnitGallery` | `<CollectionEmptyState mode="filtered" onClearFilters={onClearFilters} />` |
| Card hover | Gallery card | `hover:bg-muted/50` background shift |
| Card focus | Gallery card | Browser default focus ring (no override) |
| Card click | Gallery card | Opens `UnitDetailSheet` via existing `handleRowClick(unit)` |
| Toggle switch | View toggle | Immediate re-render (no animation); localStorage written synchronously |
| Filter change | Zustand store | `filteredUnits` recomputes; gallery re-renders with new array — no view reset |
| View toggle + active filters | Both views | Filter state in Zustand store is orthogonal to view mode — UI-06 is free by design |

Source: CONTEXT.md §Card Click & Interactions, §View Toggle Persistence, §Empty States, REQUIREMENTS.md §UI-06

|-------------|---------------|
