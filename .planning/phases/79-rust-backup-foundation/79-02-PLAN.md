Add three new Tauri commands to lib.rs, placed after the helper functions added by Plan 01 and before the `run()` function. Then update the invoke_handler registration.

**Command 1: export_backup per D-01/D-03/D-05**
- Signature: `#[tauri::command] async fn export_backup(app: tauri::AppHandle, destination: String) -> Result<String, String>`
- Steps:
  1. Call `vacuum_to_temp(&app).await?` to get temp .db path
  2. Get db_size from `std::fs::metadata(&temp_path).map_err(...)?.len()`
  3. Build `BackupManifest` with: `app_version` from `env!("CARGO_PKG_VERSION").to_string()`, `schema_version` from `get_migrations().len() as u32`, `created_at` from `format_iso8601_now()`, `platform` from `std::env::consts::OS.to_string()`, `db_size_bytes` from db_size
  4. Call `create_backup_zip(&temp_path, &metadata, &std::path::PathBuf::from(&destination))`
  5. Always clean up temp file: `let _ = std::fs::remove_file(&temp_path);` BEFORE checking the zip result
  6. Return `Ok(destination)` on success
- Error path: if create_backup_zip fails, clean up temp file first, then propagate error

**Command 2: validate_backup per D-06/D-07/D-08**
- Signature: `#[tauri::command] async fn validate_backup(path: String) -> Result<BackupManifest, String>`
- Note: This command does NOT need `app: tauri::AppHandle` since it only reads a user-provided zip file
- Steps:
  1. Open file at `path` via `std::fs::File::open`
  2. Create `zip::ZipArchive::new(file)` -- if this fails, the file is not a valid zip
  3. Check `archive.by_name("hobbyforge.db")` exists (map_err to "backup missing hobbyforge.db")
  4. Read `archive.by_name("metadata.json")` into a String via `std::io::Read::read_to_string`
  5. Parse string into `BackupManifest` via `serde_json::from_str`
  6. Return the parsed manifest
- This command is read-only: it does NOT extract files, modify the zip, or write anything to disk

**Command 3: create_safety_backup per D-09/D-10/D-11**
- Signature: `#[tauri::command] async fn create_safety_backup(app: tauri::AppHandle) -> Result<String, String>`
- Steps:
  1. Resolve `app_data_dir` via `app.path().app_data_dir()`
  2. Create `backups/` subdirectory: `std::fs::create_dir_all(app_data_dir.join("backups")).map_err(...)`
  3. Generate safety backup path: `app_data_dir.join("backups").join(format!("safety-{}.zip", format_filename_timestamp()))`
  4. Call `vacuum_to_temp(&app).await?`
  5. Build `BackupManifest` with same fields as export_backup (app_version, schema_version, created_at, platform, db_size_bytes)
  6. Call `create_backup_zip(&temp_path, &metadata, &safety_path)`
  7. Always clean up temp file before checking result
  8. Return `Ok(safety_path.display().to_string())` on success

**Registration update:**
- Change line with `invoke_handler(tauri::generate_handler![bulk_sync_rules, backup_database])` to include all 5 commands:
  `invoke_handler(tauri::generate_handler![bulk_sync_rules, backup_database, export_backup, validate_backup, create_safety_backup])`

Per D-12: Do NOT modify or remove `backup_database`. It stays as-is.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JS invoke -> Rust command | Frontend provides destination path (export) and zip path (validate) |
| Zip file -> serde parse | validate_backup reads JSON from a user-provided zip file |
| Filesystem write | Commands write to user-chosen path and app_data_dir |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-79-01 | Tampering | validate_backup zip path traversal | mitigate | Only read entries by exact name ("hobbyforge.db", "metadata.json") via archive.by_name(); never extract to arbitrary paths |
| T-79-02 | Tampering | validate_backup metadata.json | mitigate | serde typed deserialization into BackupManifest rejects unexpected types/structure |
| T-79-03 | Tampering | export_backup destination path | accept | Path comes from save dialog (Phase 80); Rust just writes to it. Single-user desktop app. |
| T-79-04 | Information Disclosure | safety backup directory | accept | Backups in app_data_dir; standard OS file permissions apply. Single-user tool. |
| T-79-SC | Tampering | npm/cargo installs | mitigate | zip crate verified (12yr, 14M dl/mo); time is existing transitive dep |
</threat_model>
