Note: u.faction_id is an INTEGER in hobbyforge.db while sup.faction_id is a Wahapedia TEXT key, so the JOIN primarily matches on unit_name. The faction_id condition is a secondary filter — when sup.faction_id IS NULL, the points are globally scoped and match all factions. This is the 5-level chain: list override > synced points > unit override > unit default > 0.
