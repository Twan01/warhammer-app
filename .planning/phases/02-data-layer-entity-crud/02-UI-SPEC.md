## Spacing Scale

Standard 8-point scale. All Tailwind spacing classes map to these values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Compact element spacing (`gap-2`, `px-2`, `py-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`), Sheet internal padding |
| xl | 32px | Layout gaps between major sections (`gap-8`) |
| 2xl | 48px | Major section breaks (page-level vertical rhythm) |
| 3xl | 64px | Page-level spacing (reserved; not used in Phase 2 surfaces) |

Exceptions:
- Sidebar collapsed width: 48px (3xl token, justified by icon-only touch target)
- Sidebar expanded width: 240px (fixed; not from scale — existing Phase 1 decision)
- Nav item vertical padding: 6px (`py-1.5`) — existing NavItem.tsx pattern; preserve as-is
- Faction color-theme left border strip: 4px — CONTEXT.md decision; not from scale

## Typography

All sizes use the system sans-serif stack (no custom font loaded in v1).

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Table cell content, form field values, description text |

| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` | Page title (e.g. "Factions"), Sheet title |

## Color

All values reference dark-mode tokens (`.dark` class active at all times in v1).

| Role | CSS Variable | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` | `hsl(240 10% 3.9%)` — near-black slate | Page background (`bg-background`) |
| Secondary (30%) | `--card` | `hsl(240 10% 5.9%)` — dark slate | Cards, Sheet panel, Sidebar (`bg-card`), list rows |
| Surface border | `--border` | `hsl(240 3.7% 15.9%)` | Table borders, card outlines, sidebar `border-r` |
| Muted | `--muted` | `hsl(240 3.7% 15.9%)` | Skeleton backgrounds, inactive row hover |
| Accent (10%) | `--accent` | `hsl(240 3.7% 15.9%)` | Active nav item background only (see reserved list) |
| Destructive | `--destructive` | `hsl(0 72.2% 50.6%)` — red | Delete buttons, destructive toast, FK-error toast |
| Primary | `--primary` | `hsl(0 0% 98%)` — near-white | Primary Button background, focus ring |
| Muted foreground | `--muted-foreground` | `hsl(240 5% 64.9%)` — slate grey | Placeholder text, secondary labels |

**Faction color-theme accent (inline, not a CSS variable):**
- Applied as `style={{ borderLeftColor: faction.color_theme }}` — 4px left border strip on each faction list row
- Scope in Phase 2: Faction list rows only
- Phase 3+ extends to unit cards — do not implement broadly in Phase 2
- Source: CONTEXT.md §Faction color picker

**Accent CSS token reserved for:**
1. Active navigation item background (`bg-accent` in NavItem.tsx)
2. Navigation item hover state (`hover:bg-accent`)
— No other elements use `bg-accent` in Phase 2

**Destructive color reserved for:**
1. Delete action buttons (ghost/outline variant with destructive text)
2. Sonner toast for FK violation errors
3. Delete confirm dialog confirm button

## Interaction Contracts

### Sheet (Add / Edit forms)

- Width: default shadcn Sheet size (`sm` = 384px from right edge) — CONTEXT.md §Claude's Discretion
- Trigger: "Add [Entity]" button (page header) or row action "Edit" button
- `key={unit.id}` / `key={faction.id}` / `key={paint.id}` on the form root — prevents stale state on re-open (POLISH-04)
- Submit: calls mutation → on success: close sheet + show success toast; on error: show error toast, keep sheet open
- Cancel: closes sheet immediately, no confirmation needed

### Unit Form — Two-Step Layout (CONTEXT.md locked decision)

```
Sheet content structure:
  ┌─ Required fields (always visible) ─────────────────────┐
  │  name (Input, required)                                  │
  │  faction (Select dropdown, required)                     │
  │  category (Combobox with suggestions, required)          │
  └──────────────────────────────────────────────────────────┘
  ┌─ "More details" collapsible (Separator + toggle) ───────┐
  │  painting_status (Select, PAINTING_STATUS_ORDER order)   │
  │  painting_percentage (Input type=number, 0-100)          │
  │  status_assembly (Checkbox)                              │
  │  status_basing (Checkbox)                                │
  │  status_varnished (Checkbox)                             │
  │  is_active_project (Checkbox)                            │
  │  priority (Input type=number)                            │
  │  target_completion_date (Input type=date)                │
  │  model_count (Input type=number)                         │
  │  owned_count (Input type=number)                         │
  │  points (Input type=number)                              │
  │  purchase_date (Input type=date)                         │
  │  purchase_price (Input type=number)                      │
  │  storage_location (Input)                                │
  │  main_image_path (Input — text only, no upload UI)       │
  │  notes (Textarea)                                        │
  └──────────────────────────────────────────────────────────┘
```

Field ordering within collapsible: status fields first (most commonly edited), then counts/points, then purchase metadata, then storage/image/notes.

### Faction Color Picker (CONTEXT.md locked decision)

- `<input type="color">` native browser control
- Live preview: colored circle (`width: 24px; height: 24px; border-radius: 50%`) displayed inline next to the color input, updates on `onChange`
- Value stored and displayed as hex string (e.g. `#4A90D9`)
- Color accent on faction list: `style={{ borderLeftColor: faction.color_theme }}` with `border-l-[4px]` Tailwind class
- The accent is on the list row container, not a badge

### Delete Confirm Flow (POLISH-01)

1. User clicks "Delete" row action button
2. `Dialog` opens with title, body naming the entity, and two buttons: "Cancel" (ghost) + "Delete" (destructive)
3. On confirm: mutation fires → Dialog closes → if FK error: Sonner toast with specific message; if success: Sonner toast "Deleted." (optional; keep silent if UX feels cleaner)
4. The Dialog closes after the user clicks either button regardless of outcome; FK error surfaces as a toast, not by re-opening the dialog

### Loading States (POLISH-02)

- Lists (factions, paints): show 3 `Skeleton` rows of the same height as real rows while `isLoading` is true
- Skeleton row height: 40px (matches `h-10` table row)
- No spinner overlays; skeleton rows only

### Error States

- Mutation errors: Sonner toast, error/destructive variant, duration 5000ms — CONTEXT.md §Claude's Discretion
- FK constraint violations: Sonner toast with entity-specific message (see Copywriting Contract above)
- Query errors (list fetch fails): inline error message below the page title: "Failed to load [entities]. Try refreshing." — no full-page error state in Phase 2

## Page Layouts

### Faction Page (`/factions`)

```
Page container: flex flex-col gap-6 p-6
  ├── Header row: flex items-center justify-between
  │     ├── <h1> "Factions" (text-xl font-semibold)
  │     └── <Button> "Add Faction" (default variant)
  ├── [Loading]: 3 skeleton rows

  └── [Data]: Table or Card list of faction rows
```

```
Page container: flex flex-col gap-6 p-6
```
