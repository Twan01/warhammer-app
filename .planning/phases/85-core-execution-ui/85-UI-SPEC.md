## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-text gaps in metadata row, inline element pairs |
| sm | 8px | Step list item internal padding, badge margins |
| md | 16px | Section navigator item padding, step content gaps |
| lg | 24px | Panel internal padding (`p-6`), section group spacing |

Exceptions: Section navigator width fixed at 280px. Paint swatch circle at 40px diameter (not on 8pt grid, but matches existing pattern scale-up from 14px timeline dots).

## Typography

This phase uses "larger typography" (PX-01, D-02) for desk-distance readability. Sizes are scaled up from the app's standard `text-sm` (14px) baseline.

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base` |
| Label | 14px | 400 (regular) | 1.4 | `text-sm` |
| Heading | 20px | 600 (semibold) | 1.3 | `text-xl font-semibold` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |

Two weights only: 400 (regular) for Body and Label, 600 (semibold) for Heading and Display. Labels are differentiated from body text by size (14px vs 16px), not weight.

Application:
- **Display (28px)**: Step name/instruction text -- the primary heading of the focal view (D-06)
- **Heading (20px)**: Section names in the navigator, paint name in focal view
- **Body (16px)**: Technique, tool, dilution values in metadata row, step notes
- **Label (14px)**: Position indicator ("Step 3 of 7"), section step counts, badges, metadata icons

## Color

All colors use existing shadcn CSS custom properties. No new color tokens introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` | Full-page background, focal view canvas |
| Secondary (30%) | `hsl(var(--card))` | Section navigator panel background, step card surface, metadata row container |
| Accent (10%) | `hsl(var(--primary))` | Mark Done button fill, current section highlight border-left, active step indicator |
| Destructive | `hsl(var(--destructive))` | Missing paint warning icon and text only |

Accent reserved for:
- "Mark Done" primary CTA button (`bg-primary text-primary-foreground`)
- Current section left-border highlight in navigator (3px left border, `border-primary`)
- Active/current step dot indicator in navigator
- Position indicator step number

Additional semantic colors:
- `text-green-500`: Completed step checkmark, paint-owned indicator dot
- `text-amber-500`: Paint readiness banner background tint (`bg-amber-500/10`)
- `text-red-500`: Missing paint dot indicator (existing `RecipeStepTimeline` pattern)
- `text-muted-foreground`: All secondary/supporting text, disabled nav buttons

## Layout Contract

### Split-Panel Architecture (D-01, D-02)

```
+------------------------------------------------------------------+
| PaintReadinessBanner (conditional, dismissible)                   |
+------------------------------------------------------------------+
| SectionNavigator (280px) |  StepFocalView (flex-1)               |
| fixed width, scroll-y    |                                       |
|                           |  [Step Name - Display 28px]           |
| Section: Basecoat  3/5   |  [Paint Swatch 40px + Paint Name]     |
|   > Step 1 (check)       |  [Metadata Row: technique|tool|...]   |
|   > Step 2 (check)       |  [Reference Photo - large]            |
|   > Step 3 (current)     |  [Notes - Body 16px]                  |
|   > Step 4               |                                       |
|   > Step 5               |  [Prev] Step 3 of 7 - Basecoat [Next]|
| Section: Wash  0/2       |  [========= Mark Done =========]      |
|   > Step 6               |                                       |
|   > Step 7               |                                       |
+------------------------------------------------------------------+
```

- Root: `flex flex-col h-full` (mirrors GameDayPage pattern)
- Banner: conditional render at top, `px-6 py-3`
- Content: `flex flex-1 overflow-hidden`
- Navigator: `w-[280px] border-r border-border overflow-y-auto p-4`
- Focal view: `flex-1 overflow-y-auto p-6`
- No app sidebar visible (PX-01 -- Phase 86 handles route-level layout isolation)

### Step Focal View Layout (D-03 through D-06)

Top-to-bottom vertical stack within the focal view:

1. **Step name** (Display 28px, semibold) -- primary heading
2. **Paint info block**: 40px swatch circle (CSS `backgroundColor: paint.hex_color`) + paint name (Heading 20px) + brand/type (Label 14px, muted). Collapses entirely for paintless steps (D-16).
3. **Metadata row** (D-04): horizontal flex with `gap-4`, each item is icon (16px) + text (Body 16px). Fields: technique (Badge outline), tool (Paintbrush icon), dilution (Droplets icon), time (Clock icon + "~Xm").
4. **Reference photo** (D-05): `max-w-full rounded-lg` image, collapses when absent. Max height 320px with `object-contain`.
5. **Step notes**: Body 16px, `text-muted-foreground`, only when notes exist.
6. **Navigation bar**: `flex items-center justify-between mt-auto pt-6 border-t border-border`. Previous button (ghost, ChevronLeft icon, `aria-label="Previous step"`), position text (Label 14px, "Step X of Y . SectionName"), Next button (ghost, ChevronRight icon, `aria-label="Next step"`). Buttons disable at boundaries.
7. **Mark Done button**: Full-width `Button` primary variant, Heading 20px weight, `h-12` height. Text: "Mark Done" or "Done" with Check icon.

### Section Navigator Layout (D-07 through D-10)

- Each section: `Collapsible` defaultOpen for current section, closed for others
- Section header: `flex items-center justify-between px-3 py-2`, section name (Label 14px), count badge ("3/5", Badge secondary variant)
- Current section: `border-l-3 border-primary bg-accent/50`
- Optional sections: additional "Optional" Badge (outline variant, `text-xs`)
- Step sub-items: `pl-6 py-1.5 flex items-center gap-2 cursor-pointer hover:bg-accent/30 rounded`
  - Completed: green Check circle icon (16px) + step name (Label 14px, `text-muted-foreground line-through`)
  - Current: filled primary dot (8px) + step name (Label 14px, `font-semibold`)
  - Pending: hollow muted dot (8px) + step name (Label 14px)
- Unsectioned steps: grouped under "General" virtual section at bottom (D-10)

### Paint Readiness Banner (D-15, D-16)

- `bg-amber-500/10 border border-amber-500/20 rounded-lg mx-6 mt-4 px-4 py-3`
- Icon: `AlertTriangle` (amber-500, 20px) + "Some paints are not in your inventory:" (Body 16px)
- List: comma-separated paint names in `font-semibold`
- Dismiss: ghost Button with X icon, right-aligned, `aria-label="Dismiss banner"`
- State: `useState(true)` local dismissed state, shown only when `missingPaints.length > 0`

### Loading State

- Full-height `flex flex-col gap-4 p-6`
- `Skeleton` blocks: one `h-8 w-48` (header), one `h-64 w-full` (focal view), one `h-40 w-[280px]` (navigator)
- Shown while `usePaintingModeState.isLoading === true`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Mark Done" (with Check icon prefix) |
| Empty state heading | "No steps in this recipe" |
| Empty state body | "This recipe has no steps to execute. Go back and add steps to the recipe first." |
| Error state | "Could not load painting steps. Check your recipe assignment and try again." |
| Paint readiness banner | "Some paints are not in your inventory:" followed by missing paint list |
| Paintless step label | "(no paint)" displayed in muted text where paint info would appear |
| All complete state | "All steps complete!" with CheckCircle icon, displayed in focal view when last step is done |
| Position indicator | "Step {X} of {Y} . {SectionName}" (dot separator, section name omitted for unsectioned) |
| General section label | "General" (for unsectioned steps group, D-10) |
| Optional badge | "Optional" (on optional section headers, D-09) |

No destructive actions in this phase. Mark Done is additive only (step completion is not reversible in painting mode context).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click "Mark Done" | Calls `useCompleteStep` mutation, advances to next step via `goNext()`. If last step, shows completion state. |
| Click Previous/Next | Calls `goPrev()`/`goNext()` from hook. Buttons disabled at boundaries (first/last step). |
| Click step in navigator | Calls `goToStep(stepId)`. Navigator and focal view update immediately. |
| Click section in navigator | Expands/collapses section via Collapsible. Clicking section header toggles expansion. |
| Dismiss readiness banner | Sets local `dismissed` state to true. Banner hidden for remainder of session. |

Note: Keyboard shortcuts (Space, Arrow keys, Escape) are Phase 86 scope. This phase is mouse/click only.

## Data Flow Contract

```
PaintingModeView (assignmentId, recipeId)
  |-- usePaintingModeState(assignmentId, recipeId)
  |     returns: orderedSteps, currentStepId, currentIndex, completedSet,
  |              canGoPrev, canGoNext, goPrev, goNext, goToStep,
  |              sectionProgressMap, isLoading
  |
  |-- useCompleteStep(assignmentId)
  |     mutation: completeStepWithSession(assignmentId, stepId)
  |     invalidates: step progress, kanban, dashboard, workflow keys
  |
  |-- useRecipePaints(recipeId) -- for paint objects (hex, brand, owned)
  |-- useRecipeSections(recipeId) -- for section metadata (optional flag)
  |
  +-- Props down to children:
      SectionNavigator: sections, sectionProgressMap, orderedSteps,
                        completedSet, currentStepId, goToStep
      StepFocalView: currentStep, paint, completedSet, onMarkDone,
                     goPrev, goNext, canGoPrev, canGoNext,
                     currentIndex, totalSteps, sectionName
      PaintReadinessBanner: missingPaints (derived from paint owned status)
```
