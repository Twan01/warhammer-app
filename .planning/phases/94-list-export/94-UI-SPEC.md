## Color

Project uses shadcn CSS variable tokens (dark theme active at runtime). All values reference existing tokens — no new colors introduced in this phase.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` — zinc ~3.9% lightness in dark | App shell, Sheet background, dialog backdrop |
| Secondary (30%) | `hsl(var(--card))` — zinc ~5.9% lightness in dark | PrintPreviewDialog card surface, dropdown menu surface |
| Accent (10%) | `var(--faction-accent)` — runtime zinc-500 default | Export DropdownMenu trigger button icon highlight, "Warlord" badge ring |

Accent reserved for: Export button icon, Warlord badge ring. Not applied to general dropdown items, unit rows, or section headers.

- All backgrounds: white (#ffffff) — force with `print-color-adjust: exact`
- All text: black (#000000)
- Section separators: 1px solid #cccccc
- Ghost/planned unit name: italic, no color — differentiated by italic style only
- Warlord marker: bold text "(Warlord)" suffix — no color dependency

## Interaction Contract

### Export DropdownMenu (ArmyListDetailSheet header)

- Trigger: Button with `Download` icon + label "Export" — variant `outline`, size `sm`
- Placement: Right side of the ArmyListDetailSheet header area, adjacent to the summary bar
- Dropdown items (in order):
  1. `Clipboard` icon + "Copy to Clipboard" — calls writeText via plugin-clipboard-manager; shows toast on success
  2. `Printer` icon + "Print" — opens PrintPreviewDialog (sibling portal state lifted to ArmyListsPage)
  3. `FileJson` icon + "Save as JSON" — calls Tauri save dialog with .json filter; shows toast on success
  4. `FileText` icon + "Save as PDF" — lazy-loads jsPDF, shows inline loading state on button, calls Tauri save dialog on ready; shows toast on success
- The Export button is always visible regardless of unit count (D-02 from CONTEXT.md)
- Exporting an empty list produces valid minimal output in each format — no error, no guard

### Clipboard Copy (EXP-01)

- Interaction: Single click on dropdown item → immediate async call → toast
- Success toast: "List copied to clipboard" (sonner, default duration)
- Error toast: "Failed to copy — check clipboard permissions" (sonner, destructive variant)
- No confirmation required
- No modal, no preview — fire-and-forget

### PrintPreviewDialog (EXP-02)

- Pattern: Dialog (not Sheet) — sibling portal at ArmyListsPage level; state: `printPreviewOpen: boolean`
- Size: `max-w-3xl` dialog, scrollable content area
- Dialog header: "Print Preview" title, army list name subtitle
- Dialog content: Scrollable unit table + enhancements + totals (see Print Layout section below)
- Dialog footer: "Print" button (primary) + "Close" button (outline)
- "Print" button calls `window.print()`
- CSS `@media print` hides: dialog chrome, app shell, all UI outside `.print-content` container
- CSS `@media print` shows: `.print-content` div only, full-width, no scrollbar

### JSON Export (EXP-03)

- Interaction: Click "Save as JSON" → Tauri `save()` dialog opens with `.json` filter
- Default filename: `[list-name-slugified]-[YYYY-MM-DD].json`
- On save confirmed: write file via plugin-fs; show toast "List saved as JSON"
- On dialog cancelled: no toast, no error
- On write error: toast "Failed to save JSON — check file permissions"

### PDF Export (EXP-04)

- Interaction: Click "Save as PDF" → dropdown item shows spinner (jsPDF loading) → Tauri `save()` dialog
- jsPDF loaded via dynamic `import()` on first PDF export click — not at startup
- Loading state: dropdown item disabled + spinner icon while import resolves (typically <500ms)
- Default filename: `[list-name-slugified]-[YYYY-MM-DD].pdf`
- On save confirmed: generate PDF buffer via jsPDF; write via plugin-fs; toast "List saved as PDF"
- On dialog cancelled: no toast, no error
- On generation error: toast "Failed to generate PDF"

## Print Layout Structure

Used by both PrintPreviewDialog (screen) and `@media print` output. Single-column layout.

```
[Army List Name]                               [Points Used]/[Points Limit]
Faction: [Name]  |  Detachment: [Name]  |  Exported: [Date]
────────────────────────────────────────────────────────────
UNITS
────────
[Unit Name]                               [Points]pts
  └ Led by: [Leader Name]                 [Points]pts     ← indented, leader-attached pair
[Ghost Unit Name] (Planned)               [Points]pts     ← italic name, "(Planned)" suffix

ENHANCEMENTS
────────────
[Enhancement Name] — [Unit Name]          [Points]pts

────────────────────────────────────────────────────────────
TOTAL: [Total]pts / [Limit]pts     Enhancement pts: [N]pts
```

Units are listed in insertion order (stable order from DB). Leader-attached pairs: target unit first, leader immediately below indented with "└ Led by:" prefix. This matches Phase 92 visual grouping logic (canonical ref in CONTEXT.md).

Warlord designation: "(Warlord)" bold suffix after unit name on the same line.

## Clipboard Text Format

Plain text, no markdown rendering. Immediately pasteable into Discord/forums.

```
[Army List Name] — [Points Used]/[Points Limit]pts
Faction: [Faction Name]
Detachment: [Detachment Name]

== Units ==
[Unit Name] — [Points]pts
  > Led by: [Leader Name] — [Points]pts
[Ghost Unit Name] [Planned] — [Points]pts
[Warlord Unit Name] (Warlord) — [Points]pts

== Enhancements ==
[Enhancement Name] on [Unit Name] — [Points]pts

Total: [Total]pts / [Limit]pts
```

No trailing whitespace on any line. Empty sections (no enhancements) are omitted entirely. Single blank line between sections.

## PDF Layout

Text-based jsPDF output. No images, no color. Tournament printout style.

| Element | jsPDF setting |
|---------|--------------|
| Page size | A4 |
| Margins | 20mm all sides |
| Army name | fontSize 18, fontStyle bold |
| Metadata line | fontSize 11, fontStyle normal, color #333333 |
| Section header | fontSize 13, fontStyle bold, underline |
| Unit row | fontSize 11, fontStyle normal |
| Leader row | fontSize 11, fontStyle italic, x-indent 8mm |
| Ghost unit | fontSize 11, fontStyle italic |
| Warlord suffix | append " (Warlord)" to unit name — no separate style |
| Totals line | fontSize 12, fontStyle bold |
| Footer | "Generated by HobbyForge" — fontSize 9, color #999999, bottom-right |
| Line spacing | 6mm between unit rows, 10mm between sections |

HobbyForge watermark/footer: Yes — "Generated by HobbyForge" in footer at bottom-right of each page (9pt, muted grey). Provides context when the file is shared externally.

## State Catalogue

| State | Trigger | UI Response |
|-------|---------|-------------|
| Idle | Default | Export button visible in sheet header |
| PDF loading | User clicks "Save as PDF" for first time | Dropdown item shows spinner + "Generating PDF..." text, item disabled |
| PDF ready | jsPDF import resolved | Tauri save dialog opens |
| Print preview open | User clicks "Print" | PrintPreviewDialog opens via sibling portal |
| Print preview printing | User clicks "Print" inside dialog | `window.print()` called; browser print dialog appears |
| Clipboard success | writeText resolves | Sonner toast: "List copied to clipboard" |
| Clipboard error | writeText rejects | Sonner toast (destructive): "Failed to copy — check clipboard permissions" |
| JSON saved | File write succeeds | Sonner toast: "List saved as JSON" |
| PDF saved | File write succeeds | Sonner toast: "List saved as PDF" |
| Save cancelled | User cancels Tauri dialog | No feedback — silent |
