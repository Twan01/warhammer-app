    Step 1: Create src/features/recipes/AssignmentChecklist.tsx.

    Props interface: AssignmentChecklistProps with assignment (RecipeAssignment, never undefined -- parent gates mount) and recipeId (number).

    Data loading: Call useRecipePaints(recipeId) for step definitions, useRecipeSections(recipeId) for sections, useStepProgress(assignment.id) for progress rows.

    Derived state (all useMemo, NO local state for completion):
    - completedSet: Set of order_index values where completed === 1, derived from stepProgressRows
    - stepsBySection: Map grouping steps by section_id (copy SectionedTimeline lines 26-35 pattern)
    - progress: computeAssignmentProgress(steps, stepProgressRows)

    Toggle handler: useToggleStepProgress().mutate({ assignmentId: assignment.id, orderIndex, completed: boolean })

    Render logic -- two branches per D-09 flat recipe fallback:
    - When sections.length > 0: Accordion type="multiple" wrapping AccordionItem per section. AccordionTrigger shows section.name + Badge variant="outline" with "{bucket.completed}/{bucket.total}". AccordionContent renders step checkboxes.
    - When sections.length === 0: Flat ul with li items, each min-h-12 flex items-center gap-2, Checkbox + step_name span.

    Step checkbox row: Checkbox checked={completedSet.has(step.order_index)}, onCheckedChange calls toggleStep. Completed steps get className "line-through text-muted-foreground".

    Top-level progress bar: Progress value={progress.percentage} className="h-2 flex-1" + span showing "{progress.percentage}% complete" with text-xs text-muted-foreground tabular-nums.
