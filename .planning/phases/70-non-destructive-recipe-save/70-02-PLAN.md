<objective>
Replace the DELETE-all + re-INSERT save pattern in RecipeFormSheet.tsx onSubmit with a five-phase three-way diff algorithm that preserves existing section and step database IDs across edits.

Purpose: This is the core deliverable of Phase 70 (REC-02). The DELETE-all pattern destroys all section/step IDs on every edit, which breaks any downstream FK references (e.g., painting session section links). The diff approach only touches rows that actually changed.

Output: Modified RecipeFormSheet.tsx with non-destructive save flow.

This ordering is critical per Pitfall 1 — surviving sections must be in the map before step processing.

    Phase 5 — Step diff (per D-05, D-12):
    5a. Collect ALL surviving step dbIds globally across ALL sections (flatMap all sections' steps, filter non-null dbId into a Set). This global collection prevents Pitfall 3 (cross-section drag deletion).
