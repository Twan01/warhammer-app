## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 16px | 600 | 1.2 |
| Display | 20px | 600 | 1.2 |

Notes:
- Body (14px/400) used for snapshot label text and unit names in compare columns.
- Label (12px/400) used for timestamps, point totals in the history list, and column headers in the compare dialog.
- Heading (16px/600) used for Sheet title ("Snapshot History") and Dialog title ("Compare Snapshots").
- Display (20px/600) not used in primary content — reserved for points delta summary line in compare dialog.
- All sizes via Tailwind utility classes (`text-sm`, `text-xs`, `text-base`, `text-lg`). Weight 400 = `font-normal`, weight 600 = `font-semibold`. Weight 500 (`font-medium`) is NOT used in this phase.

Source: Established across existing army-lists components; no new font sizes introduced.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` = zinc-950 dark | Sheet background, Dialog background |
| Secondary (30%) | `hsl(var(--card))` = zinc-900 dark | Snapshot history list rows, compare columns |
| Accent (10%) | `var(--faction-accent)` = zinc-500 default | Snapshots button in ArmyListDetailSheet header only |
| Destructive | `hsl(var(--destructive))` = red-600 | Delete snapshot button, restore confirmation button |

Accent reserved for:
1. The "Snapshots" button icon/indicator in `ArmyListDetailSheet` header (matches Export dropdown button pattern)
2. Snapshot count badge (if shown) — uses `bg-faction-accent text-white`
3. Nothing else — all other interactive elements use `bg-primary` or `variant="outline"`

Semantic diff colors in `SnapshotCompareDialog` (not accent — separate semantic colors):
- Added units: `text-green-400` on dark / `text-green-600` on light, with `bg-green-950/40` row highlight
- Removed units: `text-red-400` on dark / `text-red-600` on light, with `bg-red-950/40` row highlight
- Unchanged units: `text-muted-foreground`
- Points delta positive: `text-green-400` with "+" prefix
- Points delta negative: `text-red-400` with "−" prefix

Source: `globals.css` CSS variables; diff color pattern inferred from existing badge and status patterns in the codebase.

## Layout Contracts

### SnapshotHistorySheet

- Sheet width: `sm:max-w-md` (matches `EnhancementPickerSheet`, `LeaderAttachmentSheet` pattern)
- Header: `SheetTitle` = "Snapshot History" + `SheetDescription` = army list name
- **Primary focal point: the Save section at the top** — it is the first and most prominent interactive area; the list below is secondary
- Save section (top, above list):
  - Label: "Save current state as snapshot"
  - Row: `Input` (label text, full width) + `Button` ("Save Snapshot") inline
  - Default label value: `"Snapshot — {MMM D, YYYY}"` pre-filled, user-editable
  - `Separator` below save section
- Snapshot list (scrollable via `ScrollArea`):
  - Empty state: centered text "No snapshots yet. Save the current state above."
  - Each row = `div` with `p-2` padding, `border-b border-border`
  - Row layout: left side (label + timestamp + points) / right side (action buttons)
  - Label: `text-sm font-semibold` truncated to single line
  - Timestamp: `text-xs text-muted-foreground` — formatted as "May 22, 2026 at 14:32"
  - Points: `text-xs text-muted-foreground` — formatted as "{N} pts"
  - Auto-save rows: `Badge variant="outline"` label "Auto-save" rendered before the snapshot label
  - Action buttons (right-aligned, icon-only, `size="icon" variant="ghost"`): Compare, Restore, Delete
    - Compare button: `aria-label="Compare snapshots"`, `title="Compare snapshots"`
    - Restore button: `aria-label="Restore this snapshot"`, `title="Restore this snapshot"`
    - Delete button: `aria-label="Delete snapshot"`, `title="Delete snapshot"`
  - Compare button disabled when `<2` snapshots exist in the list; tooltip "Select a second snapshot to compare"
  - When one snapshot is selected for compare: row gets `ring-1 ring-faction-accent` highlight, Compare button becomes "Selected"
- Footer: "Close" `Button variant="outline"`

### SnapshotCompareDialog

- Dialog size: `max-w-3xl` (wider than default, matches `DatasheetBrowserDialog` pattern)
- Header: `DialogTitle` = "Compare Snapshots" + `DialogDescription` = "{Label A} vs {Label B}"
- Points delta banner: full-width `div` with `p-3 bg-muted rounded-md mb-4`
  - Text: "{Label A}: {N} pts → {Label B}: {N} pts · Delta: +{N} pts" (colored per sign)
  - Font: `text-base font-semibold` (Display role)
- Two-column diff table:
  - Column headers: snapshot label + point total (each column)
  - Rows matched by unit display name
  - Added rows (in B, not A): `bg-green-950/40` background, green unit name
  - Removed rows (in A, not B): `bg-red-950/40` background, red unit name with strikethrough
  - Unchanged rows: default background, `text-muted-foreground`
  - Points shown per unit in each column; blank cell where unit is absent
- Footer: `Button variant="outline"` "Close"

### Snapshots Button (ArmyListDetailSheet header)

- Placed alongside `ExportDropdown` in the header action row
- `Button variant="outline" size="sm"` with `History` icon + "Snapshots" label
- Optional count badge: `Badge` variant="secondary" with snapshot count, positioned top-right of button using `relative` wrapper
- Show badge only when count > 0

### Restore Confirmation Dialog

- Uses existing `AlertDialog` pattern (or `Dialog` — matches `ArmyListDeleteDialog` pattern)
- Title: "Restore snapshot?"
- Description: "This will replace your current list with '{label}'. A safety snapshot of your current state will be saved automatically before restoring."
- Footer: `Button variant="outline"` "Cancel" + `Button variant="destructive"` "Restore"

| Element | Copy |
|---------|------|
| Primary CTA | "Save Snapshot" |
| Snapshots button label | "Snapshots" |
| History sheet title | "Snapshot History" |
| History sheet description | "{List Name}" (army list name as subtitle) |
| Default snapshot label | "Snapshot — {MMM D, YYYY}" |
| Save section label | "Save current state as snapshot" |
| Empty state heading | "No snapshots yet" |
| Empty state body | "Save the current state above to start tracking versions of this list." |
| Snapshot count badge | "{N}" (numeric only, no "snapshots" suffix) |
| Auto-save badge label | "Auto-save" |
| Compare dialog title | "Compare Snapshots" |
| Compare dialog description | "{Label A} vs {Label B}" |
| Points delta format | "{Label A}: {N} pts → {Label B}: {N} pts · Delta: +{N} pts" |
| Restore button label (row) | "Restore" (icon-only on mobile, icon+text on desktop) |
| Restore confirmation title | "Restore snapshot?" |
| Restore confirmation body | "This will replace your current list with '{label}'. A safety snapshot of your current state will be saved automatically before restoring." |
| Restore confirm button | "Restore" |
| Delete snapshot — immediate | Deleted via `toast` with undo (no dialog — low-stakes, undoable) |
| Delete toast | "Snapshot deleted." with "Undo" action |
| Save success toast | "Snapshot saved." |
| Restore success toast | "List restored to '{label}'." |
| Error state | "Something went wrong. Please try again." (matches existing app-wide error copy) |
| Loading state | Skeleton rows (no text) |
| Compare button aria-label | "Compare snapshots" |
| Restore button aria-label | "Restore this snapshot" |
| Delete button aria-label | "Delete snapshot" |

Note on delete: CONTEXT.md left delete UX to Claude's Discretion. This contract chooses immediate delete with `sonner` undo toast (no confirmation dialog) because snapshots are lightweight and the auto-save pattern means the user always has recovery options. This avoids dialog fatigue alongside the restore confirmation.
