const missingPaints = useMemo(() => {
  return steps
    .filter((s) => s.paint_id != null && s.paint_id !== 0)
    .map((s) => paintMap.get(s.paint_id))
    .filter((p): p is NonNullable<typeof p> => p !== undefined && isPaintMissing(p));

Note: Use only primary paint_id for missing check — alt_paint_id does NOT affect availability per CONTEXT.md.

4. Deduplicate missingPaints by paint ID (a paint may appear in multiple steps):
```typescript
const uniqueMissingPaints = useMemo(() => {
  const seen = new Set<number>();
  return missingPaints.filter((p) => {
    if (seen.has(p.id)) return false;
    seen.add(p.id);
    return true;
  });
}, [missingPaints]);
```

5. Add the handler:
```typescript
async function handleAddMissingToWishlist() {
  if (uniqueMissingPaints.length === 0) return;

  const existingNames = new Set(wishlistItems.map((w) => w.name));
  const toAdd = uniqueMissingPaints.filter(
    (p) => !existingNames.has(`${p.brand} ${p.name}`)
  );

  if (toAdd.length === 0) {
    toast.info("All missing paints already on wishlist");
    return;
  }

  try {
    for (const paint of toAdd) {
      await createWishlistItem.mutateAsync({
        name: `${paint.brand} ${paint.name}`,
        faction_id: recipe!.faction_id ?? 1,  // faction_id NOT NULL in wishlist_items — use recipe's faction or fallback to 1
        estimated_cost_pence: null,
        notes: `From recipe: ${recipe!.name}`,
      });
    }
    toast.success(`Added ${toAdd.length} paint${toAdd.length !== 1 ? "s" : ""} to wishlist`);
  } catch {
    toast.error("Failed to add paints to wishlist.");
  }
}
```

**IMPORTANT:** `wishlist_items.faction_id` is `INTEGER NOT NULL REFERENCES factions(id) ON DELETE CASCADE` (migration 009). The recipe's `faction_id` can be null. Fallback to `1` assumes faction ID 1 always exists (it is the first faction created by seed data). This is acceptable for a local desktop app. If the user has no factions at all, the FK constraint will fail and the catch block shows a toast error.

A safer alternative: only show the button when `recipe.faction_id` is not null. Implement this guard:
```typescript
const canAddToWishlist = uniqueMissingPaints.length > 0 && recipe?.faction_id != null;
```
Use `canAddToWishlist` to conditionally render the button. This avoids FK violations entirely.
