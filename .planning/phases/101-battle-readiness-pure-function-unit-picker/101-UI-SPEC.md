## UnitPickerDialog Layout Contract

### Header Region (inside DialogHeader)

```
+-------------------------------------------------------+
| Add unit to list                              [X]     |
| Search units in this faction...                       |
|                                                       |
| 450 pts remaining          [x] Fits budget            |
+-------------------------------------------------------+
```

- **DialogTitle**: "Add unit to list" (unchanged)
- **DialogDescription**: Current text (unchanged)
- **Budget line**: Sits below DialogDescription, above CommandInput. Only visible when `pointsLimit` is provided (D-11).
  - Left: `"{remaining} pts remaining"` in `text-sm text-muted-foreground font-semibold`
  - Right: Checkbox + label `"Fits budget"` in `text-sm`
  - When `remaining <= 0`: remaining text uses `text-destructive`
  - Layout: `flex items-center justify-between px-4 pb-2`

### Unit Row (CommandItem)

```
+-------------------------------------------------------+
| Unit Name            120 pts   [StatusBadge] [o][o][o] |
+-------------------------------------------------------+
```

**Or when Battle Ready:**

```
+-------------------------------------------------------+
| Unit Name            120 pts   [Battle Ready]          |
+-------------------------------------------------------+
```

- **Unit name**: `flex-1`, 14px regular (unchanged)
- **Points**: `text-sm text-muted-foreground tabular-nums` right of name, `ml-auto`
  - Format: `"{effective_points} pts"` or `"- pts"` when 0/null
  - When unit exceeds remaining budget and filter is OFF: `text-muted-foreground/50` (dimmed)
- **Readiness indicators** (when NOT battle-ready): inline after points
  - Painting status: reuse `StatusBadge` component (dot + text) at current 14px size -- but truncate to dot-only in picker to save space. Use TIER_DOT_CLASS color mapping.
  - Assembly dot: 6px circle, `bg-emerald-400` if `status_assembly === 1`, else `bg-muted-foreground/40`
  - Basing dot: 6px circle, same logic for `status_basing`
  - Varnish dot: 6px circle, same logic for `status_varnished`
  - Dots container: `flex items-center gap-1`, with `Tooltip` on hover showing "Assembled, Based, Not varnished" (comma-separated state list)
  - Painting dot uses the existing tier color from `PAINTING_STATUS_TIER[status_painting]` via `TIER_DOT_CLASS`
- **Battle Ready badge** (when all 4 pass): replaces individual dots + painting dot with a single `Badge` using `className="bg-emerald-500/15 text-emerald-500 border-emerald-500/25 text-xs"`. Text: "Battle Ready".
- **Category badge**: `Badge variant="secondary"` (unchanged), shown after readiness indicators

### Row Layout Spec

```tsx
<CommandItem className="flex items-center gap-2">
  <span className="flex-1 truncate">{unit.name}</span>
  <span className="text-sm text-muted-foreground tabular-nums shrink-0">
    {points} pts
  </span>
  {battleReady ? (
    <Badge className="bg-emerald-500/15 text-emerald-500 border-emerald-500/25 text-xs shrink-0">
      Battle Ready
    </Badge>
  ) : (
    <Tooltip>
      <TooltipTrigger asChild>
        <span className="flex items-center gap-1 shrink-0">
          <span className={`h-1.5 w-1.5 rounded-full ${paintingDotClass}`} />
          <span className={`h-1.5 w-1.5 rounded-full ${assemblyDotClass}`} />
          <span className={`h-1.5 w-1.5 rounded-full ${basingDotClass}`} />
          <span className={`h-1.5 w-1.5 rounded-full ${varnishDotClass}`} />
        </span>
      </TooltipTrigger>
      <TooltipContent>{tooltipText}</TooltipContent>
    </Tooltip>
  )}
  {unit.category && (
    <Badge variant="secondary" className="shrink-0">{unit.category}</Badge>
  )}
</CommandItem>
```

### Dot Size

- 6px (`h-1.5 w-1.5`) -- compact enough for 4 dots in a row without crowding the CommandItem
- `rounded-full` for circle shape
- `gap-1` (4px) between dots

## Interaction Contract

### Affordability Filter Toggle

| Interaction | Behavior |
|-------------|----------|
| Toggle ON | Hide all units where `effective_points > remaining`. Remaining recalculates after each add. |
| Toggle OFF (default) | Show all units. Units exceeding budget are NOT hidden but points text is dimmed. |
| No budget context | Toggle and budget line are hidden entirely. Picker works as before. |

### Readiness Tooltip

| Trigger | Content |
|---------|---------|
| Hover on dot cluster | 4-line tooltip: "Painting: {status_painting}", "Assembled: Yes/No", "Based: Yes/No", "Varnished: Yes/No" |
| Focus on dot cluster | Same tooltip (accessible via keyboard) |

### Multi-Add (unchanged)

Dialog stays open after unit selection. Toast confirms "Unit added." on success.

| Error state | `"Failed to add unit. Please try again."` (unchanged) |
