## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` | Dialog overlay backdrop, page surface |
| Secondary (30%) | `hsl(var(--card))` | AlertDialog content surface, manifest detail rows |
| Accent (10%) | `hsl(var(--primary))` | "Restore from backup" button (default variant) |
| Destructive | `hsl(var(--destructive))` | "Replace current database" action button, error banner background |

### 2. RestorePreviewDialog

| Property | Value |
|----------|-------|
| Component | `AlertDialog` (controlled via `open` + `onOpenChange`) |
| Max width | `sm:max-w-lg` (default AlertDialogContent) |
| Trigger | Opened programmatically after successful `validate_backup` invoke |

#### Dialog Header

| Element | Copy |
|---------|------|
| Title | "Restore Backup" |
| Description | "Review the backup details below. Restoring will replace your current database. A safety backup will be created automatically." |

#### Manifest Detail Grid

A vertical list of label-value pairs rendered inside the dialog body between header and footer.

| Field | Label | Value Format | Source |
|-------|-------|-------------|--------|
| Backup date | "Created" | Human-readable date via `formatRelativeDate()` + full ISO in parentheses | `manifest.created_at` |
| App version | "App Version" | Semver string, e.g. "0.2.14" | `manifest.app_version` |
| Schema version | "Schema Version" | `{manifest.schema_version}` + compatibility badge (see below) | `manifest.schema_version` |
| Platform | "Platform" | Capitalized string, e.g. "Windows" | `manifest.platform` |
| Database size | "Database Size" | Human-readable bytes (e.g. "2.4 MB") | `manifest.db_size_bytes` |

Layout: Each row is `flex justify-between items-center` with label as `text-sm text-muted-foreground` and value as `text-sm font-medium`. Rows separated by `border-b border-border last:border-0` with `py-2` vertical padding.

#### Schema Compatibility Badge

Rendered inline next to the schema version number.

| State | Badge | Color | Copy |
|-------|-------|-------|------|
| Match | `<Badge variant="secondary">` with green styling | `bg-emerald-500/10 text-emerald-500 border-emerald-500/20` | "Matches current" |
| Older | `<Badge variant="secondary">` with amber styling | `bg-amber-500/10 text-amber-500 border-amber-500/20` | "Older version" |
| Newer | `<Badge variant="destructive">` | Default destructive | "Newer than app" |

#### Schema Compatibility Banner

Rendered between the manifest grid and the dialog footer. Only shown for older or newer schema states.

| State | Element | Classes | Icon | Copy |
|-------|---------|---------|------|------|
| Older (warning) | `<div>` | `rounded-md border border-amber-500/20 bg-amber-500/10 p-3 text-sm text-amber-500` | `AlertTriangle` (h-4 w-4) | "This backup was created with an older version. Some data may need migration after restore." |
| Newer (error) | `<div>` | `rounded-md border border-destructive/20 bg-destructive/10 p-3 text-sm text-destructive` | `ShieldAlert` (h-4 w-4) | "This backup was created with a newer version of HobbyForge. Update the app before restoring." |
| Match | Not rendered | -- | -- | -- |

#### Dialog Footer

| Element | Component | Variant | Disabled When | Copy |

| Cancel | `AlertDialogCancel` | `outline` (default) | Never | "Cancel" |

## Interaction Flow

```
1. User clicks "Restore from Backup" on BackupCard
2. Native file picker opens (filtered to .zip)
3a. User cancels file picker → nothing happens
3b. User selects a .zip file → button enters loading state ("Validating...")
4. invoke("validate_backup", { path }) fires
5a. Validation fails → toast.error("Invalid backup file: {reason}"), loading clears
5b. Validation succeeds → RestorePreviewDialog opens with manifest data
6. User reviews manifest + schema compatibility state
7a. User clicks "Cancel" → dialog closes, no side effects
7b. Schema is newer → "Replace current database" is disabled, user must cancel
7c. User clicks "Replace current database" → toast.info placeholder, dialog closes
```
