<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file selection -> Rust validate_backup | Untrusted zip file path from file picker crosses into Rust backend |
| Rust validate_backup -> Frontend | Parsed BackupManifest data flows from trusted backend to frontend display |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-02 | Tampering | validate_backup input | mitigate | Rust command validates zip structure, checks for exact entry names "hobbyforge.db" and "metadata.json", rejects malformed archives. Frontend does not parse zip. |
| T-81-03 | Spoofing | BackupManifest data | accept | Manifest is self-reported by the backup creator. Schema version comparison catches version mismatches; actual DB integrity check is Phase 82's concern. |
| T-81-04 | Denial of Service | Large zip file | accept | Phase 81 is read-only preview; validate_backup only reads metadata.json (small). Actual extraction is Phase 82. |
| T-81-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
