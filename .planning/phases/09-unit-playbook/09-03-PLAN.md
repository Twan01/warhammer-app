  <how-to-verify>
    Run the app:
    ```
    pnpm tauri dev
    ```

    Wait for the desktop window to launch with the HobbyForge sidebar.

    **Step 1 — STRAT-01: Tabs render in UnitDetailSheet.**
    1. Click "Collection" in the sidebar.
    2. Click any unit row to open UnitDetailSheet.
    3. Confirm two tab triggers are visible: "Details" (active by default, slight background fill) and "Playbook" (inactive).
    4. Confirm `SheetHeader` (unit name as title + faction Badge as description) is visible above the tabs.
    5. Confirm `SheetFooter` (Edit Unit primary button + Delete Unit ghost button with red text) is visible at the bottom of the sheet.

    **Step 2 — STRAT-01: Tab switching without closing the sheet.**
    1. Click the "Playbook" trigger. The Details content (Category, Painting Status, Painting Progress, Assembly, Basing, etc.) is replaced by the Playbook content. The sheet stays open.
    2. Confirm `SheetHeader` (unit name) and `SheetFooter` (Edit/Delete) remain visible.
    3. Click "Details" — Playbook content is replaced by the original Details content. Confirm Linked Recipes, Active Project toggle, Notes block (if any) are all still rendered correctly (no regression on Phase 3 detail layout).
    4. Switch back to "Playbook" — content reappears unchanged.

    **Step 3 — STRAT-02: Stats block display mode and suffix rendering.**
    1. On the Playbook tab, confirm a "Stats" section header at the top with a Pencil icon button to its right (aria-label "Edit stats").
    2. Confirm a horizontal row of 6 cells with labels "M", "T", "Sv", "W", "Ld", "OC" in that exact order.
    3. For a freshly-opened unit with no playbook saved yet, every cell shows "—" (em dash, muted color) below the label.

    **Step 4 — STRAT-02: Stats edit mode + suffix logic.**
    1. Click the Pencil icon. Confirm all 6 cells visually swap to number inputs (border becomes primary/accent color).
    2. Enter values: M=6, T=4, Sv=3, W=2, Ld=7, OC=1. Use Tab to move between cells.
    3. Click the Pencil icon again to exit edit mode.
    4. Confirm display shows: 6", 4, 3+, 2, 7+, 1+ — note that M shows trailing `"`, Sv/Ld/OC show trailing `+`, T/W show raw integers.
    5. Re-enter edit mode and confirm the input shows raw integers (no suffix) — e.g., the M input shows `6`, not `6"`.
    6. Exit edit mode without saving — display still shows the entered values (state preserved across edit-mode toggles).

    **Step 5 — STRAT-03: Abilities textarea and Keywords input.**
    1. Below the stats block (after a Separator), find the "Abilities" label with a multi-line textarea (3 rows tall).
    2. Type "Deep Strike. Stealth." into Abilities — confirm the text appears as you type.
    3. Below Abilities, find the "Keywords" label with a single-line text input.
    4. Type "Infantry, Battleline, Core" into Keywords — confirm the text appears as you type.

    **Step 6 — STRAT-04: 8 strategy note fields in correct order.**
    1. Below Keywords (after a Separator), confirm 8 fields appear in this exact top-to-bottom order:
       1. Battlefield Role
       2. Strengths
       3. Weaknesses
       4. Best Targets
       5. Synergies
       6. Mistakes to Avoid
       7. Rules Page References
       8. Personal Notes
    2. Each field has a small uppercase muted label and a 2-row textarea below it.
    3. Type a short note into 2 or 3 of the fields — confirm typing is reflected immediately and there is no validation error or red border.

    **Step 7 — STRAT-05: Save button dirty-state + inline save.**
    1. Scroll to the bottom of the Playbook tab. Confirm a full-width "Save Playbook" button (primary fill).
    2. With no edits since the last save (or since open if no save yet), confirm the button is disabled (greyed out, cursor: not-allowed).
    3. Edit any field — Save Playbook becomes enabled (full color).
    4. Click "Save Playbook". Confirm:
       - A sonner toast appears at the bottom-right reading "Playbook saved".
       - The Save button immediately becomes disabled again (isDirty resets).
       - The sheet stays open (no auto-close).
    5. Edit another field — Save re-enables. Click Save again — toast fires again.

    **Step 8 — Persistence (Tauri IPC live test).**
    1. Click outside the sheet (or press Escape) to close it.
    2. Re-open the same unit by clicking its row. Confirm the Playbook tab now shows the previously-saved values (stats, Abilities, Keywords, all 8 strategy notes).
    3. Switch to a DIFFERENT unit. Confirm its Playbook tab is empty (or shows that unit's own previously-saved data, not bleeding through from unit A).
    4. Switch back to the FIRST unit. Confirm its data is still there.

    **Step 9 — SheetFooter Edit/Delete on both tabs.**
    1. On the Playbook tab, click "Edit Unit" — confirm the Unit edit Sheet opens (Phase 3 component) WITHOUT touching the playbook data. Close it.
    2. Switch to Details tab. Click "Edit Unit" — confirm the same Edit Sheet opens.
    3. Switch back to Playbook. Click "Delete Unit" — confirm the Unit delete dialog opens (Phase 3 component). Click "Cancel" — dialog closes, no row removed.

    **Step 10 — Stats block escape (optional).**
    1. Enter stats edit mode. Press Escape. Per UI-SPEC §3, this SHOULD revert to display mode without saving.
    2. If Escape does NOT revert, this is a known UI-SPEC nice-to-have (not a blocker per RESEARCH.md §Open Questions Q1). Note the behavior but do not flag it as a failure unless you specifically want it implemented now.

    If every required step (1–9) passes, type "approved". If any step fails, describe what broke (which step, what you saw, what you expected). The orchestrator will route failures to `/gsd:plan-phase --gaps` for closure.
