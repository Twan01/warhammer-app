## Interaction Contracts

### 1. Tabs

- Default active tab on sheet open: "Details" (`defaultValue="details"`)
- Switching tabs does NOT close or remount the sheet
- `SheetHeader` (unit name, faction badge) and `SheetFooter` (Edit Unit, Delete Unit) remain visible regardless of active tab
- Tabs use the default variant (pill/rounded background on active trigger — `bg-muted` list, `data-[state=active]:bg-background` trigger)
- Tab list is placed between `SheetHeader` and the scrollable content `div` — at the top of the scrollable zone

Source: CONTEXT.md (Tabs integration decisions), `tabs.tsx`

### 2. Stats Block — Display Mode

- Six cells in a single `flex flex-row` container spanning the full width of the sheet content
- Each cell: `flex-1 flex flex-col items-center justify-center min-h-[44px] border border-border rounded-sm bg-card gap-0.5 px-1 py-2`
- Top line: stat abbreviation — `text-[10px] font-semibold text-muted-foreground uppercase` — exactly: M, T, Sv, W, Ld, OC
- Bottom line: value with suffix — `text-base font-semibold text-foreground`
- Suffix logic (display only, never stored):
  - M → append `"` (e.g. `6"`)
  - Sv, Ld, OC → append `+` (e.g. `3+`)
  - T, W → raw integer (e.g. `5`)
- Empty value: render `—` in `text-muted-foreground`
- Edit trigger: small "Edit" icon button (`Pencil` icon from lucide-react, `variant="ghost" size="icon"` — `h-6 w-6`) placed to the right of the stats row heading or as a floating top-right icon on the stats block

Source: CONTEXT.md (stats block visual decisions, suffix logic), REQUIREMENTS.md STRAT-02

### 3. Stats Block — Edit Mode

- Clicking the "Edit" icon button switches all six cells to `<Input type="number" min={0}>` simultaneously
- In edit mode: each cell shows the raw integer input, no suffix; cell border changes to `border-primary`
- "Save Stats" replaces the "Edit" icon button — a small `Button variant="ghost" size="sm"` labeled "Save" inline next to the stats row, or the global "Save Playbook" button at the bottom covers it
- Cancelling edit mode (Escape or clicking outside) reverts cells to display mode without saving
- Saving: inline save is handled by the global "Save Playbook" button — edit mode stays open until Save is clicked

Clarification on approach (Claude's Discretion resolved): Edit mode is triggered by a dedicated small `Pencil` icon button to the right of the "Stats" section label — not by clicking any cell. This avoids accidental edits and is more discoverable on the compact sheet width.

Source: CONTEXT.md (Claude's Discretion — edit trigger)

### 4. Abilities and Keywords Fields

- Abilities: `<Textarea rows={3} placeholder="Enter unit abilities…" className="resize-none" />`  — uses shadcn Textarea (native `<textarea>` styled with the existing inline class pattern from `PaintSheet.tsx`)
- Keywords: `<Input type="text" placeholder="e.g. Infantry, Battleline, Core" />` — single-line, comma-separated; no tag chip UI

Source: CONTEXT.md (Playbook tab content order, Claude's Discretion for Keywords input type)

### 5. Strategy Notes Fields (8 fields)

Ordered exactly as specified in STRAT-04:
1. Battlefield Role
2. Strengths
3. Weaknesses
4. Best Targets
5. Synergies
6. Mistakes to Avoid
7. Rules Page References
8. Personal Notes

Each field rendered as:
```
<div className="flex flex-col gap-1">
  <Label className="text-xs font-semibold text-muted-foreground uppercase tracking-wide">
    {fieldName}
  </Label>
  <Textarea rows={2} placeholder="…" className="resize-none" />
</div>
```

The 8 fields are stacked in a single `flex flex-col gap-4` container with no grouping, collapsibles, or grid. Single-column layout only — fits `sm:max-w-md` sheet width.

Source: CONTEXT.md (strategy notes layout decision), REQUIREMENTS.md STRAT-04

### 6. Dirty State and Save Button

- "Save Playbook" button: `Button variant="default"` — full-width at the bottom of the Playbook tab scroll area (`w-full mt-4`)
- Disabled when: no field has changed since last load or last save (`isDirty` flag tracked in component state)
- Enabled as soon as any input is changed
- On success: `toast.success("Playbook saved")` via sonner — sheet remains open
- On error: `toast.error("Failed to save playbook")` — sheet remains open, button re-enables
- No separate edit/view mode toggle for strategy notes — textareas are always editable

Source: CONTEXT.md (Save button placement decision, toast copy)

### 7. Data Loading

- `useStrategyNote(unitId)` called eagerly when the sheet opens (`key={unit?.id ?? "none"}` on `SheetContent` handles reset on unit switch)
- Loading state: render the Playbook tab content area with `opacity-50 pointer-events-none` while `isLoading` is true (skeleton not required — load is instantaneous on local SQLite)
- If no strategy note exists yet (null result): all fields empty, Save creates a new record via upsert

Source: CONTEXT.md (data loading decision), `UnitDetailSheet.tsx` key pattern
