## Animation Contract

Phase 11 introduces count-up animation. These are visual contracts, not spacing/color — specified here for executor clarity.

| Property | Value | Source |
|----------|-------|--------|
| Hook | `useCountUp(target: number, duration?: number): number` | CONTEXT.md decision |
| Location | `src/hooks/useCountUp.ts` | CONTEXT.md decision |
| Duration | 600ms | CONTEXT.md decision |
| Easing | Cubic ease-out: `1 - Math.pow(1 - progress, 3)` | CONTEXT.md (Claude's discretion) |
| Output type | Integer only — `Math.round(eased * target)` — no decimals mid-animation | CONTEXT.md decision |
| Optional delay param | Accept `delay?: number` (ms) for staggered starts — default 0 | Claude's discretion |
| Trigger | Every Dashboard mount (TanStack Router unmounts on navigate) + every `dashboard-stats` query refetch | CONTEXT.md decision |
| Scope | 4 top hero StatCards only: Total Models, Fully Painted, Battle-Ready Points, Active Projects | CONTEXT.md decision |
| Excluded | Progress section 3 cards (Painting %, Assembly %, Basing %) — static, no animation | CONTEXT.md decision |

**AnimatedStatCard pattern:** `StatCard` gains an `animate?: boolean` prop. When `animate={true}`, it calls `useCountUp(value as number)` internally and renders the animated integer. When `animate={false}` or `value` is a string (e.g. `"72%"`), it renders `value` as-is. This avoids a separate wrapper component.

**Skeleton branch is unaffected:** The loading skeleton (`<Skeleton className="h-24 w-full" />`) remains unchanged — animation only activates in the populated data branch.

## Interaction States

| State | Component | Behavior |
|-------|-----------|----------|
| Loading | Top hero row | 4× `<Skeleton className="h-24 w-full" />` in `grid-cols-4 gap-6` — unchanged |
| Loaded, animating | Hero StatCard (4×) | Counter ticks from 0 to target over 600ms with cubic ease-out; integer values only |
| Loaded, settled | Hero StatCard (4×) | Displays final integer value; no idle animation |
| Data refetch | Hero StatCard (4×) | Counter re-animates from 0 to new target value when `dashboard-stats` query returns new data |
| Navigate away + return | Hero StatCard (4×) | Counter re-animates from 0 on every Dashboard mount |
| Active faction set | FactionSummaryCard | `ring-2 ring-faction-accent` + filled Star icon in faction-accent color |
| No active faction | FactionSummaryCard | No ring, unfilled Star in muted-foreground |
| Hover | FactionSummaryCard | `hover:bg-muted/50` (existing, unchanged) |
| Error | DashboardPage | Existing error message — no changes |
