Note: read migration 022 carefully to confirm the exact table name and column names after the rebuild. The table is recipe_step_paints (created in migration 013, rebuilt in 022 to make paint_id nullable).

    Test "non-destructive save preserves IDs" (D-08): Create a recipe, create a section, INSERT two recipe_step_paints rows. Record their IDs. UPDATE one row's step_label to a new value. UPDATE the section's name. SELECT both step rows back and the section — assert all IDs match the original IDs (same lastInsertRowid values). This validates the Phase 70 non-destructive pattern: UPDATE in place rather than DELETE + re-INSERT.

SELECT recipe_step_paints WHERE section_id = deleted section's id — assert zero rows returned. This validates ON DELETE CASCADE on recipe_step_paints.section_id FK (set in migration 018 or the table definition).

    Important: Use `?` parameter syntax for better-sqlite3, NOT `$1, $2`. Use db.prepare(sql).run(...args) for INSERTs and db.prepare(sql).all(...args) or .get(...args) for SELECTs.
