## Spacing Scale

Project-standard 8-point scale. All values confirmed in existing card and layout patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), dot-to-text in StatusBadge |
| sm | 8px | Inline element spacing, card interior row gaps (`gap-2`) |
| md | 16px | Default element spacing, card horizontal padding (`px-4`) |
| lg | 24px | Section padding, card group gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Gallery card top padding removed (photo hero flush to card top edge, `rounded-t-lg` matches card border)
- Gallery card body padding: `px-3 pb-3 pt-2` (12px sides, 12px bottom, 8px top ‚Äî tighter because image already provides visual breathing room)
- Kanban card padding: `p-3` (12px all sides ‚Äî unchanged from current)
- Kanban metadata row top margin: `mt-2` (8px ‚Äî sm ‚Äî standard small gap between progress bar and metadata)
- Kanban next-action hint top margin: `mt-1` (4px ‚Äî xs ‚Äî tight coupling to metadata row above it)
- Progress bar height: 2px (`h-0.5`) ‚Äî sub-grid, decorative only
- StatusBadge dot: 4px √ó 4px (`h-1 w-1`) ‚Äî sub-grid, visual indicator only
- Log Session button hit target: 24px √ó 24px (`h-6 w-6`) ‚Äî below 44px minimum because it is always adjacent to the 3-dot actions menu which provides an expanded clickable area

## Responsive Behavior

### Gallery Grid

Unchanged from Phase 12 locked decision:
```
grid-cols-2           (< 768px)
md:grid-cols-3        (768px ‚Äì 1023px)
lg:grid-cols-4        (1024px+)
gap-4                 (16px ‚Äî consistent across breakpoints)
```

Photo thumbnails scale naturally: `w-full aspect-square` means the photo is always square and fills the card width at every breakpoint.

### Kanban Board

Unchanged ‚Äî column widths and board scroll behavior are not modified in Phase 28.

Kanban card width is fixed by its column. The metadata row uses `flex-wrap` so all three metadata items (last updated, recipe, photo count) wrap gracefully if the card is narrow. `gap-x-3` maintains visual separation; `gap-y-1` keeps the wrapped line tight.

### Minimum Card Width

Gallery card minimum effective width: `(viewport - sidebar - gaps) / 2` at the smallest breakpoint. At 900px minimum window width (from `tauri.conf.json`), with sidebar at ~220px: `(900 - 220 - 16) / 2 = 332px`. Photo area square at 332px is fully legible.

## Accessibility Notes

### Gallery Cards

- `role="button"` and `tabIndex={0}` on the `<Card>` ‚Äî unchanged, already implemented
- `aria-label={unit.name}` ‚Äî unchanged
- Photo `<img>` requires `alt={`${unit.name} photo`}`
- Placeholder div is decorative; initials text is aria-hidden (the card's `aria-label` already names the unit): add `aria-hidden="true"` to the initials `<span>`
- Progress bar does not need `aria-valuenow` ‚Äî the `{painting_percentage}%` text below it provides the same information in text form

### Kanban Cards

- Log Session button requires `aria-label={`Log session for ${unit.name}`}` ‚Äî prescribed above
- Camera icon: `aria-hidden="true"` on the `<Camera>` element; the adjacent text provides the count; wrap in `<span aria-label="${photoCount} photos">` for screen readers
- Next-action hint `<p>` is informational ‚Äî no interactive role needed; screen readers will read it as paragraph text

### StatusBadge in Table

- Popover trigger button requires `aria-label` that describes both current state and affordance: `aria-label={`Change status: currently ${unit.status_painting}`}`
- StatusBadge dot: `aria-hidden="true"` on the dot `<span>` ‚Äî the text label beside it is sufficient

### Focus Management

- Log Session button: standard focus ring via `focus-visible:ring-1 focus-visible:ring-ring`
- StatusBadge trigger: `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`
- LogSessionSheet: on open, focus moves to first interactive element in the Sheet (existing Sheet behavior from shadcn)
