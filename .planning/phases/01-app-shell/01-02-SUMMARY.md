## Runtime Database Location

```
%APPDATA%\com.hobbyforge.app\hobbyforge.db
```

The connection string `sqlite:hobbyforge.db` (no absolute path) causes tauri-plugin-sql to resolve via `app_data_dir()` automatically. This is by design — do NOT use an absolute path in the connection string.

## lib.rs Setup() Order of Operations

```
setup() entry
  └─ app.path().app_data_dir()            → resolves %APPDATA%\com.hobbyforge.app\
  └─ std::fs::create_dir_all(&app_data_dir) → creates directory if missing (SETUP-07)
  └─ println! diagnostic                   → visible in dev console
.plugin(tauri_plugin_opener::init())      → opener plugin
.plugin(tauri_plugin_sql::Builder::default()
        .add_migrations("sqlite:hobbyforge.db", get_migrations())
        .build())                          → SQL plugin registered AFTER dir exists
```

This order is critical. If the SQL plugin tried to open hobbyforge.db before the directory existed, it would fail on first launch of a production binary (development restores from previous runs).
