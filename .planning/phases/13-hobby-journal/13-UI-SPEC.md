| Token | Value | Usage |

## Interaction Contracts

### Journal Tab — Session Log Section

**Inline form (always visible at top):**
- Date `<Input type="date">` — defaults to today's date (`new Date().toISOString().split("T")[0]`) on mount AND after each successful submit. Never clears without submit.
- Duration `<Input type="number" min={1} placeholder="e.g. 45">` — always in minutes; no unit suffix in field (suffix "min" rendered as static text to the right of the input).
- Notes `<textarea>` — same class as PlaybookTab `TEXTAREA_CLASS`; `rows={3}`; placeholder "Optional notes about this session…"
- "Log Session" `<Button variant="default">` — disabled when Date or Duration is empty. On click: INSERT row, invalidate query, reset all three fields to defaults, keep form open.
- Form layout: vertical stack, `gap-4`, no horizontal split — Sheet is narrow (448px); horizontal layout would be cramped.

**Sessions list (below form):**
- Sorted newest first (ORDER BY session_date DESC, id DESC).
- Each row: date left-aligned (text-sm, muted-foreground), duration right-aligned (text-sm "45 min"), notes below (text-sm, full width if present).
- Row background: `bg-card` with `rounded-md` and `p-3`.
- Delete button: ghost icon button (`<Button variant="ghost" size="icon">`), Lucide `Trash2` icon, `text-destructive` color, right side of row. On click: **immediate delete with optimistic rollback** — no confirmation dialog. Rollback on failure with `toast.error("Failed to delete session — changes were not saved.")`.
- Empty state: centered text at list position (not full-screen empty state), "No sessions logged yet." (muted-foreground, text-sm).

### Journal Tab — Photo Timeline Section

**Attach photo form:**
- "Attach Photo" `<Button variant="outline">` — triggers native file picker via `tauri-plugin-fs` open dialog. Filters: image/* (jpg, png, webp, gif).
- Stage label `<Select>` — options: Primed / Base coat / Washed / Layer / Highlighted / Finished / Other. Default: no selection (placeholder "Select a stage").
- When "Other" selected: `<Input type="text" placeholder="Describe this stage…">` appears below the Select with `mt-2` spacing. Required when "Other" is active.
- Caption `<Input type="text" placeholder="Optional caption…">` — always visible below stage controls.
- "Save Photo" `<Button variant="default">` — disabled until: (a) file selected, (b) stage selected (and if "Other", custom label non-empty). On click: copy file to appDataDir with UUID filename, INSERT image_assets row, invalidate query, reset attach form.
- Form layout: vertical stack within a `bg-card rounded-md p-3` container.

**Thumbnail grid:**
- CSS grid: `grid grid-cols-3 gap-2`.
- Each cell: fixed height 96px (h-24), full width of cell.
- Thumbnail `<img>`: `object-cover w-full h-full rounded-sm`. Loading: `<Skeleton className="w-full h-24 rounded-sm">`.
- Stage label: `text-xs text-muted-foreground` centered below image, single line, truncated with `truncate` if overflow.
- Caption display: hover `<Tooltip>` showing full caption text. Caption is NOT shown as visible text in the grid to avoid cluttering the 3-column layout. If no caption, tooltip not rendered.
- Hover state: thumbnail cell gets a semi-transparent overlay (`bg-black/40`) and the × delete button appears centered or top-right.
- Delete (×) button: `<Button variant="ghost" size="icon" className="absolute top-1 right-1 h-6 w-6 text-white bg-black/50 hover:bg-destructive">` with `Lucide X` icon (h-3.5 w-3.5). Cell wrapper must have `relative` and `group` class for CSS hover targeting.
- Photo deletion: **immediate delete with brief undo window** — delete file from disk + DELETE image_assets row, then show `toast("Photo removed", { action: { label: "Undo", onClick: undoFn } })` with 4-second duration. Undo re-copies the temp-saved file and re-inserts the row. If undo window expires, file is permanently removed.

Source for caption/delete/undo decisions: CONTEXT.md §Claude's Discretion delegated these choices to researcher.

**Full-size lightbox:**
- Sibling `<Dialog>` mounted at CollectionPage level (same level as `<UnitDetailSheet>`), following the established sibling portal pattern.
- `<DialogContent className="max-w-2xl">` — wide enough for full-size photo viewing.
- Shows `<img>` with `max-h-[70vh] w-auto mx-auto object-contain`.
- `<DialogHeader>` shows stage label as `<DialogTitle>` and caption as `<DialogDescription>` (empty string if no caption, hiding the element).
- Close via Dialog's built-in × button or Escape key.
- Previous/Next navigation: not in scope for Phase 13 (JOUR-08 deferred).

## Layout Contract

### JournalTab internal structure (top to bottom, inside `overflow-y-auto` SheetContent)

```
<div className="flex flex-col gap-6 p-4">
  {/* Section: Sessions */}
  <section>
    <h3 className="text-base font-semibold mb-3">Sessions</h3>

    {/* Inline log form — always visible */}
    <div className="flex flex-col gap-4 bg-card rounded-md p-3 mb-4">
      <Label + Input[type=date]>
      <Label + Input[type=number] + "min" suffix>
      <Label + textarea[rows=3]>
      <Button "Log Session">
    </div>

    {/* Session rows or empty state */}
    <div className="flex flex-col gap-2">
      {sessions.map(session => <SessionRow />)}
      {sessions.length === 0 && <EmptyState "No sessions logged yet." />}
    </div>
  </section>

  <Separator />

  {/* Section: Photos */}
  <section>
    <h3 className="text-base font-semibold mb-3">Photos</h3>

    {/* Attach photo form */}
    <div className="flex flex-col gap-3 bg-card rounded-md p-3 mb-4">
      <Button "Attach Photo" variant="outline">
      <Label + Select[stage]>
      {stage === "Other" && <Input[type=text]>}
      <Label + Input[type=text caption]>
      <Button "Save Photo" variant="default">
    </div>

    {/* 3-column thumbnail grid or empty state */}
    <div className="grid grid-cols-3 gap-2">
      {photos.map(photo => <ThumbnailCell />)}
    </div>
    {photos.length === 0 && <EmptyState "No photos added yet." />}
  </section>
</div>
```

