## Visual Hierarchy

**Focal point:** ApplyRecipeDialog — the "Suggested" group is the primary visual anchor, appearing first in the command palette. RecipeFormSheet focal point is inherited (form fields in existing layout).

## Component Inventory

This phase modifies two existing components. No new components are created.

### RecipeFormSheet (modified)

| Aspect | Contract |
|--------|----------|
| New props | `defaultFactionId?: number \| null`, `defaultUnitId?: number \| null` |
| Pre-fill behavior | When `defaultFactionId` provided, faction Select initializes with that value. When `defaultUnitId` provided, unit Select initializes with that value. |
| Visual treatment | No special visual indicator for pre-filled fields (D-07). Standard Select controls with pre-populated values. |
| Editability | All pre-filled fields are fully editable -- user can clear, change, or override (D-08). |
| Default value timing | Defaults freeze at sheet open time via `useForm({ defaultValues })` -- not reactive to external state changes (STATE.md decision). |
| QuickAdd entry | No pre-fill -- `defaultFactionId` and `defaultUnitId` remain undefined (D-02). |

### ApplyRecipeDialog (modified)

| Aspect | Contract |
|--------|----------|
| New prop | `factionId?: number \| null` |
| Grouping | When `factionId` is a valid number, recipes split into two `CommandGroup` components: "Suggested" (faction match) on top, "Other" (remaining) below. |
| No faction | When `factionId` is null or undefined, single flat list -- current behavior unchanged (D-06). |
| Group header text | "Suggested" (12px, medium weight, muted foreground). "Other" (same styling). |
| Group header count | Show count in parentheses: "Suggested (3)" / "Other (12)". |
| Search behavior | Search filters across both groups simultaneously. If a group becomes empty after filtering, its header hides automatically (CommandGroup native behavior). |
| Recipe row | Existing layout preserved: recipe name + faction Badge. No changes to individual row rendering. |

### UnitDetailSheet (threading only)

| Aspect | Contract |
|--------|----------|
| Changes | Pass `defaultFactionId={unit.faction_id}` and `defaultUnitId={unit.id}` to RecipeFormSheet when opening recipe creation. Pass `factionId={unit.faction_id}` to ApplyRecipeDialog. |
| Visual changes | None. |

## Interaction States

### Pre-filled Select (RecipeFormSheet)

| State | Behavior |
|-------|----------|
| Initial (with context) | Select shows pre-populated faction/unit name. Indistinguishable from user-selected value. |
| Initial (without context) | Select shows placeholder text ("Select a faction...", "Select a unit..."). Current behavior preserved. |
| User clears | User opens Select and selects a different value or the placeholder/empty option. Form tracks new value normally. |
| Form reset | If form resets (e.g., after successful save), values return to the defaults passed via props. |

### Grouped Command Palette (ApplyRecipeDialog)

| State | Behavior |
|-------|----------|
| Loading | Existing loading state -- no change. |
| Populated with faction | Two groups visible: "Suggested (N)" then "Other (N)". Keyboard navigation flows across both groups seamlessly. |
| Populated without faction | Single flat list -- current behavior. |
| Search active | Both groups filter simultaneously. Empty groups auto-hide. CommandEmpty shows "No recipes found." if all filtered out. |
| Recipe selected | Existing two-step flow preserved: select recipe, preview sections/steps, confirm apply. |
