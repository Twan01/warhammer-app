## Typography

Exactly 2 weights are declared. Size differences (text-xs vs text-sm) create visual hierarchy between table headers and body text without requiring a third weight.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Default prose, muted column text, filter button labels |
| Label | 14px (text-sm) | 400 (regular) | 1.4 | Table headers use text-xs at weight 400; paint name uses text-sm at weight 400 |
| Heading (page title) | 20px (text-xl) | 600 (semibold) | 1.2 | Only role using weight 600 |

Weight mapping:
- **400 regular** ŌĆö Body, Label, table headers (text-xs), recipe count badge text, filter button labels, muted secondary text (brand, type, color-family columns)
- **600 semibold** ŌĆö Page heading ("Paints") only

Notes:
- Paint name in `PaintRow.tsx` previously used `font-medium` (weight 500) ŌĆö change to `font-normal` (weight 400); size at text-sm is sufficient to distinguish it from muted secondary columns
- Recipe count badge text: `text-xs` / weight 400
- Muted secondary text (brand, type, color-family columns): `text-sm text-muted-foreground` at weight 400

Source: `PaintsPage.tsx` uses `text-xl font-semibold` for the page heading. shadcn Table component uses `text-xs` for headers. Body text is `text-sm` throughout the app. Weight collapsed from 3 ŌåÆ 2 per checker requirement.

## Interaction Contracts

### Visual Hierarchy

The filter bar + "Add Paint" CTA row is the primary interactive anchor for the page. Table rows are the secondary content area. Users orient to the filter bar first (action and narrowing), then scan the table (content). This ordering is reinforced by vertical stacking: filter bar above, table below, with `lg` (24px) gap between them.

### Filter Bar

Layout: single horizontal row, `flex items-center gap-2 flex-wrap`

Order: `[ Brand ] [ Type ] [ Color-family ] [ Running Low ] [ Wishlist ] [ Clear (conditional) ]`

| Control | Component | Behavior |
|---------|-----------|----------|
| Brand | `MultiSelectPopover` (same as `UnitFilters.tsx`) | Options derived dynamically from paint list via `useMemo`; sorted alphabetically |
| Type | `MultiSelectPopover` | Options: hardcoded from `paint_type` enum ŌĆö Primer, Base, Layer, Contrast, Shade, Technical, Dry, Air, Metallic, Ink, Oil, Enamel, Pigment, Other |
| Color-family | `MultiSelectPopover` | Options derived dynamically from paint list via `useMemo`; sorted alphabetically; null values excluded |
| Running Low | `Button variant="outline" size="sm"` ŌåÆ `variant="default"` when active | `aria-pressed={runningLow}` |
| Wishlist | `Button variant="outline" size="sm"` ŌåÆ `variant="default"` when active | `aria-pressed={wishlist}` |
| Clear | `Button variant="ghost" size="sm"` with `X` icon | Appears only when `hasAny === true`; positioned `ml-auto`; clears all filters and presets |

Filter state: Zustand store `usePaintInventoryFilters` ŌĆö exact same shape as `useCollectionFilters`. Resets on navigation (call `clearAll` in a `useEffect` with router location dependency, matching existing pattern).

Filter logic: all active filters AND-combined. Running Low checks `running_low === 1`. Wishlist checks `wishlist === 1`. Multiple presets can be active simultaneously (independent toggles).

### Inline Owned Toggle

Since `Switch` is not installed, use a clickable `Badge` with two states:

- Owned: `<Badge variant="default" role="button" tabIndex={0}>Owned</Badge>` ŌĆö primary color (dark filled)
- Not owned: `<Badge variant="secondary" role="button" tabIndex={0}>Not owned</Badge>` ŌĆö muted

Interaction:
- Click triggers optimistic update following `handleToggleActive` pattern from `CollectionPage.tsx`
- Uses `qc.getQueryData(PAINTS_WITH_RECIPES_KEY)` ŌåÆ `qc.setQueryData` (optimistic) ŌåÆ `useUpdatePaint.mutate({ id, owned: next })` ŌåÆ `onError: rollback + toast.error`
- Keyboard: responds to `Enter` and `Space` (add `onKeyDown` handler)
- `aria-label={`Toggle owned status for ${paint.brand} ${paint.name}`}`
- Does NOT affect `running_low` or `wishlist` on toggle

### Recipe Count Badge

Per row, after Color-family column:

- Always rendered (not hidden when count is 0)
- Count = 0: `<Badge variant="outline">0 recipes</Badge>` ŌĆö not interactive (no link destination makes sense for 0)
- Count > 0: `<Badge variant="secondary" className="cursor-pointer hover:bg-secondary/80" role="link" tabIndex={0}>Used in N recipes</Badge>` ŌĆö navigates to `/recipes?paintId={id}` using TanStack Router `useNavigate`
- `aria-label={`${paint.name} used in ${recipeCount} recipe${recipeCount === 1 ? '' : 's'} ŌĆö click to filter recipes`}` (only on count > 0)

### Table Column Order

| # | Column | Source | Notes |
|---|--------|--------|-------|
| 1 | Name (with swatch) | `paint.name`, `paint.hex_color` | Swatch: 16├Ś16 circle, `rounded-full border border-border`; gray `bg-muted` if no hex |
| 2 | Brand | `paint.brand` | `text-muted-foreground` |
| 3 | Type | `paint.paint_type` | `text-muted-foreground` |
| 4 | Color Family | `paint.color_family` | `text-muted-foreground`; dash `ŌĆö` if null |
| 5 | Owned | `paint.owned` | Clickable badge toggle |
| 6 | Recipes | `paint.recipe_count` | Badge, links to `/recipes?paintId=X` when count > 0 |
| 7 | Actions | ŌĆö | Edit (Pencil) + Delete (Trash2) ghost icon buttons |

Source: CONTEXT.md `### Claude's Discretion` ŌĆö column order confirmed.

### Loading State

Three skeleton rows (`<Skeleton className="h-10 w-full" />`) ŌĆö matches existing `PaintsPage.tsx` pattern.

### Pagination

Use simple list rendering (no TanStack Table pagination) for Phase 7. Seed data ships 6 paints (SEED-03); production collections are expected to stay under 100 paints for hobby-scale users. If row count exceeds 200 at runtime, pagination is deferred to a future phase. This is Claude's discretion call per CONTEXT.md.

### TanStack Router Search Params

The `/recipes` route requires `validateSearch` added to support `paintId` navigation. The `RecipesPage` reads `paintId` from search params on mount and initializes the recipe Zustand filter store. This is new work in Phase 7 ŌĆö first use of `validateSearch` in this app.

## Accessibility

- Filter toggle buttons: `aria-pressed` attribute on Running Low and Wishlist buttons
- Swatch circles: `aria-hidden="true"` (decorative)
- Recipe badge (count > 0): `role="link"` + descriptive `aria-label`
- Owned badge: `role="button"` + `tabIndex={0}` + `aria-label` with paint name
- `MultiSelectPopover` checkboxes: `aria-hidden="true"` on native checkbox (screen reader reads CommandItem label)
- Ghost action buttons: `aria-label` with paint brand + name (existing pattern in `PaintRow.tsx`)
- Table: semantic `<table>` via shadcn Table component (already used)
