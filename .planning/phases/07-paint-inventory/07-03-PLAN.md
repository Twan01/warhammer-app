<objective>
Add typed URL search-param support to the `/recipes` route and wire RecipesPage to consume `paintId` on mount, narrowing the visible recipe list to those that include the selected paint. PINV-05 requires that clicking a "Used in N recipes" badge on the Paint Inventory page navigates to `/recipes?paintId=X` and shows only recipes containing that paint. This is the receiving end of that flow — Plan 04 wires the badge click on the sender side.

Purpose: This is the FIRST use of TanStack Router `validateSearch` in the codebase. RecipesPage currently uses local `useState` for filters and has no URL search-param support. We add a typed schema on the route, export the route object so RecipesPage can call `recipesRoute.useSearch()` with full typing, and wire a one-shot `useEffect` to seed local `paintFilter` state from the URL on first mount. The user can clear it like any normal filter afterward.
Output: Two files modified — `router.tsx` gets one new import + 5 lines of route additions; `RecipesPage.tsx` gets one new import, one new state field, one new `useEffect`, one new query call, one new line in the `filtered` useMemo, and one updated Clear-filters condition.
</objective>
