<objective>
Create the `saveRecipeGraph()` function in `src/db/queries/recipes.ts` that wraps
the entire recipe save flow (create and edit) in a single SQLite transaction, plus
unit tests that verify atomic commit and rollback behavior.

Purpose: Eliminates the risk of partial saves — if any DB operation fails mid-save,
all changes are rolled back and the recipe remains in its previous state.

Output: Exported `saveRecipeGraph()` function + passing test suite.
