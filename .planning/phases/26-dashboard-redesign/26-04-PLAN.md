    5. Click Log Session — a "Log Session" Sheet slides in from the right with a Unit picker, Date input (today by default), Duration input, Notes textarea. Close the Sheet (Cancel).

    DASH-03 (CurrentFocusCard):
    6. Below the header, a full-width card titled "Current Focus" appears IF you have at least one unit marked is_active_project=1. The card shows the unit name, faction name, a colored StatusBadge for the painting stage, painting % with a thin progress bar, and a "Next: …" hint. The card has a colored left border matching the unit's faction color_theme.
       - Edge case: if you have NO active projects, the card shows the muted "No active project — mark one in Projects to focus on it here." placeholder.

    DASH-04 (HobbyPipeline):
    7. A horizontal strip card labeled "Pipeline" is shown below the top stat row. It lists ALL 11 stages: Not Started, Built, Primed, Basecoat, Shaded, Layered, Highlight, Details, Based, Varnish, Done. Each stage shows a small label and a count bubble.
    8. Spot-check 3 stages: navigate to the Collection page, filter to the same status, and confirm the count matches what HobbyPipeline showed. Pick one each from prep tier (e.g. Built), painting tier (e.g. Layered), and done tier (Completed).

    DASH-05 (FactionSummaryCard):
    9. In the "By Faction" section, every faction card shows three things clearly: unit count (e.g. "8 models"), painting progress as `{X}% painted` with a thin horizontal progress bar fill, and a prominent battle-ready points line (e.g. "200 pts battle-ready"). Below that line, a smaller "X pts owned" line is visible but de-emphasized.

    DASH-06 (Recent Activity feed):
    10. At the bottom of the Dashboard, a "Recent Activity" card shows up to 10 rows. Each row has an icon, a label, and a relative time (e.g. "2h", "3d", "1w").
    11. Trigger one event of each type and confirm a new row appears at the TOP of the feed:
        a. Click Quick Add and create a unit (e.g. "Smoke Test Unit"). New row should read "Added Smoke Test Unit" with a Plus icon.
        b. Open the unit you just created from the Collection page and edit a field (e.g. change painting %). Save. Return to Dashboard. New row should read "Updated Smoke Test Unit" with a PenLine icon.
        c. Click Log Session, pick the smoke test unit, set duration 30, click Log Session. New row should read "Session: Smoke Test Unit" with a Paintbrush icon.
        d. Navigate to Battle Log, log a new battle (any opponent_faction, mission, result). Return to Dashboard. New row should read "Battle vs {opponent_faction}: {result}" with a Sword icon.
    12. Click a unit_added or unit_updated row in the feed — UnitDetailSheet for that unit opens as an overlay (NO navigation away from the Dashboard).

    Cleanup: feel free to delete the "Smoke Test Unit" and the test battle log to keep your data clean.
