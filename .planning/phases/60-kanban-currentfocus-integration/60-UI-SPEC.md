## Component Inventory

This phase modifies two existing components. No new shadcn primitives needed.

### KanbanCard (modify)

**Current structure (relevant section):**
```
[recipe name row]         -- text-xs text-muted-foreground (line 97-101)
[next action hint]        -- text-xs italic text-muted-foreground/70 (line 109-113)
```

**After modification:**
```
[recipe name row]         -- unchanged
[workflow context line]   -- NEW: replaces next-action hint when workflow position available
[next action hint]        -- fallback: shown only when no workflow position
```

**Workflow context line specification:**

| Scenario | Display | Example |
|----------|---------|---------|
| Full position (section + next step) | `"--> SectionName: NextStepName"` | `--> Armour: Layer Highlight` |
| Section-only (no step detail) | `"--> SectionName"` | `--> Armour` |
| Flat recipe (no sections, step position) | `"--> step N/M"` | `--> step 4/12` |
| Recipe complete (last step of last section) | `"--> Complete"` | `--> Complete` |
| No workflow position | Fall through to existing `getNextActionHint()` | `--> Apply base coat` |

**Rendering rules:**
- Workflow context line uses same CSS as existing hint: `text-xs italic text-muted-foreground/70`
- The `"--> "` prefix is preserved from the existing hint pattern
- The workflow line replaces (not appends to) the next-action hint -- they are mutually exclusive
- Line truncates with ellipsis at card width via `truncate` class
- `aria-label` not needed -- the text is self-describing

**New optional prop:**
```ts
workflowPosition?: WorkflowPosition | null;
```

### CurrentFocusCard (modify)

**Current structure (relevant section):**
```
[faction name]            -- text-sm text-muted-foreground
[recipe name + palette icon] -- text-xs text-muted-foreground (line 67-70)
[model count + points]    -- text-sm text-muted-foreground tabular-nums
```

**After modification:**
```
[faction name]            -- unchanged
[recipe name + palette icon] -- unchanged
[workflow guidance line]  -- NEW: inserted between recipe name and model count
[model count + points]    -- unchanged
```

**Workflow guidance line specification:**

| Scenario | Display | Example |
|----------|---------|---------|
| Full position (section + step + technique) | `"SectionName: Technique -- step N/M"` | `Armour: Drybrush -- step 4/12` |
| Full position (section + step, no technique) | `"SectionName -- step N/M"` | `Armour -- step 4/12` |
| Section-only (no step detail) | `"SectionName"` | `Armour` |
| Flat recipe (no sections, step position) | `"Step N/M"` | `Step 4/12` |
| Recipe complete | `"Recipe complete"` | `Recipe complete` |
| No workflow position | Line not rendered | (nothing) |

**Rendering rules:**
- Typography: `text-xs text-muted-foreground` -- matches the recipe name row above it
- Icon prefix: `Layers` icon at 12px (from lucide-react) with `gap-1`, matching `Palette` icon pattern on recipe name row
- `tabular-nums` on the step count portion (`N/M`) for alignment stability
- Line truncates with `truncate` class
- Entire line is conditionally rendered -- no empty space when absent

**New optional prop:**
```ts
workflowPosition?: WorkflowPosition | null;
```

## Copywriting Contract

|---------|------|

### Workflow Display Copy

| State | KanbanCard Copy | CurrentFocusCard Copy |
|-------|----------------|----------------------|
| Full workflow (section + next step) | `"--> {sectionName}: {nextStepName}"` | `"{sectionName}: {technique} -- step {N}/{M}"` |
| Full workflow (no technique) | `"--> {sectionName}: {nextStepName}"` | `"{sectionName} -- step {N}/{M}"` |
| Section only | `"--> {sectionName}"` | `"{sectionName}"` |
| Flat recipe with steps | `"--> step {N}/{M}"` | `"Step {N}/{M}"` |
| Recipe complete | `"--> Complete"` | `"Recipe complete"` |
| No position / no recipe | `"--> {getNextActionHint(status)}"` (existing) | Line hidden (existing behavior) |

**Separator patterns:**
- Colon + space (`: `) separates section name from step/technique
- En dash + spaces (` -- `) separates technique from step count in CurrentFocusCard
- Forward slash (`/`) separates current step from total: `4/12`
- Arrow prefix (`--> `) on KanbanCard only -- matches existing hint arrow

## Interaction Contract

### States

| State | KanbanCard Behavior | CurrentFocusCard Behavior |
|-------|--------------------|-----------------------|
| Loading | Card renders without workflow line (null position) | Card renders without workflow line |
| Idle with position | Workflow line displayed below recipe name | Workflow guidance line displayed between recipe name and model count |
| Idle without position | Falls through to existing `getNextActionHint()` | Line not rendered -- no visual change from current |
| Recipe complete | Shows "--> Complete" in italic | Shows "Recipe complete" with Layers icon |
| Drag (KanbanCard only) | Opacity 0.4, workflow line included in dragged card | N/A |

### No New Interactions

This phase is purely informational display. No new:
- Click handlers
- Hover states
- Tooltips
- Modals/sheets
- Form inputs
- Keyboard shortcuts

All interactivity on both cards (drag, actions menu, Log Session button, Open/Log buttons) remains unchanged.

## Graceful Degradation Matrix

Source: CONTEXT.md decisions D-15 through D-19.

| Condition | Workflow Position | KanbanCard | CurrentFocusCard |
|-----------|------------------|------------|-----------------|
| No recipe linked | null | Existing hint via `getNextActionHint()` | No workflow line |
| Recipe linked, no sessions | null | Existing hint | No workflow line (recipe name row shown) |
| Recipe + sessions, no sections (flat) | Step-level only | `--> step N/M` | `Step N/M` |
| Recipe + sessions + sections | Full position | `--> Section: NextStep` | `Section: Technique -- step N/M` |
| Recipe fully completed | Complete marker | `--> Complete` | `Recipe complete` |
