  <what-built>
The complete Phase 15 feature stack (data layer → utilities → infrastructure → sync hook → picker → import dialog → PlaybookTab integration → CollectionPage sibling portals) shipped across Plans 15-00 through 15-05. The live app should now allow:
- A user to sync Wahapedia 40K datasheet data to a local rules.db file
- A user to pick a datasheet for any unit (auto or manual)
- A user to view imported stats / abilities (organized into Core/Faction/Unit sub-groups) / keywords / sources in the Playbook tab
- A user to resolve field-level conflicts via a review dialog when re-importing on a unit with existing data
- All imported data displaying without HTML markup or entities
- Empty-state banner + Last-synced label working correctly per UI-SPEC
- Multi-profile units showing the model-profiles note when applicable
  </what-built>
  <how-to-verify>
**Pre-flight:**

1. Run `pnpm test` and confirm all tests pass (baseline before manual gate).
2. Run `cargo build --manifest-path src-tauri/Cargo.toml` and confirm it exits 0.
3. Delete any pre-existing `rules.db` file from `%APPDATA%\com.hobbyforge.app\` so the empty-state branch is observable. Leave `hobbyforge.db` alone.
4. Run `pnpm tauri dev` to launch the app.

**Step 1 — DS-02 empty-rules-db banner:**
- Open the Collection page → click any unit to open UnitDetailSheet → switch to the Playbook tab.
- EXPECTED: Below the Stats section header, an inline banner reads literally `Sync datasheets to auto-fill stats.` followed by an underlined `Sync now` link. NO "Last synced:" label is shown. NO "Import stats" button is shown.
- Result: PASS / FAIL

**Step 2 — DS-01 sync execution:**
- Click the `Sync now` link in the banner.
- EXPECTED: The link text changes to `Syncing…` while disabled. After ~2-15 seconds (depending on network), a green toast appears reading literally `Datasheets synced`. The banner disappears.
- Verify in DevTools console: NO red error logs. If a network error occurs, the toast should read `Sync failed — check your connection and try again` and the banner should remain.
- Verify on disk: `%APPDATA%\com.hobbyforge.app\rules.db` now exists (and is non-zero bytes).
- Result: PASS / FAIL

**Step 3 — DS-03 Last-synced label appears:**
- After sync completes, with the Playbook tab still open on the same unit, look at the Stats section header.
- EXPECTED: A `Last synced: 04 May 2026` style label is visible (date format `DD Mon YYYY`).
- Result: PASS / FAIL

**Step 4 — DS-05 Import stats button visible after sync:**
- Same view as Step 3.
- EXPECTED: A `variant="outline" size="sm"` button labeled `Import stats` is visible alongside the Last-synced label. A `Re-sync datasheets` icon-only button (RefreshCw icon) is NOT yet visible (it appears only AFTER a unit has a datasheet link).
- Result: PASS / FAIL

**Step 5 — DS-04 picker auto-open + faction filter + search:**
- Close the Playbook tab. Pick a unit that has a faction matching a Wahapedia faction name (e.g. an existing Space Marines unit in your Collection) AND has all 6 stats blank in the Playbook (M/T/Sv/W/Ld/OC = "—").
- Open that unit's Playbook tab.
- EXPECTED: The DatasheetPicker Dialog opens AUTOMATICALLY (without clicking Import stats). DialogTitle reads `Select Datasheet`. DialogDescription reads `Searching {YourFactionName} datasheets`.
- The list below the search box shows datasheets pre-filtered to that faction (~50-200 items).
- Type "term" into the search box.
- EXPECTED: List filters case-insensitively to only datasheet names containing "term" (e.g. Terminator Squad). Type a nonsense string like "zzzzz" and verify the empty state copy `No datasheets found. Try a different search term.` appears.
- Click `Skip` to dismiss without choosing.
- Result: PASS / FAIL

**Step 6 — DS-06 + DS-07 imported stats appear after picker selection:**
- Click the `Import stats` button in the Stats section header.
- EXPECTED: The picker re-opens (manual trigger).
- Pick a known unit, e.g. "Intercessor Squad" (or any other Space Marines datasheet you can verify against Wahapedia).
- EXPECTED: The picker closes. The 6 stat cells populate with the Wahapedia values (e.g. for Intercessors: M 6, T 4, Sv 3, W 2, Ld 6, OC 2). The Playbook tab is now in dirty state — Save Playbook button is enabled.
- The Personal Ability Notes textarea remains EMPTY (the structured Datasheet Abilities collapsible takes over the imported abilities role).
- The Keywords field populates with comma-separated keywords (e.g. "Infantry, Battleline, Imperium, Adeptus Astartes").
- Click Save Playbook. Toast `Playbook saved` appears.
- Close UnitDetailSheet. Re-open the same unit's Playbook tab.
- EXPECTED: Stats persisted. The Stats section header now shows `Re-import` button (replacing `Import stats`) AND a `RefreshCw` icon-only button labeled `Re-sync datasheets`.
- Restart the app entirely (close + relaunch). Re-open the same unit's Playbook tab.
- EXPECTED: Stats still populated; Re-import button still shown (datasheet_id link survived restart).
- Result: PASS / FAIL

**Step 7 — DS-09 Datasheet Abilities collapsible:**
- With the same unit's Playbook tab open after import, scroll down past the Stats block.
- EXPECTED: Below a Separator, a section heading `Datasheet Abilities` is visible with a ChevronDown icon. The collapsible is OPEN by default.
- Inside the collapsible, abilities are grouped by sub-headings: `Core Abilities`, `Faction Abilities`, `Unit Abilities`. Empty sub-groups are HIDDEN.
- Each ability entry shows the ability NAME in bold, then DESCRIPTION below in muted text. Both the name and description are PLAIN TEXT — no `<b>`, `<i>`, `<br>` markup, no `&amp;` or `&nbsp;` entities.
- Click the `Datasheet Abilities` heading to collapse. ChevronDown rotates 180°. Click again to expand.
- Result: PASS / FAIL

**Step 8 — DS-10 Sources list:**
- Same view as Step 7. Scroll to the area below the Datasheet Abilities collapsible.
- EXPECTED: A separate section heading `Sources` is visible. Below it, a `<ul>` list shows the publication name(s) for the imported datasheet (e.g. `Codex: Space Marines`).
- Result: PASS / FAIL

**Step 9 — DS-11 Personal Ability Notes textarea label rename:**
- Same view. Scroll to the textarea section between Sources and Keywords.
- EXPECTED: The textarea is labeled `Personal Ability Notes` (NOT `Abilities`). The textarea is the same `<textarea>` from Phase 9 — typing into it works exactly as before.
- Result: PASS / FAIL

**Step 10 — DS-12 multi-profile note:**
- Use the `Re-import` button to pick a multi-profile unit (e.g. a unit with a sergeant or merged unit; if you can't find one, this step records "N/A — no multi-profile unit available in current rules.db" and the underlying logic is verified by the unit test in Plan 15-05 instead).
- EXPECTED: The text `Additional model profiles available — see Datasheet Abilities for details.` appears below the Stats grid.
- For a single-profile unit (most cases), this note must NOT appear.
- Result: PASS / FAIL / N/A

**Step 11 — DS-08 conflict review dialog:**
- Switch to a unit that has Save Playbook'd values (e.g. M=5 from manual entry) AND has a different incoming datasheet (e.g. M=6 from Wahapedia).
- Click `Re-import`. Pick a datasheet that conflicts.
- EXPECTED: The DatasheetImportDialog opens with title `Review Import` and description `Some fields already have values. Choose which to keep for each field.`
- Each conflicting field row shows `Yours: 5` and `Datasheet: 6` style values plus `Keep` / `Use datasheet` buttons. The `Use datasheet` button is highlighted (variant=default) by default.
- Click `Keep` on one field — its variant flips to default, `Use datasheet` becomes outline.
- Click `Apply import`. EXPECTED: The dialog closes. The Stats grid reflects the user's per-field choices (Keep stayed, Use datasheet replaced). Dirty state active. Save Playbook commits.
- Re-open the picker, re-import. EXPECTED: Click `Discard changes` instead — the dialog closes and the Playbook form remains unchanged (no fields modified).
- Result: PASS / FAIL

**Step 12 — Empty banner re-appears after deleting rules.db:**
- Close the app. Delete `%APPDATA%\com.hobbyforge.app\rules.db`. Restart the app.
- Open ANY unit's Playbook tab.
- EXPECTED: Stats values that were saved BEFORE deleting rules.db remain (they live in unit_strategy_notes, NOT rules.db). The Datasheet Abilities collapsible and Sources list disappear (no datasheet linked → useDatasheet returns null). The empty-rules-db banner reappears with `Sync now` link.
- Result: PASS / FAIL

**Step 13 — Re-sync icon button visible only after link exists:**
- With a unit that has a datasheet_id link AND rules.db populated, observe the Stats section header.
- EXPECTED: A `RefreshCw` icon-only button with aria-label `Re-sync datasheets` is visible alongside `Re-import`. Click it: same `Sync now` flow as Step 2 (button shows Loader2 spinner while pending; success toast on completion).
- Result: PASS / FAIL
