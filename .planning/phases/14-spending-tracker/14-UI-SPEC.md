## Component Inventory

### New: SpendingPage (`src/features/spending/SpendingPage.tsx`)

Layout:
```
<page wrapper: max-w-3xl mx-auto p-8 flex flex-col gap-12>
  <hero card>
    <Card with ring-2 ring-faction-accent rounded-lg px-6 py-6>
      <span class="text-sm text-muted-foreground">Total Hobby Spend</span>
      <span class="text-3xl font-semibold">{formatCurrency(totalPence)}</span>
    </Card>
  </hero card>
  <section>
    <h2 class="text-xl font-semibold mb-4">Breakdown</h2>
    <shadcn Table>
      <TableHeader>Faction | Spend</TableHeader>
      <TableBody>
        {factions 4 rows — always shown, £0.00 if empty}
        <separator row>
        {Paints row — owned paints only}
      </TableBody>
    </shadcn Table>
  </section>

```

Loading state: Three `<Skeleton>` blocks — one at hero card dimensions (h-20 w-full), one at section heading dimensions (h-6 w-32), one at table dimensions (h-40 w-full). Use `<Skeleton className="rounded-lg" />` matching existing dashboard pattern.

Error state: Inline error message below hero card position. Text: "Could not load spending data. Restart the app or try again." No full-page takeover.

## Interaction Contract

### Spending page

- Page renders with Skeleton loading state while `useSpendingStats` resolves
- On data load: hero card fades in at natural render (no animation — this is passive data, not a hero counter)
- Breakdown table rows: no hover interaction (read-only)
- Faction rows always rendered in the same order as the DB factions table (no sorting by spend amount in Phase 14)
- "Paints" row is always the final row, visually separated from faction rows by a `<Separator />` row or a `border-t` class on the Paints `<TableRow>`

### UnitSheet price input

- Field is inside the collapsible "More details" section — no change to expand/collapse behavior
- Input type="number", step=1 (integer pence only)
- Display value in form: raw integer pence (e.g. 1250)
- Formatted currency display only in UnitDetailSheet read-only view and Spending page
- On form reset: field clears to empty string (null) when unit is null (create mode)

### PaintSheet price input

- Field appended after existing fields, before SheetFooter
- Same integer pence discipline as UnitSheet
- No collapsible section in PaintSheet — field is always visible

## Currency Formatting

`formatCurrency(pence: number | null, locale?: string, currency?: string): string`

- Default locale: `'en-GB'`, default currency: `'GBP'`
- Returns `'—'` when pence is null or undefined
- Uses `Intl.NumberFormat` with `{ style: 'currency', currency }` and divides pence by 100
- Location: `src/lib/formatCurrency.ts` (new utility)
- Call sites: SpendingPage, UnitDetailSheet Details tab, any future display
