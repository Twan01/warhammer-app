## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot size (StatusBadge dot is 4×4px) |
| sm | 8px | Compact element spacing, badge internal padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card gap |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: PageHeader bottom padding is `pb-6` (24px) — matched to existing inline pattern across all 9 pages. StatCard progress bar height is 2px (not on the 8-point scale — fixed design element, not a spacing token).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (text-base) | 400 (regular) | 1.5 | Default prose, card body text |
| Label | 14px (text-sm) | 400 (regular) | 1.4 | Muted labels, subtitles, StatusBadge text, StatCard label |
| Heading | 20px (text-xl) | 600 (semibold) | 1.25 | Section headings, card titles |
| Display | 28px (text-3xl) | 600 (semibold) | 1.2 | Page titles (PageHeader h1), StatCard numeric values |

Notes:
- `text-3xl font-semibold tracking-tight` is the locked pattern for `<h1>` in PageHeader — sourced from existing inline header pattern confirmed across all 9 pages in codebase scan.
- StatCard value span stays `text-3xl font-semibold tabular-nums` (existing implementation).
- `tabular-nums` class applied to all numeric displays: StatCard values, any trend/progress numbers.
- Font family: `'Geist Variable', ui-sans-serif, system-ui, sans-serif` — from `--font-sans` in globals.css.

### Phase 25 New Color Value (Battle Gold)

| Token | CSS Variable | Value | Format | Tailwind Utility |
|-------|-------------|-------|--------|-----------------|
| Battle Gold | `--battle-gold` | `oklch(0.78 0.17 85)` | oklch | `bg-battle-gold` / `text-battle-gold` |

Resolved value: `oklch(0.78 0.17 85)` — warm amber-gold (~#D4A843 approximate), readable against dark zinc backgrounds at both `text-sm` and `text-base`. Chosen to be visually distinct from the muted zinc palette without clashing with `--faction-accent`.

**Battle Gold is defined but NOT applied in Phase 25.** Phases 26+ apply it to HobbyPipeline "Done" tier, StatusBadge Done tier, and Army readiness panel.

### 60 / 30 / 10 Distribution

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--forge-black` / `--background` | Page backgrounds, full-bleed app surface |
| Secondary (30%) | `--panel-elevated` / `--card` + `--panel-surface` / `--secondary` | Cards, sidebar, nav items, panel backgrounds |
| Accent (10%) | `--faction-accent` | Active nav items, ring highlights, StatCard progress bar fill, DashboardEmptyState icon; limited to interactive/focus indicators |
| Destructive | `--destructive` | Delete buttons and confirmation dialogs only |
| Battle Gold (defined, held) | `--battle-gold` | Reserved for Phase 26+ Done/Battle-Ready tier indicators |

Accent reserved for: active nav item background + text, faction ring highlights, StatCard progress bar fill, DashboardEmptyState wordmark icon. Never used for generic interactive elements.

## Interaction Contracts

### PageHeader
- Static display component — no interactive behaviour.
- `actions` slot renders any `ReactNode` — interactive children (buttons, dropdowns) are caller-supplied and follow their own interaction rules.
- No hover, focus, or active states on the header container itself.

### StatusBadge
- Static display component — no interactive behaviour.
- Renders inline — safe for use inside `<td>`, `<div>`, flex rows.
- Dot size: 4×4px (`h-1 w-1`) — always visible, never animated.
- Text is `text-sm` — legible at standard table row heights.

### StatCard (enriched)
- Progress bar animates width via `transition-all duration-500` — purely CSS, no JS animation.
- Icon is decorative — no interactive state, no tooltip required.
- Trend value is display-only text — no interactive state.
- `animate` prop behavior unchanged from Phase 11 implementation.
