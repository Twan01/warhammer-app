## Interaction Contracts

### LeaderAttachmentSheet

**Trigger:** Click Link2 button on a leader-eligible unit row in ArmyListDetailSheet.

**Sheet layout (top to bottom):**
1. **Header:** Link2 icon + "{unit_name} -- Leader Attachment" title, "{faction_name} leader targets" description
2. **Current attachment banner** (conditional): If leader is already attached, show a highlighted row with the current target name + "Detach" destructive button. Separated by a subtle border-b.
3. **Valid targets list:** Each target is a bordered card (rounded-md border p-3) with:
   - Left: target unit name (14px font-medium) + points Badge (variant="secondary", 12px)
   - Right: "Attach" Button (variant="outline", size="sm", h-7 text-xs)
4. **Empty state:** Shown when no valid targets exist in the army list.

**States:**

| State | Visual Treatment |
|-------|-----------------|
| Target available | "Attach" button enabled (variant="outline") |
| Target already led by another leader | "Attach" button disabled + Tooltip: "Already led by {leader_name}" |
| Leader already attached to this target | Row highlighted with bg-secondary, "Detach" button (variant="destructive") shown instead of "Attach" |
| No valid targets in list | Empty state message |
| No faction selected | Guard message: "No faction selected for this list." |
| Mutation pending | Button shows disabled state during mutate |

**Close behavior:** Sheet closes on overlay click, Escape key, or explicit close. Does NOT close on attach/detach (user may want to change attachment).

### ArmyListUnitRow -- Leader Trigger

**Visibility logic:** Show trigger when unit name matches a `leader_name` in synced_leader_targets for the list's faction AND unit is not an Epic Hero.

| Unit State | Trigger Appearance |
|------------|-------------------|
| Leader, not attached | Button: Link2 icon + "Attach" (variant="outline", h-7 text-xs) |
| Leader, already attached | Button: Link2 icon + "Attached" (variant="secondary", h-7 text-xs) — clicking opens sheet showing current attachment |
| Not a leader | No trigger shown |

### ArmyListUnitRow -- Visual Grouping

**Indented leader row** (when `isIndentedLeader === true`):
- Left padding: pl-8 (32px) on the first TableCell
- Left border: 2px solid var(--faction-accent) on the TableRow via `border-l-2` + inline style
- Row background: no change (inherits table stripe if any)
- All other columns render normally (points, status, actions)

**Target row with leader badge** (when unit has a leader attached):
- Below the unit name, render: `<Badge variant="outline" className="text-xs mt-0.5"><Link2 className="h-3 w-3 mr-1" />Leader: {leaderName}</Badge>`
- Badge is static (not clickable) — keeps interaction simple
- Pattern matches existing enhancementName badge placement

### Sort/Group Contract

The `groupUnitsWithLeaders()` function produces a `GroupedUnit[]` array:
- Each entry has `{ unit: ArmyListUnitRow, isIndentedLeader: boolean }`
- Target units appear at their normal insertion-order position
- Attached leader rows appear immediately after their target (removed from original position)
- Unattached units maintain original `ORDER BY alu.id ASC` position
- Function is pure, returns new array, never mutates input

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Trigger button | `aria-label="Attach {unit_name} as leader"` / `aria-label="{unit_name} attached to {target_name}"` |
| Sheet | SheetTitle + SheetDescription provide accessible name/description |
| Disabled buttons | Wrapped in `<span className="inline-flex">` for Tooltip to fire (Radix disabled element workaround) |
| Keyboard | Sheet focus-trapped by Radix. Escape closes sheet. Tab navigates target rows. |
| Leader grouping | Visual only -- screen readers see rows in DOM order (target then leader) which is correct reading order |
