## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(var(--background)) — dark: hsl(240 10% 3.9%) | Page background |
| Secondary (30%) | hsl(var(--card)) — dark: hsl(240 10% 5.9%) | StatCards, Chart container card |
| Accent (10%) | var(--faction-accent) — default: #71717a zinc-500 | Hero total card ring ONLY (ring-2 ring-faction-accent on SpendingPage hero); NOT applied to chart bars |
| Destructive | hsl(var(--destructive)) | No destructive actions in this phase |

Accent reserved for: The SpendingPage hero "Total Hobby Spend" card ring (pre-existing pattern). Chart bars use a muted chart color from shadcn's default palette, NOT faction-accent — chart colors should not shift with faction changes.

Chart bar fill: Use `hsl(var(--chart-1))` from shadcn chart token — a consistent muted blue/teal that remains stable across faction theme changes. This is set by `shadcn add chart` alongside `chart.tsx`.

Muted note text: `text-muted-foreground` — used for the "Add purchase dates..." hint below the chart when all bars are zero.

Source: globals.css (color tokens), CONTEXT.md (Claude's Discretion — bar fill color), SpendingPage.tsx (faction-accent ring pattern).

## Layout Contracts

### Dashboard HOBBY HEALTH section

Position: Between the PROGRESS section and the BY FACTION section.

```
PROGRESS
[ Painting % ] [ Assembly % ] [ Basing % ]

HOBBY HEALTH                               ← new section, same gap-12 from Progress
[ Hobby Velocity    ] [ Painting Streak ]  ← grid-cols-2 gap-6, full width

BY FACTION
...
```

Section wrapper pattern (mirrors PROGRESS exactly):
```
<section className="flex flex-col gap-4">
  <p className="text-sm font-semibold uppercase tracking-widest text-muted-foreground">
    Hobby Health
  </p>
  <div className="grid grid-cols-2 gap-6">
    <StatCard value={velocityString} label="Hobby Velocity" animate={false} />
    <StatCard value={streakString} label="Painting Streak" animate={false} />
  </div>
</section>
```

Notes:
- Velocity value is a formatted string (`"3.1"` + sub-label `"units/month"`) — StatCard animate={false} because it is not a plain integer
- Streak value is a string (`"14 days"`) — StatCard animate={false}
- Both cards always render even when sessions = 0 (shows `"0.0"` / `"0 days"`)
- Loading state: 2× `<Skeleton className="h-24 w-full" />` inside the grid

StatCard sub-label for velocity: The existing StatCard component only has `value` and `label` props. The `"units/month"` suffix is rendered as part of the label string: `label="Hobby Velocity · units/month"` — no prop extension required. The dot separator (·) is the same visual pattern used in other multi-part labels.

### SpendingPage Monthly Trend section

Position: Between the hero total card and the Breakdown section.

```
[ Total Hobby Spend hero card          ]

Monthly Trend                           ← new section heading
[ Bar chart — 12 months               ]  ← h-60 (240px height)
  "Add purchase dates..." note if zero  ← text-sm text-muted-foreground, text-center

Breakdown
[ Faction | Spend table                ]
```

Section wrapper:
```
<section className="flex flex-col gap-4">
  <h2 className="text-base font-semibold">Monthly Trend</h2>
  <SpendTrendChart data={monthlyData} />
</section>
```

SpendTrendChart internal layout:
- `ChartContainer` with `config` matching `chart-1` token and a 240px height (`className="h-60"`)
- `BarChart` with `data` array of `{ month: string, pence: number }` — 12 entries, rolling window
- X-axis: `XAxis dataKey="month"` — short month format `"Jan"`, `"Feb"` etc. (3-char abbreviation, no year unless crossing calendar year boundary where year displayed as `"Jan '25"`)
- Y-axis: `YAxis tickFormatter={(v) => formatCurrency(v)}` — tick values formatted as currency
- Grid lines: `CartesianGrid vertical={false}` — horizontal grid lines only (clean linear aesthetic)
- Bars: `<Bar dataKey="pence" fill="var(--color-chart-1)" radius={[3, 3, 0, 0]} />` — 3px top corner radius, square bottom
- Tooltip: `<ChartTooltip content={<ChartTooltipContent formatter={(v) => formatCurrency(Number(v))} />} />`
- Bar width: default (recharts auto-sizes based on container — no fixed barSize override)
- The "Add purchase dates" note renders as a separate `<p>` below the `ChartContainer` when all 12 pence values are 0

## Interaction Contracts

### HOBBY HEALTH section (Dashboard)

| Interaction | Behaviour |
|-------------|-----------|
| Render | Always rendered once stats data resolves — no conditional hide when sessions = 0 |
| Loading | Shows 2× Skeleton h-24 inside grid-cols-2 while `isLoading` |
| Error | Not independently sectioned — falls under DashboardPage error branch (pre-existing full-page error) |
| Data update | Invalidating `["hobby-analytics"]` key triggers refetch — section updates reactively when a new session is logged from any page |
| Animation | No useCountUp animation — both values are strings; StatCard animate={false} |

### Monthly Trend chart (SpendingPage)

| Interaction | Behaviour |
|-------------|-----------|
| Render | Always rendered within the `!isEmpty` branch (chart section is never conditionally hidden) |
| Loading | Shows `<Skeleton className="h-60 w-full rounded-lg" />` during load |
| Hover tooltip | Recharts default hover + shadcn ChartTooltip — shows month label + formatted currency amount |
| Zero state | 12 bars render at zero height; `"Add purchase dates..."` note appears below chart |
| Bar click | No action — read-only analytics display |
| X-axis overflow | 12 bars with 3-char labels fit comfortably in max-w-3xl at desktop width; no rotation needed |

## State Machine: SpendingPage with chart

The page now has two independently rendered sections in the data branch:

```
isEmpty = totalPence === 0 && factionBreakdown.length === 0 && paintsPence === 0
```

When `isEmpty = true`: empty state (Receipt icon + copy). No chart shown.
When `isEmpty = false`:
1. Hero card (ring-2 ring-faction-accent)
2. Monthly Trend section (SpendTrendChart — always present)
3. Breakdown section (Table — always present)

The chart uses its own `monthlyData` derived from `useHobbyAnalytics` (or a dedicated query in the same hook). It does NOT block the hero card or breakdown from rendering if analytics data is still loading — the chart renders its own Skeleton independently while the rest of the page is visible.

## Data → Display Mapping

| Data | Source | Display Format |
|------|--------|----------------|
| Hobby Velocity | `painting_sessions` — distinct unit_ids ÷ months between first/last session | String `"3.1"` with 1 decimal place; StatCard value prop |
| Painting Streak | `painting_sessions` — consecutive calendar days ending today (via dates.ts) | String `"14 days"` or `"0 days"`; StatCard value prop |
| Monthly spend bars | `units.purchase_price_pence + units.purchase_date` + `paints.purchase_price_pence + paints.purchase_date` — NULL purchase_date rows excluded — bucketed by calendar month | Array of 12 `{ month: string, pence: number }` for last 12 calendar months |
| Chart Y-axis ticks | pence integers from monthly data | `formatCurrency(pence)` — e.g. `"£12.50"` |
| Chart tooltip | pence integer for hovered bar | `formatCurrency(Number(value))` |

Integer pence discipline: spend data travels as integer pence through the entire data pipeline. `formatCurrency` is the ONLY /100 division site — no raw pence displayed in the UI.

## Anti-Patterns (Do Not Do)

| Anti-pattern | Correct approach |
|-------------|-----------------|
| Conditionally hiding HOBBY HEALTH when no sessions | Always render the section — show `"0.0"` / `"0 days"` |
| Conditionally hiding Monthly Trend chart | Always render inside the !isEmpty branch — show zero bars + note |
| Using `animate={true}` on string StatCard values | `animate={false}` for velocity and streak — they are strings, not integers |
| Applying faction-accent to chart bars | Use `var(--color-chart-1)` — chart colors must not shift with faction theme |
| Nesting SpendTrendChart Dialog/Tooltip inside another Radix portal | SpendTrendChart is NOT a Radix component — no portal nesting issue |
| Floating-point pence display | All math stays in integer pence until `formatCurrency()` |
| `hover:bg-muted/50` on StatCard | StatCard uses shadow-sm with no hover (Pattern 5 anti-pattern from Phase 16) |
